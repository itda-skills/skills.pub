# kacem-tender-fetch 사용 가이드

군인공제회(KACEM) 입찰 게시판에서 공고 목록을 수집하고, 첨부 ZIP 파일을 받아 모집공고 파일을 자동으로 식별합니다.
**Claude에게 말로 부탁하면 됩니다** — "군인공제회 최근 공고 받아줘", "지난 한 달 공고 모아줘"처럼요.

---

## 자주 쓰는 요청

| 하고 싶은 것 | 이렇게 말하세요 |
|---|---|
| 최근 공고 받기 | "군인공제회 최근 공고 받아줘" |
| 저장 위치 지정 | "군인공제회 최근 공고를 ./mmaa-2026-04 폴더에 받아줘" |
| 날짜 이후 공고 전부 | "2026년 3월 30일 이후 공고를 5페이지 분량까지 모아줘" |
| 새 공고만 | "새로 올라온 공고만 다운받아줘" |
| 개수 제한 | "최근 공고 10건만 받아줘" |
| 이미 받은 것도 다시 | "이미 받은 공고도 포함해서 다시 받아줘" |
| 상세 진행 상황 보기 | "공고 받으면서 자세히 로그 보여줘" |

---

## 어떻게 동작하나요

1. 군인공제회 입찰 게시판(공동주택감리 카테고리)에서 공고 목록을 수집합니다.
2. 각 공고의 첨부 ZIP 파일을 다운로드하고 압축을 해제합니다.
3. 압축 해제된 파일 중 "모집공고" 파일(`core_document`)을 자동으로 식별합니다.

기간 범위를 지정하면 해당 날짜 이전 글을 만나는 순간 수집을 멈춥니다. 이미 받은 공고는 건너뛰고, 새 공고만 추가로 받습니다.

---

## 저장 결과물

공고를 받으면 아래 구조로 파일이 생깁니다.

```
{저장폴더}/
├── _index.json                    # 전체 공고 목록
└── {글번호}_{제목}/
    ├── meta.json                  # 게시글 정보 + 핵심 문서 경로
    ├── attachment/
    │   └── {글번호}_attachment.zip   # 원본 ZIP
    └── extracted/
        ├── 모집공고_xxx.hwp       # 핵심 문서 (core_document)
        └── 기타_파일들.*
```

---

## kacem-tender-extract와 함께 쓰기

공고를 받은 뒤 "사업비 정리까지 해줘"라고 말하면, `kacem-tender-extract` 스킬이 이어서 핵심 문서를 읽고 사업개요·사업비를 표·JSON으로 정리합니다.

---

## 안 될 때

| 증상 | 원인 / 해결 |
|---|---|
| 게시판에 접속이 안 된다 | 네트워크 상태를 확인하거나 잠시 후 다시 시도하세요 |
| 파일이 이미 있어 건너뛴다 | 다시 받으려면 "이미 받은 것도 다시 받아줘"라고 말하세요 |
| 공고가 없다고 나온다 | 지정한 날짜 이후 게시된 공고가 없을 수 있습니다. 날짜 범위를 조정하거나 페이지 수를 늘려 보세요 |

---

## 참고

- 저장 위치를 따로 말하지 않으면 현재 작업 폴더에 저장합니다.
- 게시글 디렉토리 이름은 URL의 글번호를 기준으로 만들어집니다(목록에 표시되는 번호와 다를 수 있으나 고유합니다).
- 중간 캐시 파일은 내부 경로에 저장되며, 최종 결과만 지정 폴더에 남습니다.
