# realty-deals 사용 가이드

국토교통부 부동산 실거래가 12개 유형(아파트·오피스텔·연립다세대·단독다가구·토지·상업업무용·공장창고·분양입주권의 매매/전월세)을 한 번에 수집합니다.
**Claude에게 말로 부탁하면 됩니다** — "강남구 2026년 1~6월 아파트 매매 실거래 모아줘", "분당 연립다세대 매매 2025년 데이터 CSV로 줘"처럼요. 터미널 명령을 직접 입력할 필요는 없습니다.

---

## 처음 설정하기

이 스킬은 공공데이터포털 API 키가 필요합니다. 조회할 거래 유형별로 개별 활용신청을 해야 합니다.

1. [data.go.kr](https://www.data.go.kr)에 가입합니다.
2. 원하는 실거래가 서비스를 개별 활용신청합니다.
   - 아파트 매매: https://www.data.go.kr/data/15126469/openapi.do
   - 아파트 전월세: https://www.data.go.kr/data/15126474/openapi.do
   - (오피스텔·연립다세대·토지 등도 동일하게 유형별로 신청)
3. 발급받은 API 키를 환경변수에 등록합니다.

```dotenv
KO_DATA_API_KEY=발급받은_키
```

> 자동승인이라도 동기화에 5~30분이 걸릴 수 있습니다. 신청 직후 오류가 나면 잠시 기다렸다가 다시 요청하세요.

## 알아두면 좋은 점

- **전량 수집**: 구(舊) 스킬은 첫 페이지만 가져오는 버그가 있었습니다. 이 스킬은 국토교통부 API의 전체 건수(`totalCount`)를 기준으로 모든 페이지를 끝까지 수집합니다.
- **다월 범위 자동 처리**: "1월부터 6월까지"처럼 기간 범위를 말하면 달별로 자동으로 순회하며 수집합니다.
- **이전 스킬 사용자**: 기존 `itda-gov/skills/realestate`를 쓰던 분은 `KO_DATA_API_KEY`를 그대로 사용하면 됩니다. 기존 4유형(아파트 매매·전월세, 오피스텔 매매·전월세)은 동일하게 동작합니다.

## 안 될 때

| 증상 | 원인 / 해결 |
|------|-------------|
| "API 키가 없다"는 안내 (`config`) | `.env`에 `KO_DATA_API_KEY`가 비어 있음. 위 "처음 설정하기" 확인 |
| "지역을 모르겠다"는 안내 (`args`) | 지역명을 다시 말하거나 "지원하는 지역 목록 알려줘"로 확인 |
| 권한 오류 (`api`, 코드 20/30) | 활용신청 직후 동기화 대기 중. 5~30분 후 다시 요청 |
| API 서비스 오류 (`api`) | 공공데이터포털 활용신청 승인 상태를 확인하고 다시 요청 |
