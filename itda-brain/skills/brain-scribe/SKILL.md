# brain-scribe (v1.5 — 스캐폴드)

> **상태: v1.5 스캐폴드.** 아래는 확정된 orchestration outline 이며, 구현·라이브 검증은 v1 안정화 후 진행한다(SPEC-BRAIN-VERTICAL-001 REQ-041, #1122 비범위).

"읽어주는 뇌"를 넘어 **"일하는 뇌"**의 마지막 조각 — 규약을 배워 문서를 **쓰기까지** 한다.

---

## Claude 오케스트레이션 지시서 (outline)

> [HARD] **초안 폴더 격리.** 생성 문서는 `초안/` 폴더에만 만든다. **원본 폴더에 직접 생성하지 않는다**(원본 불가침 연장 — SPEC EXC-2). 사용자 승인 후에만 사람이 원본 폴더로 옮긴다.
> [HARD] **근거 없는 값 금지.** 숫자·사실은 위키 근거에서만 가져온다. 근거 없는 필수 칸은 지어내지 말고 사이드카에 "추정"으로 명시한다.

### 관문1 — 형식: 규약에서 서식 복제

생성할 문서 유형의 `규약/양식/{유형}.md`(모범 원본 지정)를 읽어 **레이아웃·서식을 복제**한다. 규약이 없으면(신규 유형) brain-ingest 신규 유형 학습 루프로 먼저 역공학한다.

### 관문2 — 내용: 위키 근거에서 값 채우기

문서에 들어갈 수치·날짜·대상 정보는 업무DB 위키에서 가져오되, 각 값의 근거 원본을 추적한다. 같은 대상에 모순이 있으면(검수리포트 참조) 어느 값을 쓸지 사용자에게 확인한다.

### 관문3 — 근거 사이드카 (투명성)

근거가 없는 **필수 칸**은 빈칸으로 두거나 지어내지 않고, `{파일}.근거.md` 사이드카에 **칸 단위로 "추정"**을 명시한다("이 값은 위키에 근거가 없어 추정입니다 — 확인 필요").

### 관문4 — 승인 게이트

초안 + 사이드카를 사용자에게 제시하고, 승인 전까지 원본 폴더로 이동하지 않는다.

## 원칙

- **형식은 규약, 숫자는 근거** — 서식은 모범 원본에서, 값은 위키에서. 둘을 섞지 않는다.
- **투명성 > 완결성** — 근거 없는 칸을 채워 "완성된 것처럼" 보이게 하지 않는다. "추정"이 정직하다.
