# 질답 E2E 프로토콜 (REQ-060) — surface_question 실사격 채점

brain-fixture 로 만든 데이터셋을 brain-build 로 업무DB(뇌)화한 뒤, **인사이트가 실제로 "종합해야만
답이 나오는지"** 를 질답으로 실사격 검증한다. `qa_sheet.py` 가 질문지(응답자용)와 채점지(채점자용)를
분리 산출하고, 이 문서는 그 두 파일로 채점을 진행하는 절차다.

```
insights[] ──qa_sheet.py──▶ qa-questions.md (응답자)  +  qa-key.json (채점자, 비공개)
                                   │
         업무DB 폴더만 연 zero-context 응답자 에이전트 ──▶ qa-answers.md
                                   │
                       채점자(사람/오케스트레이터): qa-answers ↔ qa-key 대조 ──▶ 채점표
```

## ① 응답자 계약 (zero-context)

- **업무DB 폴더만 열람**한다. **원장(ledger.json)·정답지(answer_sheet)·검수리포트·qa-key.json 에 접근하지 않는다** — 접근하면 실사격이 성립하지 않는다(정답을 이미 본 셈).
- 업무DB 의 `CLAUDE.md` **질의응답 규율을 따른다**(brain-build 가 심은 Layer 3 스키마). 폴더를 연 Cowork/에이전트가 자동 적용.
- **답마다 근거 파일 경로를 명시**한다(업무DB 상대경로). 근거 없는 값을 지어내지 않는다.
- **미적재는 미적재로** 답한다 — 폴더에 없는 정보는 "해당 문서 없음/판단 불가"로. 추측 창작 금지.
- 질문지(`qa-questions.md`)에는 번호·tier·질문만 있다. 수치·근거·결론은 없다(정답 미누출은 qa_sheet 가 기계 검증).

## ② 실행 절차

1. `qa_sheet.py <ledger> --out-dir <dir>` 로 `qa-questions.md`·`qa-key.json` 을 만든다(채점지는 채점자만 본다).
2. 응답자 에이전트를 **새 컨텍스트**로 띄우고 업무DB 폴더 + `qa-questions.md` 만 준다.
3. 질문지 **번호 순서대로** 답하게 하고, 답변을 `qa-answers.md` 로 수집한다(문항별 답 + 근거 경로).

## ③ 채점 기준 (qa-answers ↔ qa-key 대조)

각 문항을 qa-key 의 `expected` 와 대조한다:

- **결론 일치** — 답이 `expected.result` 의 수치(numeric: `value`) 또는 관계(relation: 방향)를 포함하는가. numeric 은 값(오차 허용은 tier 규칙 참조), relation 은 부등호 방향.
- **근거 포괄** — 답이 인용한 근거 경로가 `expected.evidence`(≥2 문서)를 **포괄**하는가. 한 문서만 인용하면 "합성 실패"(단일 문서로 답한 것).
- **창작 0** — 원장/업무DB 에 없는 값을 지어내지 않았는가. 지어낸 수치가 있으면 감점.

## ④ tier별 합격선 (제안 — 채점자가 근거와 함께 확정)

| tier | 합격 기준 |
|------|-----------|
| **tier 1** | **전건** — 결론·근거(≥2)·창작0 모두 충족. 쉬운 2문서 합성이므로 만점 기대. |
| **tier 2** | **전건** — 수치 결론이 `expected.value` 와 일치(반올림 자리 내), 근거 포괄, 창작0. |
| **tier 3** | **결론 방향 일치면 부분 인정** — 정확한 수치까지 못 맞춰도 결론의 방향(예: "렌탈이 회수기간 관점에서 특정 시점부터 불리")이 맞고 근거를 포괄하면 부분 점수. 방향까지 틀리면 불합격. **부분 인정 사유를 채점표에 명시**한다. |

합격선은 강의·검증 목적에 따라 채점자가 조정할 수 있다 — 조정 시 그 기준을 채점표 상단에 적는다.

## ⑤ 채점표 양식

```markdown
# 질답 채점표 — <업무DB 이름> (<채점일>)
합격선: tier1 전건 / tier2 전건 / tier3 방향일치 부분인정   ← 조정 시 여기 명시

| # | id | tier | 결론 일치 | 근거 포괄(≥2) | 창작0 | 판정 | 비고 |
|---|----|------|----------|--------------|-------|------|------|
| 1 | I1 | 1 | ✅ | ✅ (2/2) | ✅ | 합격 | |
| 2 | I2 | 1 | ✅ | ✅ | ✅ | 합격 | |
| 3 | I3 | 2 | ✅ 116.8% | ✅ | ✅ | 합격 | |
| 4 | I4 | 2 | ✅ 38.5% | ⚠️ 1문서만 인용 | ✅ | 불합격 | 합성 실패 — 매출표만 인용 |
| 5 | I5 | 3 | 방향 일치(수치 근접) | ✅ | ✅ | 부분합격 | 13.7개월 대신 "약 1년" |

종합: tier1 2/2 · tier2 1/2 · tier3 부분1 — 인사이트 합성 검출력 …
```
