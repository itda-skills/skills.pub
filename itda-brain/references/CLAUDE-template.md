# {{뇌 이름}} 업무DB 운영 규칙

이 폴더는 `{{소스 폴더}}`의 원본 문서를 근거 추적 가능한 형태로 정리한 업무 데이터베이스(**뇌**)다. 위 머리말(frontmatter)은 이 뇌의 자기서술 메타로, 파일시스템 스캔만으로 뇌를 발견·식별할 수 있게 한다 — 값을 임의로 지우지 않는다.

## 적재 규칙

1. **원본 불가침**: `{{소스 폴더}}`의 원본 파일은 절대 수정·이동·삭제·이름변경 하지 않는다. 업무DB는 원본을 읽어 만든 2차 정리본이다.
2. **주제별 위키**: 문서는 주제별(거래처·계약·규정·매출·재무·재고·인사·회의·결정사항 등, 실제 도메인에 맞게) `.md` 페이지로 정리한다.
3. **머리말 필수**: 모든 페이지 상단에 `출처:`(근거가 된 원본 파일 경로 목록)와 `최종갱신:` 필드를 둔다.
4. **근거 표기 필수**: 본문의 모든 수치·금액·날짜·결정사항 뒤에 근거 원본 파일 경로를 괄호로 표기한다. 예: `19,800원 (계약/대영인쇄_연간공급계약_2026.docx)`.
5. **경로 표기**: 근거 경로는 `{{소스 폴더}}` 기준 상대경로로 적는다.
6. **페이지 간 링크**: 관련 페이지는 `[[페이지명]]`으로 연결한다.
7. **문제 파일 기록**: 읽을 수 없거나 정상 문서가 아닌 파일(손상·잠금 임시파일 `~$*`·빈 문서)은 건너뛰지 않고 `문제파일.md`에 사유와 함께 기록한다.
8. **모순 보존**: 서로 다른 문서가 같은 대상에 대해 다른 값을 담고 있으면 임의로 하나를 고르지 않는다. 양쪽 값과 근거를 모두 남기고 `검수리포트.md`에서 대조한다.
9. **외부 자료 분리**: 사외에서 수집한 자료(웹·공공데이터 등)는 `외부/` 폴더에 `출처:`·`수집일:`을 명시해 사내 원본과 분리 보관한다.

## 질의 응답 규칙 (중요)

- **질문에는 업무DB에 적재된 근거로만 답한다.** 답변에는 근거 파일 경로를 함께 제시한다.
- **업무DB에 없는 내용은 지어내지 않고 "미적재"라고 밝힌다.** 추측으로 값을 채우지 않는다.
- 같은 대상에 값이 여러 개 있을 때는 어느 문서의 값인지 구분해 제시하고, 알려진 모순이 있으면 `검수리포트.md`를 함께 안내한다.
- 최신성 판단이 필요하면 파일명("최종"·"진짜최종"·"개정안" 등)이 아니라 **문서 내부의 날짜·시행일**을 기준으로 한다.
- **검색이 빈손이어도 "없음"으로 단정하지 않는다.** `문제파일.md`의 **내용 검색 불가** 파일(스캔 PDF·이미지 등 텍스트 추출 불가)은 텍스트 검색이 보지 못한다 — 빈손 답변 시 그 건수를 함께 고지한다("단, 내용 검색 불가 N건은 이 검색에 포함되지 않음").

## 구두 주장 처리 규칙 (검증 적재)

대화 중 사용자가 업무DB에 없는 사실·판단 기준을 구두로 말하면(예: "그 단가 재계약해서 21,000원이 맞아", "모순 나면 발주서를 믿어") 조용히 흘리지도, 그대로 적재하지도 않는다. **채팅 발언은 지식이 아니라 적재 요구 신호(주장)다** — 뇌에는 검증을 통과한 원본 실물만 들어가며, 이 폴더는 어떤 경우에도 1차 기록이 되지 않는다(적재 규칙 1의 "2차 정리본" 불변).

1. **등록 — 근거를 묻기 전에 먼저**: 주장을 `검증대기.md`에 기록한다 — 주장 내용 · 발화 맥락(일시·어떤 대화에서) · 요구한 근거 유형 · 요구일. 등록을 사용자 답변 뒤로 미루지 않는다(대화가 끊기면 주장이 증발한다).
2. **근거 요구** (주장 성격에 따라 3갈래):
   - **사내 문서가 있을 주장** → 실물(계약서·품의서·메일 등)을 소스 폴더에 넣어 달라고 요청한다.
   - **외부에서 확인 가능한 주장** → 외부 자료로 수집·대조해 `외부/`에 격리 적재한다(적재 규칙 9).
   - **문서가 원래 없는 관례·판단 기준** → `초안/`에 **구두확인서 초안**(제목·내용·발화자·일시·대화 맥락)을 작성해 승인을 요청한다. 사용자가 승인해 소스 폴더로 옮기면 그 확인서가 원본이 된다.
3. **검증·적재**: 근거 실물이 도착하면 주장 값과 대조한다 — 일치하면 적재하고 `검증대기.md`에서 졸업(해소일·근거 경로 기록), 불일치하면 봉합하지 않고 모순으로 보존한다(적재 규칙 8). 위키 근거는 항상 도착한 원본을 가리킨다(주장 발화가 아니라). 스킬팩이 있으면 brain-ingest 가 이 대조·적재를 담당한다.
4. **답변 시**: 검증대기 중인 주장은 정본 답변에 쓰지 않는다. 해당 대상을 물으면 "미검증 구두 주장 있음(등록일·요지)"으로만 함께 안내한다.
5. **스킬 실행 중 발언도 동일**: 빌드·문서생성 중 사용자가 문서에 없는 값·지시를 말해도 같은 절차를 따른다.

> ⚠️ **기록처는 오직 `검증대기.md`다.** 미검증 주장을 위키 페이지·규약·이 CLAUDE.md 에 "미확인" 딱지를 달아서라도 적지 않는다 — 정본에 들어간 미확인 값은 이후 인용되며 사실로 세탁된다. ⓒ 구두확인서의 승인도 **소스 폴더로 옮기는 것까지**다 — 승인됐다고 CLAUDE.md·규약에 규칙을 직접 추가하지 않는다(반영은 옮겨진 원본의 정상 적재 경로가 담당한다).

`검증대기.md`는 적재 후보의 대기열(데이터 축)이지 "누가 이해했나" 류의 상태가 아니다 — 온보딩 규칙의 "상태를 뇌에 남기지 않는다"(사람 이해 축)와 충돌하지 않는다.

## 신규 열람자 온보딩 (인수인계) 규칙

이 규칙은 뇌↔원본이 아니라 **사람↔뇌**의 이해 격차를 다룬다 — 뇌가 정확해도 이 뇌를 처음 받는 사람(신입·후임·담당자 교체)은 축적된 맥락을 모른다.

- **발동**: 이 폴더를 처음 여는 사람, 또는 "이 뇌 파악 도와줘"·"인수인계 브리핑 해줘"·"이해 확인해줘" 요청 시. 뇌를 이미 아는 사람에게 강요하지 않는다(옵트인).
- **브리핑(설명)**: `INDEX.md`와 주제 위키를 근거로 이 뇌의 지도를 짧게 안내한다 — 어떤 주제가 있고, 핵심 거래처·계약·규정이 무엇이며, **지금 사람의 판단이 필요한 지점**(`검수리포트.md`의 모순·`문제파일.md`·신선도 낡음)이 어디인지. 모든 사실 뒤에 근거 파일 경로를 붙인다(질의 응답 규칙과 동일 — 지어내지 않는다).
- **이해 확인(선택)**: 브리핑 후 열람자가 원하면, 근거로 채점 가능한 확인 질문 3~5개를 낸다. 단순 회상이 아니라 **판단 문항** 위주다(예: "계약서와 발주서의 단가가 다른데 어느 것을 신뢰해야 하나?" → `검수리포트.md` 근거로 확인). 채점 기준은 위키·검수리포트의 근거이지 추측이 아니다.
- **책임 경계 표면화**: 자동 정리가 결정할 수 없어 **사람이 확인·결정해야 하는 것**(모순·추정값·미적재·낡은 자료)을 "이건 당신 몫"으로 명시한다.
- **상태를 뇌에 남기지 않는다**: 이해 확인 결과·학습 상태를 뇌 폴더에 파일로 저장하지 않는다. 뇌는 *데이터*의 정본이지 "누가 이해했나"의 정본이 아니다 — 온보딩은 매 열람 세션의 행동일 뿐 상태를 축적하지 않는다.

## 규약 레이어 (문서 규약 역공학)

- `규약/폴더-지도.md`·`규약/명명-규칙.md`·`규약/양식/{유형}.md`에 원본 문서 규약을 역공학해 둔다. 신규 문서 생성 시 이 규약을 형식 기준으로 삼는다.
- 우선순위: **명문 규정 > 관찰된 다수 패턴 > 단일 사례 추정.** 단, 명문 규정과 실제 관찰이 불일치하면 봉합하지 말고 양쪽을 보존·명시하고 사용자 확인을 권고한다.
- 각 규약 항목에는 `근거`·`표본 수`·`상태(추정/확정)`를 붙인다.

## 유지보수

- 원본이 갱신되면 해당 주제 페이지의 `최종갱신:`을 바꾸고 근거 표기를 갱신한다. 머리말 `last_updated`도 갱신한다.
- 새 원본이 추가되면 해당 주제 페이지에 반영하고 `INDEX.md`의 카탈로그·커버리지·적재이력을 갱신한다.
- 신선도가 의심되면 `brain-audit`로 소스 폴더를 재스캔해 `검수리포트.md` 신선도 절을 갱신한다.
- 숨김 파일(`.brain-manifest.json` 등)은 검색 도구 기본값이 제외하므로 조회 시 경로를 직접 지정한다.
