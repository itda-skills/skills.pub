# itda-brain — 비정형 문서 → 근거 추적 업무DB vertical 스킬팩

회사 공유폴더의 **비정형 문서 무더기**(워드·엑셀·PPT·PDF·txt 수십~수백 개)를
근거 추적 가능한 **업무DB(뇌)**로 만들고, 그 뇌로 질답·검수·문서생성을 하는 스킬팩입니다.
NotebookLM 류 "읽어주는 뇌"와 달리 ① 로컬 오피스 문서가 1급 시민, ② 독립 검수가
**안 물어본 모순까지 선제 적발**(신뢰 레이어), ③ 규약을 배워 문서를 **쓰기까지** —
"읽어주는 뇌"가 아니라 **"일하는 뇌"**입니다. (SPEC-BRAIN-VERTICAL-001, #1122)

`itda-data`(정형 CSV/엑셀의 거짓확신 차단)와 나란히 서는 **비정형 문서**의 근거 추적 vertical입니다.

## 스킬 목록

| 스킬 | 역할 | 출력 |
|------|------|------|

## 사용 시나리오

> "이 공유폴더를 업무DB로 만들어줘" → `brain-build`가 원본을 전수 읽어 주제별 위키 + CLAUDE.md 스키마 생성, 끝에 자동 검수
> "대영인쇄 A4 용지 단가 얼마야?" → 생성된 `업무DB/`를 연 Cowork이 근거(계약 vs 발주)와 알려진 모순을 함께 제시 (질답은 `CLAUDE.md` 규율이 담당 — 스킬 아님)
> "이 뇌 아직 최신이야?" → `brain-audit`가 소스 폴더 재스캔 → 빌드 시점 수정시각과 대조해 신규/변경 파일 표면화
> "새 견적서 몇 개 들어왔어, 반영해줘" *(v1.1)* → `brain-ingest`가 증분 적재 + 적재이력 갱신
> "이번 달 견적서 규약대로 써줘" *(v1.5)* → `brain-scribe`가 `규약/양식/견적서` 서식으로 `초안/` 생성(승인 후 이동)
> "이 뇌 인수인계 브리핑 해줘" *(v0.3)* → 폴더를 연 Cowork이 `CLAUDE.md`의 **신규 열람자 온보딩 규칙**대로 뇌 지도·판단 필요 지점(모순·문제파일·신선도)을 근거와 함께 브리핑하고, 원하면 근거로 채점 가능한 이해 확인 문항을 낸다 (사람↔뇌 이해 격차 해소 — brain-build/audit이 보는 뇌↔원본 축과 직교)

## 3-Layer 아키텍처

```
Layer 1  Raw    소스폴더/ (원본 무더기)          ← 불가침(읽기 전용)
   ▼  brain-build (규약 역공학 동반)
Layer 2  Wiki   업무DB/*.md (주제별) + INDEX.md + 규약/ + 문제파일.md
   ▼  brain-auditor (격리 컨텍스트, 원본 재열람)
Layer 3  Schema 업무DB/CLAUDE.md (질답·운영 규율 + 뇌 메타 자기서술) → 검수리포트.md
```

**뇌의 정본은 폴더 그 자체**입니다 — 파일시스템만으로 완결(hyve·데몬·외부 DB 무의존). Cowork이 폴더를 열면 `CLAUDE.md` 규율이 자동 적용됩니다(스킬팩 없는 동료에게 폴더만 줘도 규율을 따름). 질답 규율은 셸/데몬이 아니라 **생성된 `CLAUDE.md`(Layer 3)**가 담당합니다.

**v1 은 단일 소스**입니다 — 뇌 하나 = 소스 폴더 하나(`CLAUDE.md` 머리말 `sources:` 절대경로 1개). 신선도 기준선(manifest)·근거 상대경로가 단일 루트 전제라, 폴더 여러 개는 뇌를 분리해 빌드합니다(brain-build 가 2개 이상 요청을 명시 거부). 다중 소스는 실수요 확인 시 manifest 포맷 확장으로 후속.

## 원본 불가침 (최우선 계약)

모든 스킬·에이전트는 Layer 1 원본에 **읽기 전용**으로만 접근합니다 — 절대 수정·이동·삭제·이름변경하지 않습니다. 업무DB는 원본을 읽어 만든 2차 정리본이며, 검수·정정 권고도 원본 수정을 포함하지 않습니다(정정은 원본 소유 부서 확인 후 사람이 수행).

## 환경 변수 / 의존성

- **환경 변수**: 없음. 외부 API 키·인증 불필요.
- **오피스 문서 판독**: 에이전트 경유(document-skills `docx`/`xlsx`/`pptx`/`pdf` 또는 로컬 파서). 스킬 Python이 문서를 직접 파싱하지 않습니다.
- **Python 의존성**: `brain-audit`의 신선도 스캔(`scripts/freshness.py`)은 **stdlib only**(Python 3.10+, `os.stat` mtime). 그 외 스킬은 오케스트레이션(SKILL.md 지시).
- **hyve 무의존**: Cowork에 스킬팩 설치만으로 전 기능 동작합니다. hyve 셸 연계(뇌 자동 발견 레지스트리·MCP 카탈로그·상주 신선도 감시)는 후속(SPEC-BRAIN-VERTICAL-001 EXC-1) — 지금은 스키마 필드만 선매설.

## 검수 4각도 + 제5각

`brain-auditor`가 빌드 기억과 격리된 컨텍스트에서 원본을 다시 열어 검수합니다:

1. **전수성** — 원본 전건 커버, 누락 0, 유령 경로 0.
2. **수치 재대조** — 무작위 추출 + 파생 재계산, 전사 오류 0.
3. **근거 추적** — 인라인 근거·`출처:` 머리말 누락 0, 유령 근거 0.
4. **교차 모순** — 원본 문서 상호 대조로 실질 모순을 심각도순으로 ("어느 파일의 어떤 값 vs …"). 거래 방향·기간이 다른 값은 모순으로 오인하지 않음(오탐 경계).

**제5각(신선도)** — `brain-audit`가 소스 폴더를 재스캔해 커버리지 표의 빌드 시점 수정시각과 대조, 뇌가 낡았는지 온디맨드로 감지.

## evals — 검수관 검출력 회귀 가드

`evals/`에 함정 6종(단가 모순·거래방향 미끼·파일명 함정·잠금 임시파일·빈 문서·시계열 미끼)을 심은
골든 fixture 와 assertions(A1~A10)가 있습니다. `brain-audit/tests/`(pytest)가 저울(freshness.py)을
지킨다면 evals 는 검수관(SKILL.md·brain-auditor 지시서)의 유능함을 지킵니다 — 지시서를 고치면
`evals/README.md` 절차로 재실행해 검출률·오탐률 유지를 확인합니다.
