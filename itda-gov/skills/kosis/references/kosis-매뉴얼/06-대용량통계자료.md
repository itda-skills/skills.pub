# 2.3 대용량 통계자료 (statisticsBigData.do)

출처: 통계청 발간 "KOSIS 공유서비스(OpenAPI) 개발가이드 v1.0" 2.3 (페이지 102~109)
원본 PDF: [openApi_manual_v1.0.pdf](openApi_manual_v1.0.pdf)
추출일: 2026-04-28

## 2.3.1 특징

- 통계표의 수치자료 및 메타정보 제공을 위한 OpenAPI로, **통계표 전체, 분류 전체(일부), 항목 전체(일부)**를 선택적으로 요청한다.
- 통계표의 수치자료를 다중계열, 여러시점으로 제공
- 통계표의 수록정보, 분류/항목, 출처, 단위 등 메타정보 제공
- **자료 제공형태**: SDMX (DSD, Generic, StructureSpecific), **CSV** (대용량 한정 추가; XLS는 출력 결과 형태)

## 2.3.2 서비스 활용

### 2.3.2.1 자료 등록

개발가이드 > 대용량 통계자료 > URL생성 > 자료등록

- '작성기관', '통계조사명', '통계표명' 등을 입력하여 사용하고자 하는 자료를 조회한 뒤 조회결과에서 등록할 자료의 사용여부 항목을 선택하고 통계표 등록 버튼을 누른다.
- 통계표조회 버튼을 클릭하면 해당 자료의 통계표를 볼 수 있다.

> 안내문: "대용량 통계자료 API를 이용하여 위한 호출변수, 출력항목, 샘플데이터를 제공합니다. 단, 출력형태로는 SDMX, XLS를 제공합니다. 데이터 출력건수가 40,000개 이상일 경우는 SDMX 출력은 불가하여 가능합니다. 보안, 시스템에서 출력건수가 4만건이 넘을 경우 ALL로도 사용시 검수단으로 대용건수을 검수해야 사용하셔야 합니다."

### 2.3.2.2 등록된 자료

마이페이지 > 등록한 자료 > 대용량통계자료

- 자료등록의 통계표 등록을 마치고 등록된 자료 탭을 클릭하면, 이용자가 등록한 자료들의 목록이 나타나며, 등록된 자료 중 URL생성을 원하는 자료의 URL 생성 버튼을 누른다.
- 통계표조회 버튼을 클릭하면 해당 자료의 통계표를 볼 수 있다.
- 삭제: 마이페이지 > 등록한 자료 > 대용량통계자료 > 삭제. 버튼을 클릭한 후 하단에 조회되는 사용자 생성 URL 목록을 삭제할 수 있다.

### 2.3.2.3 URL생성

개발가이드 > 통계자료 > 자료등록 URL생성 > 등록된 자료 > URL 생성

- URL생성 단계에서는 URL생성 조건 설정의 '활용 자료명', '분류/항목선택'을 입력 후 URL생성 버튼을 누르면 URL생성 상세조건 화면으로 이동 후 URL이 하단에 생성된다.
- URL생성 상세조건 화면에서 상세설정 후 URL보기, 결과값보기 버튼을 클릭하여 페이지 하단에서 결과를 확인할 수 있고, URL복사 버튼을 누르면 생성된 URL이 클립보드에 복사된다.

URL 생성 조건 설정 항목: 데이터 포멧 (JSON/SDMX), 조회기간 설정 (기간설정/최근시점기준), 최근시점부터, 수록시점간격, 필요자료명, URL 상세 정보 등.

## 2.3.3 활용방법

### 2.3.3.1 SDMX(DSD)

- **호출 URL**: `http://kosis.kr/openapi/statisticsBigData.do`

#### 입력 변수

| 항목명(영문) | 변수타입 | 항목설명 | 비고 |
|------------|---------|---------|------|
| apiKey | String | 발급된 인증키 | 필수 |
| userStatsId | String | 사용자 등록 통계표 | 필수 |
| type | String | SDMX의 유형(DSD, Generic, StructureSpecific) | 필수 |
| format | String | 결과 유형(json, sdmx) | 필수 |
| version | String | 결과값 구분 | 생략시 구버전으로 데이터 출력 |

#### 출력 변수 (대용량 SDMX DSD — 통계자료 DSD와 동일 구조)

```
- Header
    - Id (기관코드_통계표ID)
    - Name (통계표명)
    - Prepared (전송시간)
    - Sender (Id, Name, Contact{Department, Telephone})
    - Source (출처)
- Codelist > Codelist
    - Id (코드리스트ID)
    - Name (코드리스트명)
    - Description (코드리스트영문명)
    - Code (Id 코드ID, Name 코드명)
- Concepts > ConceptsScheme
    - Id (컨셉스키마ID)
    - Name (컨셉스키마명)
    - Description (컨셉스키마영문명)
    - concept (Id 컨셉ID, Name 컨셉명, Description 컨셉영문명)
- DataStructures > DataStructure
    - Id (통계표ID)
    - Name (통계표명)
    - DataStructureComponents
        - Dimension (Id 디멘젼Id, ConceptIdentity > Id)
```

### 2.3.3.2 SDMX(Generic)

- **호출 URL**: `http://kosis.kr/openapi/statisticsBigData.do`

#### 입력 변수

| 항목명(영문) | 변수타입 | 항목설명 | 비고 |
|------------|---------|---------|------|
| apiKey | String | 발급된 인증키 | 필수 |
| userStatsId | String | 사용자 등록 통계표 | 필수 |
| type | String | SDMX의 유형(DSD, Generic, StructureSpecific) | 필수 |
| prdSe | String | 수록주기 | 필수 |
| startPrdDe / endPrdDe | String | 시점기준 (시작/종료수록시점) | 선택 (시점기준 또는 최신자료기준 택1) |
| newEstPrdCnt / prdInterval | String | 최신자료기준 (최근수록시점 개수, 수록시점 간격) | 선택 |
| format | String | 결과 유형(json, sdmx) | 필수 |
| version | String | 결과값 구분 | 생략시 구버전으로 데이터 출력 |

#### 출력 변수 (대용량 SDMX Generic — Series 단순화 트리)

```
- Header (Id, Name, Prepared, Sender{Id, Name, Contact{Department, Telephone}}, Source)
- Series
    - SeriesKey > Value (Id 시리즈키ID, value 시리즈키값)
    - Obs (ObsDimension.Value 시점, ObsValue.Value 수치자료값)
```

> 통계자료(2.2.3.3) Generic과 차이: 대용량은 SeriesKey에 UNIT 노드가 명시되지 않음.

### 2.3.3.3 SDMX(StructureSpecific)

- **호출 URL**: `http://kosis.kr/openapi/statisticsBigData.do`

#### 입력 변수

| 항목명(영문) | 변수타입 | 항목설명 | 비고 |
|------------|---------|---------|------|
| apiKey | String | 발급된 인증키 | 필수 |
| userStatsId | String | 사용자 등록 통계표 | 필수 |
| type | String | SDMX의 유형(DSD, Generic, StructureSpecific) | 필수 |
| prdSe | String | 수록주기 | 필수 |
| startPrdDe / endPrdDe | String | 시점기준 | 선택 (택1) |
| newEstPrdCnt / prdInterval | String | 최신자료기준 | 선택 |
| format | String | 결과 유형(json, sdmx) | 필수 |
| version | String | 결과값 구분 | 생략시 구버전 |

#### 출력 변수 (대용량 SDMX StructureSpecific)

```
- Header (Id, Name, Prepared, Sender{Id, Name, Contact{Department, Telephone}}, Source)
- Series
    - UNIT (단위)
    - ITEM (항목)
    - FREQ (주기)
    - 분류
    - Obs (ObsDimension.Value 시점, ObsValue.Value 수치자료값)
```

> 통계자료(2.2.3.4)에서는 `FIEQ`로 표기되었으나 대용량 매뉴얼은 `FREQ`로 표기. 매뉴얼 자체 오기 또는 표기 차이로 보임. **표준 SDMX 명칭은 FREQ**이므로 실제 응답은 FREQ가 정확.

### 2.3.3.4 XLS

- **호출 URL**: `http://kosis.kr/openapi/statisticsBigData.do`

#### 입력 변수

| 항목명(영문) | 변수타입 | 항목설명 | 비고 |
|------------|---------|---------|------|
| apiKey | String | 발급된 인증키 | 필수 |
| userStatsId | String | 사용자 등록 통계표 | 필수 |
| prdSe | String | 수록주기 | 필수 |
| startPrdDe / endPrdDe | String | 시점기준 | 선택 (택1) |
| newEstPrdCnt / prdInterval | String | 최신자료기준 | 선택 |
| format | String | 결과 유형(json, sdmx) | 필수 |

#### 출력 결과 예시

엑셀 워크시트 형식:

| 행정구역별 | 연령별 | 항목 | 2038 | 2039 | 2040 |
|----------|--------|------|------|------|------|
| 전국 | 계 | 추계인구(명) | 51,470,244 | 51,291,401 | 51,091,352 |
| 전국 | 계 | 추계인구(남)(명) | 25,474,360 | 25,375,247 | 25,264,681 |
| 전국 | 계 | 추계인구(여)(명) | 25,995,884 | 25,916,154 | 25,826,671 |
| 서울특별시 | 계 | 추계인구(명) | 10,003,809 | 9,965,542 | 9,924,373 |
| ... | ... | ... | ... | ... | ... |

대용량 XLS 출력은 통계표 단위로 행이 펼쳐진 표 형식이며, 4만 셀 이상의 데이터는 SDMX가 불가하고 XLS만 가능.

## 핵심 차이 요약 (2.2 통계자료 vs 2.3 대용량)

| 항목 | 통계자료 (2.2) | 대용량 (2.3) |
|------|--------------|------------|
| 호출 URL | statisticsData.do | statisticsBigData.do |
| 호출 방식 | URL 선택 + 자료등록 | 자료등록 전용 |
| 출력 형태 | JSON, SDMX(3종), XML | SDMX(3종), CSV/XLS |
| 데이터 한도 | 4만 셀 이내 | 4만+ 셀 가능 (XLS) |
| 통계표선택 직접 가능? | ○ (orgId+tblId+objL+itmId) | ✕ (userStatsId 필수) |
