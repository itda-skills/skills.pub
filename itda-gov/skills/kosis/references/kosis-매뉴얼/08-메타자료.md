# 2.5 메타자료 (statisticsData.do?method=getMeta)

출처: 통계청 발간 "KOSIS 공유서비스(OpenAPI) 개발가이드 v1.0" 2.5 (페이지 137~146)
원본 PDF: [openApi_manual_v1.0.pdf](openApi_manual_v1.0.pdf)
추출일: 2026-04-28

## 2.5.1 특징

- 통계자료에 대한 메타자료 제공을 위한 OpenAPI
- 자료 제공형태: XML, JSON
- **호출 URL 공통**: `http://kosis.kr/openapi/statisticsData.do?method=getMeta&type={TYPE}`

## 2.5.2 type별 메타 조회 (9종, JSON·XML 동일)

### type=TBL (통계표 명칭) — 2.5.2.1 / 2.5.2.10

| 입력 항목 | 변수타입 | 설명 | 비고 |
|---------|---------|------|------|
| apiKey | String | 인증키 | 필수 |
| orgId | String | 기관 ID | 필수 |
| tblId | String | 통계표 ID | 필수 |
| format | String | json/sdmx | 필수 |
| content | String | html/json (헤더) | 선택 |

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| TBL_NM (또는 tblNm) | 통계표 국문명 | VARCHAR2(300) |
| TBL_NM_ENG (또는 tblNmEng) | 통계표 영문명 | VARCHAR2(300) |

> JSON은 대문자(TBL_NM), XML은 lowerCamelCase(tblNm) 사용.

### type=ORG (기관 명칭) — 2.5.2.2 / 2.5.2.11

입력: apiKey, orgId, format (+ XML은 위 3개로 끝, JSON은 content 추가).

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| ORG_NM (또는 orgNm) | 기관 국문명 | VARCHAR2(300) |
| TORG_NM_ENG (또는 orgNmEng) | 기관 영문명 | VARCHAR2(300) |

> JSON 출력 키 `TORG_NM_ENG`는 매뉴얼 표기. 코드 작성시 응답 확인 권장 (Typo 가능성).

### type=PRD (수록정보) — 2.5.2.3 / 2.5.2.12

| 입력 항목 | 설명 | 비고 |
|----------|------|------|
| apiKey, orgId, tblId, format | 동일 | 필수 |
| detail | 전체시점 정보 제공 여부 | 선택 |
| content | html/json (헤더) | 선택 (JSON만) |

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| PRD_SE / prdSe | 수록주기 | VARCHAR2(300) |
| PRD_DE / PrdDe | 수록시점 | VARCHAR2(8) |

### type=ITM (분류/항목) — 2.5.2.4 / 2.5.2.13

| 입력 항목 | 설명 | 비고 |
|----------|------|------|
| apiKey, orgId, tblId | | 필수 |
| objId | 분류코드 | 선택 |
| itmId | 자료코드 | 선택 |
| format | json/xml | 필수 |
| content | (JSON 한정) | 선택 |

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| OBJ_ID / objId | 분류 ID | VARCHAR2(40) |
| OBJ_NM / objNm | 분류 국문명 | VARCHAR2(3000) |
| OBJ_NM_ENG / objNmEng | 분류 영문명 | VARCHAR2(3000) |
| ITM_ID / itmId | 자료코드 ID | VARCHAR2(40) |
| ITM_NM / itmNm | 자료코드 국문명 | VARCHAR2(3000) |
| ITM_NM_ENG / itmNmEng | 자료코드 영문명 | VARCHAR2(3000) |

| UNIT_ID / unitId | 단위ID | VARCHAR2(40) |
| UNIT_NM / unitNm | 단위 국문명 | VARCHAR2(1000) |
| UNIT_ENG_NM / unitEngNm | 단위 영문명 | VARCHAR2(1000) |

### type=CMMT (주석) — 2.5.2.5 / 2.5.2.14

입력: apiKey, orgId, tblId, format (+JSON content 추가).

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| CMMT_NM | 주석유형 | VARCHAR2(40) |

| OBJ_NM | 분류 명 | VARCHAR2(3000) |

| ITM_NM | 자료코드 명 | VARCHAR2(3000) |

### type=UNIT (단위) — 2.5.2.6 / 2.5.2.15

JSON 입력: apiKey, orgId, tblId, format, content.
XML 입력: apiKey, **unitId** (단위코드), format. (서로 다름!)

| 출력 항목 | 설명 | 형식 |
|----------|------|------|
| UNIT_NM / unitNm | 단위 국문명 | VARCHAR2(1000) |
| UNIT_NM_ENG / unitNmEng | 단위 영문명 | VARCHAR2(1000) |

### type=SOURCE (출처) — 2.5.2.7 / 2.5.2.16

| 출력 항목 | 설명 | 형식 |

| 입력 항목 | 설명 | 비고 |
|----------|------|------|

## 호출 패턴 요약

```
http://kosis.kr/openapi/statisticsData.do?
  method=getMeta&
  type=TBL|ORG|PRD|ITM|CMMT|UNIT|SOURCE|WGT|NCD&
  apiKey=<KEY>&
  orgId=101&
  tblId=DT_1B01003&
  format=json   (or xml)
```

dart_api.py 패턴 참고 (페이지 65~67): 통계자료 본 호출과 별개로 통계표 명칭/항목/단위/분류 메타 조회를 같은 endpoint(`statisticsData.do`)에서 method 만 `getMeta`로 바꿔 사용.

## 코드 사용 관점

`kosis_api.py`에는 메타 조회 endpoint 구현이 없음. 향후 통계표 정보 보강 시 이 API 활용 가능.
