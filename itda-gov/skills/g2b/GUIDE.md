## 빠른 시작

조달청 나라장터(G2B) 정부 입찰공고를 검색·조회하는 가장 간단한 방법입니다.

```
나라장터에서 소프트웨어 개발 입찰 공고 찾아줘
3월 한달간 AI 관련 입찰 공고 목록 만들어줘
오늘 기준 1주일간 나라장터 입찰공고 표로 정리해줘
데이터 관련 모든 입찰공고를 상세정보 포함해서 뽑아줘
```

Claude가 자연어 요청을 `--keyword`, `--from`, `--to`, `--format`, `--detail` 플래그로 변환해 CLI를 호출하고, 공고별 요약과 나라장터 원본 링크를 함께 돌려줍니다.

## 사전 준비

공공데이터포털(https://www.data.go.kr) 회원가입 후 `나라장터 입찰공고정보서비스` 활용신청이 필요합니다. 승인(수시간~수일)이 완료되면 마이페이지에서 인증키가 발급됩니다.

```bash
# Claude Cowork 환경변수 등록 (권장)
claude config set env.KO_DATA_API_KEY "발급받은_키"

# 또는 .env 파일
KO_DATA_API_KEY=발급받은_키
```

Python 3.10 이상이 필요하며 추가 패키지 설치는 필요 없습니다. macOS/Linux는 `python3`, Windows는 `py -3`로 실행합니다.

## 활용 시나리오

### 입찰 제안서 작성 전 시장 조사

관심 분야 키워드로 최근 공고를 모아 제안등록마감일·예정가격·발주기관을 파악하고 제안서 우선순위를 결정합니다.

```
AI 관련 입찰공고 중에서 제안마감이 아직 안 지난 것만 상세정보로 뽑아줘
```

### 주기적 입찰 모니터링

매일·매주 기본 키워드로 신규 공고를 확인해 경쟁 기회를 놓치지 않습니다. cron이나 예약 실행과 결합하면 자동화할 수 있습니다.

```
지난 7일간 데이터 관련 신규 입찰공고를 표로 보여줘
```

### 경쟁사 낙찰 이력 분석

특정 분야·기관의 과거 입찰 공고를 조회해 예상 규모와 발주 패턴을 분석합니다.

```
3월 한달간 교육청 발주 소프트웨어 공고를 모두 정리해줘
```

## 출력 옵션

| 옵션 | 설명 | 사용 시점 |
|------|------|-----------|
| `--format json` | JSON 구조화 데이터 (기본값) | Claude가 요약·비교·변환을 이어갈 때 |
| `--format table` | 터미널 테이블 | 사람이 터미널에서 즉시 훑어볼 때 |
| `--detail` | 발주기관·예정가격·지역제한 등 상세 필드 포함 | 실제 입찰 가능성·편사 판단이 필요할 때 |

## 팁

- **상세 조회는 `--detail`로**: 기본 조회는 공고명·공고번호 중심이라 입찰 가능성을 판단하기 어렵습니다. `--detail`을 붙이면 발주기관, 예정가격, 지역제한 같은 핵심 판단 필드가 함께 나옵니다.
- **터미널 확인은 table, 후속 가공은 JSON**: 사람이 직접 훑을 때는 `--format table`, Claude에게 비교·요약·CSV 변환을 맡길 때는 기본 JSON이 훨씬 다루기 쉽습니다.
- **복합어 검색은 따옴표로**: `--keyword "소프트웨어 개발"`처럼 공백을 따옴표로 묶으면 두 단어가 모두 포함된 공고만 필터링되어 노이즈가 줄어듭니다.
- **자동화는 cron과 결합**: `py -3 scripts/collect_g2b.py --keyword "AI" --from 2026-04-13 --to 2026-04-20` 형태를 예약 실행으로 돌려 일·주 단위 신규 공고만 검토할 수 있습니다.

## 제한사항

- **일별 API 호출 횟수 제한**: 공공데이터포털은 활용신청 시 일별 트래픽 한도가 지정됩니다. 대량 조회를 실행하기 전 마이페이지에서 남은 호출 한도를 확인하세요.
- **첨부파일 다운로드 미지원**: API는 공고 메타데이터(제목·기간·발주기관 등)만 제공합니다. 공고 본문·RFP 첨부파일은 나라장터 웹사이트에서 원본 링크로 직접 확인해야 합니다.
- **공고명 키워드 매칭만 지원**: `--keyword`는 공고명(사업명)에만 적용됩니다. 공고 본문 전문 검색이나 발주기관 이름으로 역검색하는 기능은 제공하지 않습니다.
- **장기간 한 번에 조회 지양**: 1년 이상 긴 기간을 한 번에 조회하면 응답이 잘리거나 타임아웃될 수 있습니다. 3개월 단위로 나눠 조회하세요.
