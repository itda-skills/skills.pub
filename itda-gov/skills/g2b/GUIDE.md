# g2b 사용 가이드

조달청 나라장터(G2B)에서 정부 입찰공고를 검색·조회합니다.
**Claude에게 자연어로 부탁하면 됩니다** — "소프트웨어 개발 입찰 공고 찾아줘", "3월 AI 관련 입찰 목록 만들어줘"처럼요. 터미널 명령을 직접 입력할 필요는 없습니다.

---

## 사전 준비 — API 키 발급

g2b 스킬은 공공데이터포털 API 키가 필요합니다.

1. [조달청 나라장터 공공데이터개방표준서비스 활용신청 페이지](https://www.data.go.kr/data/15058815/openapi.do)에서 회원가입 후 활용신청 (**자동승인**)
2. 승인 후 마이페이지 → **개발계정 → Decoding** 키(일반 인증키) 복사
3. 발급받은 키를 **`.env` 파일에 넣습니다 (권장)** — Cowork에 연결한 작업 폴더(연결한 폴더가 여러 개면 아무 폴더나) 루트에 `.env` 파일을 만들고 아래 한 줄을 넣어 두면 스킬이 자동으로 찾아 읽습니다. 점(`.`)으로 시작하는 파일을 만들기 어렵다면 **`환경변수.txt`** 라는 이름으로 만들어도 똑같이 읽힙니다(메모장이 `.txt` 를 붙여 `.env.txt` 가 되어도 됩니다).

```dotenv
KO_DATA_API_KEY=발급받은_Decoding_키
```

> 자세한 가입·키 발급 절차(Decoding 키 주의사항 포함)는 [공공데이터포털 발급 가이드](https://skills.itda.work/credentials/data-go-kr/)를 참고하세요.

Claude Desktop의 "Claude 지침"(설정 → 일반)에 같은 내용을 적는 방식도 동작하지만, 대화 컨텍스트에 값이 노출되므로 `.env` 파일을 권장합니다.

> 개발자라면 셸 환경변수로 넣어도 됩니다.

> 자동승인이지만 게이트웨이 동기화에 5~30분(드물게 1시간) 걸립니다. 신청 직후 "인증 실패"가 뜨면 잠시 후 다시 요청해보세요.

> 같은 키를 `funding`·`realestate` 스킬에서도 함께 사용할 수 있습니다. 단, **각 API는 별도로 활용신청**해야 합니다. 미신청 서비스를 조회하면 인증 실패 응답이 반환됩니다.

## 자주 쓰는 요청

| 하고 싶은 것 | 이렇게 말하세요 |
|--------------|-----------------|
| 최근 입찰공고 | "지난 7일간 데이터 관련 신규 입찰공고를 표로 보여줘" |
| 키워드 검색 | "나라장터에서 소프트웨어 개발 입찰 공고 찾아줘" |
| 기간 지정 | "3월 한달간 AI 관련 입찰 공고 목록 만들어줘" |
| 상세 정보 포함 | "데이터 관련 모든 입찰공고를 상세정보 포함해서 뽑아줘" |
| 시장 조사 | "AI 관련 입찰공고 중에서 제안마감이 아직 안 지난 것만 상세정보로 뽑아줘" |

## 활용 시나리오

### 입찰 제안서 작성 전 시장 조사

관심 분야 키워드로 최근 공고를 모아 제안등록마감일·예정가격·발주기관을 파악하고 제안서 우선순위를 결정합니다.

```
AI 관련 입찰공고 중에서 제안마감이 아직 안 지난 것만 상세정보로 뽑아줘
```

### 주기적 입찰 모니터링

매일·매주 기본 키워드로 신규 공고를 확인해 경쟁 기회를 놓치지 않습니다.

```
지난 7일간 데이터 관련 신규 입찰공고를 표로 보여줘
```

### 경쟁사 낙찰 이력 분석

특정 분야·기관의 과거 입찰 공고를 조회해 예상 규모와 발주 패턴을 분석합니다.

```
3월 한달간 교육청 발주 소프트웨어 공고를 모두 정리해줘
```

---

## 상세 정보 요청

기본 조회는 공고명·공고번호 중심입니다. 발주기관·예정가격·지역제한 같은 핵심 판단 정보가 필요하면 "상세정보 포함해서"라고 말해주세요.

```
소프트웨어 입찰공고를 상세정보 포함해서 보여줘
```

---

## 팁

- **"상세정보 포함해서"를 붙이면 판단에 필요한 정보가 더 나옵니다.** 기본 조회는 공고명·공고번호 중심이라 입찰 가능성을 판단하기 어렵습니다. 발주기관, 예정가격, 지역제한 같은 정보가 필요하면 "상세정보 포함해서 보여줘"라고 말하세요.
- **기간을 좁게 말하세요.** 1년 이상의 긴 기간을 한 번에 조회하면 응답이 잘리거나 느려질 수 있습니다. 3개월 단위로 나눠 요청하는 것을 권장합니다.
- **공고명 키워드로만 검색됩니다.** "소프트웨어 개발"처럼 공고명(사업명)에 포함된 단어로 검색합니다. 발주기관 이름으로 역검색하거나 공고 본문 전문을 검색하는 기능은 지원하지 않습니다.
- **매주 같은 요청을 반복해 신규 공고를 모니터링할 수 있습니다.** "AI 키워드로 지난주 신규 입찰 공고 정리해줘"를 매주 월요일 아침에 반복하면 새 공고만 검토할 수 있습니다.

## 안 될 때

| 증상 | 원인 / 해결 |
|------|-------------|
| "API 키가 없다"는 안내 | **`.env` 파일**(또는 Claude 지침)에 `KO_DATA_API_KEY`가 없거나 키가 비어 있음. 위 "사전 준비" 확인 |
| "인증 실패" 또는 HTTP 403 | 활용신청 직후 게이트웨이 동기화 중. 5~30분 후 다시 요청. 또는 나라장터 API를 별도 활용신청하지 않은 경우 |
| "공고를 찾을 수 없습니다" | 해당 조건 공고 없음. 기간을 넓히거나 키워드를 바꿔 재요청 |
| 응답이 너무 느리거나 잘림 | 조회 기간이 너무 길 수 있음. 3개월 단위로 나눠 요청 |

---

## 주의사항

- ⚠️ **첨부파일 다운로드는 지원하지 않습니다.** API는 공고 메타데이터(제목·기간·발주기관 등)만 제공합니다. 공고 본문·RFP 첨부파일은 나라장터 웹사이트에서 원본 링크로 직접 확인해야 합니다.
- ⚠️ **일별 API 호출 횟수 한도가 있습니다.** 대량 조회 전 공공데이터포털 마이페이지에서 남은 호출 한도를 확인하세요.
