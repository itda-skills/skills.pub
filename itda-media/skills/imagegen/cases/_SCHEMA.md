# 케이스 카드 스키마 + 라이프사이클

`cases/*.md` 한 파일 = 한 케이스. 프롬프트 템플릿은 **실측 없이는 추측일 뿐**이다 —
모든 카드는 실측 기록을 쌓아야만 verified 가 된다. (#329)

## 라이프사이클

```
draft ──(실측 라운드 ≥1 + 마스터 최종 판정 통과)──▶ verified
  ▲                                                    │
  └──────(모델/CLI 메이저 변경으로 재검증 필요)─────────┘
```

- **draft** — 템플릿 작성됨, 실측 0회 또는 판정 대기. 사용 시 "실측 중" 단서를 단다.
- **verified** — 실측 라운드 1회 이상 + **마스터 최종 판정 통과**. SKILL.md 라우팅 표에서 기본 추천.
- 판정 주체: Claude 가 생성 이미지를 Read 로 보고 체크리스트 1차 평가 → 마스터가 최종 판정.
  Claude 단독으로 verified 승격 금지.
- codex CLI 메이저 업이나 기반 이미지 모델 변경 시 verified 카드도 재실측 대상으로 강등 검토.

## 카드 필수 섹션

```markdown
# <케이스명>

상태: draft | verified
최종 실측: YYYY-MM-DD (codex-cli X.Y.Z)

## 용도
어떤 요청이 이 케이스로 라우팅되는지 1~3줄.

## 프롬프트 템플릿
image.generate(prompt)에 전달할 프롬프트 본문. {주제} {스타일변형} 같은 슬롯 표기.
공통 품질 하한(SKILL.md) + 케이스 고유 품질 문구 포함.

## 품질 체크리스트
생성 결과를 Read 로 보고 판정하는 항목. 공통 항목 + 케이스 고유 항목.

## 실측 기록
라운드별 역순 누적. 각 라운드:
- 날짜 / codex 버전 / 사용 프롬프트(실제 전문)
- 결과: 파일·해상도·소요시간 / 체크리스트 판정 (항목별 ✅/❌)
- 썸네일: measurements/<케이스>-rN.jpg (512px 축소본)
- 교훈: 무엇이 품질을 올렸/내렸는가

## 실패 변형 (안티프롬프트)
"이 문구를 빼면/넣으면 이렇게 망가진다" 누적. 부정 사례가 실측 가치의 절반.
```

## 공통 품질 체크리스트 (모든 카드가 상속)

| # | 항목 | 판정 기준 |
|---|---|---|
| C1 | 스타일 일치 | 템플릿이 지정한 스타일로 생성됐는가 |
| C2 | 텍스트 아티팩트 없음 | 깨진 글자·워터마크·서명이 없는가 |
| C3 | 구조 왜곡 없음 | 비정상 손가락·왜곡 사물·짜부라진 형태가 없는가 |
| C4 | 구도 반영 | 요청한 배치·여백이 반영됐는가 |
| C5 | 비율/해상도 | 요청 비율(가로/세로/정사각)이 나왔는가 |
| C6 | 조명·깊이감 | 평면적 클립아트·무드 없는 결과가 아닌가 — 조명/분위기 층이 체감되는가 (r1 마스터 피드백 반영, `_PATTERNS.md` 5층 공식) |

## 실측 산출물 보관

- 원본 PNG 는 커밋하지 않는다(용량). `measurements/` 에는 **512px JPEG 축소본**만 커밋.
  변환: `sips -Z 512 -s format jpeg <원본>.png --out measurements/<케이스>-rN.jpg`
- 라운드 번호 rN 은 카드 내 실측 기록과 1:1.

## 신규 카드 추가 기준

같은 유형 요청을 2회 이상 다루면 draft 카드 신설을 제안한다.
1회성 특수 요청은 가장 가까운 카드 변형으로 처리하고 그 카드의 실측 기록에만 남긴다.
