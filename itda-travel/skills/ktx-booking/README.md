# ktx-booking — 동작 원리와 주의

> ⚠️ **실험적 스킬입니다. 배포하지 마세요.**
> 코레일 비공식 API에 의존하며 이용약관(ToS) 위반 소지가 있습니다. 개인 학습·실험
> 목적으로만 사용하고, 공개 marketplace에 배포하지 않습니다. 자세한 사용법은
> [GUIDE.md](GUIDE.md), 실행 계약은 [SKILL.md](SKILL.md) 참조.

## 무엇을 하는가

KTX 열차 **검색·예약**까지만 합니다. 결제·취소는 하지 않습니다(사용자가 코레일
앱/웹에서 직접). 자동 연타·취소표 매크로도 제공하지 않습니다.

## 어떻게 동작하는가 — DynaPath 안티봇 우회

코레일 모바일 앱(`com.korail.talk`)은 서버 요청마다 `x-dynapath-m-token` 헤더를
실어 보내고, 코레일의 **Dynapath** 안티봇이 *"이 요청이 변조되지 않은 진짜 공식
앱에서 왔는가"* 를 검증합니다. 자동화 클라이언트(봇)를 막기 위한 장치입니다.

`korail2-ncard` 라이브러리의 `DynaPathMasterEngine` 은 이 토큰을 **위조**해 공식
앱인 것처럼 위장합니다:

1. **앱·기기 신원 위장** — 평문 파라미터에 앱 ID, 기기 모델(`SM-S928N`, 갤럭시
   S24 Ultra), Android 13을 박고, 결정적으로 **무결성 플래그를 전부 "정상"으로
   선언**합니다: `루팅 아님·디버깅 아님·에뮬레이터 아님·후킹 아님`. "나는 변조되지
   않은 진짜 앱"이라는 거짓 선언입니다.
2. **동적 키** — `v1 + 4자리 난수 + 밀리초 타임스탬프` 로 매 요청 키를 생성합니다.
3. **독자 난독화 인코딩** — 표준 암호가 아니라, 앱 내부에 박힌 고유 변환 알고리즘을
   **리버스 엔지니어링**해 파이썬으로 포팅한 것입니다(문자→바이트 변환, 진법 기반
   인코딩, 키 파생 치환 테이블).
4. 로그인 시 별도로 **AES-CBC** 로 세션 ID(SID)도 생성합니다.

즉 **코레일이 봇을 막으려고 만든 무결성 검증 알고리즘을 통째로 베껴 무력화**하는
것입니다. "예의 바른 봇"이라도 봇은 봇입니다.

## 왜 이것이 문제인가

- **ToS 위반 소지** — 코레일 이용약관은 자동화 수단을 통한 접속·예약을 금지하고,
  위반 시 회원 자격 정지·이용 제한을 둡니다. 안티봇 우회는 그 금지를 정면으로
  거스릅니다.
- **지속 불가능성** — `SM-S928N`·앱 버전 등이 하드코딩이라, 코레일이 값만 바꿔도
  동작이 멈춥니다(라이브러리 업데이트에 영구히 의존).
- **부분 기능 / cancel 버그** — `cancel` 은 `DYNAPATH_PATHS`(토큰 적용 경로 목록)에서
  누락돼 토큰 없이 전송됩니다 → 코레일이 봇으로 차단 → JSON 아닌 응답 → 파싱 실패.
  즉 **예약은 되지만 취소는 라이브러리 차원에서 깨져 있습니다**. 이것이 우리 스킬이
  `cancel` 을 비목표로 둔 (결과적으로 타당했던) 이유입니다.

## 인접역 보기

코레일은 "서대전→용산" 같은 검색에서 **인접역(대전→서울) 출발 열차도 함께
반환**합니다(배차가 적은 역의 대안 안내). 따라서 결과의 `dep_name` 이 검색한 역과
다를 수 있으니, 실제 승차역을 확인하세요(예: `dep_code` 0025=서대전 vs 0010=대전).

## 배포 금지

이 스킬은 **개인 학습·실험 목적**입니다. 공개 배포는 ToS 위반 도구를 불특정 다수에
확산시키므로 하지 않습니다. PRIVATE 저장소에서만 관리하며, 사용에 따른 책임(계정
제재 등)은 사용자 본인에게 있습니다.
