## 빠른 시작

여행지나 동네에서 지금 뜨는 맛집·음식 트렌드를 검색량 surge로 찾습니다. 이렇게 말해보세요.

```
제주 요즘 뜨는 맛집 알려줘
강릉 지금 핫한 디저트 뭐야
성수에서 트렌디한 국밥집 찾아줘
```

지역만 말하면 핫키워드 탐지(모드 B), 동네+주제를 말하면 검색 보강(모드 A)으로 자동 분기합니다. 네이버 데이터랩·검색·검색광고 API 키가 필요합니다.

## 활용 시나리오

### 여행지 핫키워드 탐지 (모드 B)

지역/테마 하나로 지금 뜨는 음식·디저트·테마 키워드를 3레인으로 받습니다.

```
제주 요즘 뜨는 맛집
부산 트렌디한 카페
```

🆕 신규출현 / 🔥 검증상승 / 📺 미디어스파이크로 나뉘고, 각 키워드의 검색량 추이(근거)와 실제 가게가 매핑됩니다.

### 동네×주제 트렌디 판별 (모드 A)

특정 동네에서 한 음식이 지금 떠오르는 맥락인지 가게목록과 함께 봅니다.

```
성수에서 트렌디한 국밥집
연남동 지금 뜨는 파스타
```

평범한 노포에 억지로 "뜬다"를 붙이지 않습니다 — 이름이 바이럴된 소수만 surge를 표기합니다.

### LLM 음식 판정 흐름 (정확도 핵심)

"LLM 음식 판정 필요" 목록(예: 동문시장=먹거리, 섭지코지=관광)이 나오면 Claude가 음식/관광을 가려낸 뒤 다시 분석합니다.

```
제주 뜨는 맛집 — needs_llm 후보까지 판정해서 다시 분석해줘
```

## 출력 옵션

| 옵션 | 설명 | 사용 시점 |
|------|------|-----------|
| `--mode B --seed <지역>` | 핫키워드 탐지 (주 파이프라인) | "요즘 뭐가 떠?" 지역/테마 |
| `--mode A --region <동네> --topic <주제>` | 검색 보강 | "이 동네 이 음식" |
| `--emit-candidates` → `--judge-file` | LLM 판정 2단계 (후보 추출 → 판정 반영) | 가게명·신조어 정확도 |
| `--json` | 구조화 출력 | 후속 처리·디버깅 |

## 팁

- **넓게→좁게**: 먼저 `제주`처럼 넓게 보고, 관심 키워드가 보이면 `제주 디저트`로 좁혀 다시 물어보세요.
- **속도 vs 레벨 구분**: surge·상대YoY(속도)가 유행 지표입니다. 월검색량(레벨)이 높아도 안 뜨면 이미 유명한 클래식이지 유행이 아닙니다.
- **미디어스파이크는 식는중 확인**: 📺(성시경·흑백요리사 발) 키워드에 "막주 모멘텀 붕괴" 경고가 있으면 지금은 줄이 빠졌을 수 있습니다.
- ⚠️ **API 키 필요**: NAVER OpenAPI·검색광고 키가 셸 환경변수·`~/.claude/settings.json`·cwd/`$HOME`의 `.env` 중 하나에 있어야 합니다. 없으면 해당 소스는 `🚨 소스 상태`로 실패를 표시합니다(크래시 아님, fail-loud). 자동완성은 무인증.
- ⚠️ **결과가 비었는데 소스 경고가 있으면** 데이터 일시 오류이니 잠시 후 재시도하세요(없는 게 아니라 못 가져온 것).
- ⚠️ **비지원**: 예약·결제·평점·메뉴가격(트렌드 탐지 전용) / 실시간 분 단위(데이터랩 일·주 lag) / 갓 터진 신상(검색데이터 축적 전이라 늦게 잡힘) / 인스타 직접 스크래핑(영구 비목표, 검색량이 그 그림자) / 전국 디저트 트렌드(로드맵).
