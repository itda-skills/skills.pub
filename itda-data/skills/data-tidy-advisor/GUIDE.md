## 빠른 시작

지저분한 엑셀이나 CSV 파일을 주면 문제를 진단하고 정돈된 새 파일로 만들어 줍니다.

```
이 엑셀 파일 좀 정리해줘
```

```
헤더가 2줄인데 깔끔하게 정돈해줘
```

```
소계 행 빼고 깔끔하게 정돈해줘
```

```
날짜 형식이 제각각인데 통일해줘
```

원본 파일은 절대 건드리지 않습니다. 정돈본은 `tidy_[원본이름]_[날짜].csv` 형식으로 새 파일에 저장됩니다.

## 이런 파일에 씁니다

- **병합 셀이 섞인 엑셀** — 셀이 합쳐져 있어 그대로는 분석이 안 되는 경우
- **헤더가 여러 줄** — 첫 2~3행이 제목·설명이라 데이터 시작점이 애매한 경우
- **중간에 소계·합계 행** — 데이터 중간중간 합계 행이 끼어 있어 그대로 못 쓰는 경우
- **날짜 형식 제각각** — 같은 열에 `2024-01-01`과 `01/01/24`가 섞여 있는 경우
- **불필요한 공백·특수문자** — 셀에 공백이나 이상한 문자가 묻어 있는 경우
- **같은 항목이 여러 열에 흩어진 표** — 예: `1월`·`2월`·`3월` 식으로 가로로 펼쳐진 표 (이런 형태를 **감지해 "세로로 펼치면 분석하기 좋다"고 알려줍니다**. ⚠️ 다만 가로→세로 자동 변환은 v1 범위 밖이라, 정돈본은 가로 형태 그대로 두고 변환은 권고만 합니다)

## 파일 전달 방법

### 파일 경로 알려주기

```
/Users/me/Downloads/sales_report.xlsx 정돈해줘
```

### 현재 폴더 기준 경로

```
data/monthly_report.csv 파일 정리해줘
```

### 구체적인 문제 설명 추가

문제가 어디 있는지 알면 더 빠르게 처리합니다.

```
이 파일 3행부터 실제 데이터인데 1-2행은 제목이야
```

```
D열 소계 행은 제거하고 정돈해줘
```

## 3단계 안전 절차

이 스킬은 파일을 바로 수정하지 않고 3단계를 거칩니다.

1. **진단** — 파일 구조를 살펴보고 어떤 문제가 있는지 정리해 알려줍니다.
2. **가설 확인** — 발견한 문제와 교정 방향을 제시하고 사용자에게 "이렇게 해도 될까요?" 묻습니다.
3. **정돈** — 사용자가 승인한 뒤에만 정돈본 파일을 만듭니다.

가설 확인 단계에서 "아니요"라고 하면 즉시 멈춥니다. 잘못된 교정이 원본을 망치는 일은 없습니다.

## 정돈 후 분석하려면

정돈이 끝난 `tidy_*.csv` 파일은 data-analysis-advisor에 바로 넘길 수 있습니다.

```
정돈한 파일로 매출 분석해줘
```

두 스킬이 연결되어 "정돈 → 분석" 파이프라인을 만듭니다.

## 원본 파일 보호 규칙

- **원본은 절대 안 바뀝니다** — 어떤 경우에도 원본 파일을 덮어쓰지 않습니다.
- **정돈본은 별도 새 파일** — `tidy_[원본이름]_[날짜].csv` 형식으로 따로 저장됩니다.
- **사용자 확인 없이는 저장 안 함** — 가설 확인 단계에서 승인해야만 파일이 만들어집니다.

## 변환 결과 확인

정돈이 끝나면 두 가지를 받습니다.

- **정돈본 파일 경로** — 바로 쓸 수 있는 새 파일 위치
- **변환 로그** — 어떤 행이 제거됐는지, 날짜가 어떻게 바뀌었는지 기록

변환 로그를 보고 의도와 다른 처리가 있으면 가설을 조정해 다시 실행할 수 있습니다.

### 변환 전후 예시

원본 (지저분):

```
[2024년 1분기 매출]
부서       매출일        매출
영업1팀    2024-01-05    100
영업1팀    2024.2.3      120
영업2팀    20240115      80
소계                     300
```

정돈본 (분석 가능):

```
부서,매출일,매출
영업1팀,2024-01-05,100
영업1팀,2024-02-03,120
영업2팀,2024-01-15,80
```

이 정돈본은 실제로 스킬을 돌려 나온 결과입니다. 적용된 정돈:

- **제목 행 제거** — 맨 위 `[2024년 1분기 매출]` 같은 비데이터 머리 행을 걷어냅니다.
- **소계·합계 행 제거** — 중간에 끼어 있던 `소계` 행을 본문에서 분리합니다.
- **날짜 형식 통일** — `2024.2.3`, `20240115`처럼 제각각인 날짜를 `2024-02-03`, `2024-01-15`(ISO 8601 `YYYY-MM-DD`)로 맞춥니다.
- 이 밖에 **여러 줄 헤더 평탄화**, **선후 공백 제거**, **완전 중복 행 제거**를 합니다.

⚠️ **가로→세로(피벗 해제)는 하지 않습니다.** `1월`·`2월`·`3월`이 가로로 펼쳐진 표를 받으면 "세로로 펼치면 분석하기 좋다"고 **진단·권고만** 하고, 정돈본은 가로 형태를 그대로 둡니다. 실제 세로 변환이 필요하면 정돈본을 data-analysis-advisor로 넘겨 이어가세요.

## 정돈 범위와 한계 (v1)

지금 버전(v1)에서 자동으로 해주는 것과 아직 안 하는 것을 명확히 알려드립니다.

**해주는 것**

- 제목·주석 등 비데이터 머리 행 제거
- 여러 줄로 나뉜 헤더를 한 줄로 평탄화
- 소계·합계 행을 본문에서 분리(제거)
- 날짜 형식 ISO 8601(`YYYY-MM-DD`) 통일 (순서가 명확한 경우만 — `5/1/26`처럼 d/m·m/d가 모호하면 추측하지 않고 어느 형식인지 되묻습니다)
- 셀 선후 공백 제거, 완전 중복 행 제거

**아직 안 하는 것 (v1 한계)**

- ⚠️ **여러 표가 한 파일에 있으면 첫 번째 표만 정돈본에 담깁니다.** 빈 행 등으로 표가 둘 이상 나뉘어 있으면 두 번째 표부터는 제외되고, 변환 로그에 "v1 첫 표만 포함"으로 기록됩니다. 두 번째 표가 필요하면 그 부분만 따로 떼어 다시 정돈해 주세요.
- ⚠️ **가로→세로(피벗 해제), 결측값 채우기, 이상치 제거, 파생 변수·조인·통계 분석은 범위 밖**입니다 — 이런 작업은 data-analysis-advisor로 안내합니다.

## xlsx 파일 처리

엑셀 파일(`.xlsx`)은 병합된 셀을 풀어서 읽는 추출 단계가 먼저 필요합니다. 이 추출은 Claude가 보조 작업으로 수행하며(openpyxl 활용), 그 결과를 받아 이후 정돈 과정은 CSV와 동일하게 진행됩니다. ⚠️ CSV는 스킬이 곧바로 읽지만, xlsx는 이 추출 단계가 정상 동작해야 정돈이 됩니다. 추출이 안 되면 엑셀을 CSV로 저장해서 주시면 가장 확실합니다.

## 자주 묻는 질문

**Q. 원본 파일이 정말 안 바뀌나요?**
예. 스킬은 원본 파일을 열어서 읽기만 합니다. 쓰기는 `tidy_*.csv`에만 합니다.

**Q. 여러 시트가 있는 엑셀은요?**
시트 이름을 지정하면 해당 시트만 처리합니다.

```
1월시트 내용 정돈해줘
```

**Q. 정돈 결과가 마음에 안 들면요?**
가설 확인 단계에서 "이 항목은 빼줘"라고 하면 조정됩니다. 파일이 생성된 후에도 다시 실행하면 됩니다.

**Q. 별도 설치나 키 발급이 필요한가요?**
아닙니다. 설치·회원가입·API 키 모두 필요 없이 바로 씁니다. CSV 파일은 스킬이 곧바로 읽어 정돈합니다.

**Q. 엑셀(xlsx)도 CSV처럼 바로 되나요?**
완전히 같지는 않습니다. CSV는 스킬이 직접 읽지만, xlsx는 병합 셀을 풀어 표로 바꾸는 추출 단계가 먼저 필요합니다(Claude가 openpyxl로 보조 수행). 이 추출이 정상 동작하면 이후는 CSV와 동일하게 정돈됩니다. 혹시 추출이 잘 안 되거나 결과가 비면, 엑셀을 CSV로 저장해서 주시는 편이 가장 확실합니다.

**Q. 정돈된 파일은 어디에 저장되나요?**
지금 작업 중인 폴더 안에 자동으로 만들어집니다. 결과 보고에 정확한 경로가 같이 표시되니 그대로 열어보면 됩니다.
