---
title: "data-audit 상세 가이드"
---

엑셀·스프레드시트의 수식 오류와 흔한 실수를 훑어 위험한 셀을 짚어주는 감사 스킬입니다. **Claude에게 말로 부탁하면 됩니다** — "이 시트 감사해줘", "복붙하다 뭐 깨졌는지 봐줘"처럼요. 발견을 먼저 보고하고, 확인 없이는 셀을 바꾸지 않습니다.

---

## 빠른 시작

```
이 시트 감사해줘
수식 오류 찾아줘
복붙하다 뭐 깨졌는지 봐줘
```

파일 경로나 "모델에 뭔가 이상해" 같은 요청만 주면, 스킬이 전체 워크북(숨은 시트 포함)을 훑어 위험한 셀을 정리해 보고합니다.

---

## 활용 시나리오

### 치명적 오류 찾기

`#REF!`·`#VALUE!`·`#N/A`·`#DIV/0!`·`#NAME?` 같은 수식 오류, 존재하지 않는 시트를 가리키는 깨진 링크, 순환참조를 Critical로 짚어줍니다.

```
이 워크북에서 오류난 수식이랑 깨진 링크 찾아줘
```

### 복붙 실수 점검

수식이 값으로 뭉개진 셀, 이웃과 다른 계산식, `=A1*1.05`처럼 숫자를 박아둔 하드코드를 Warning으로 표시합니다.

```
복붙하다 수식 깨진 데 있는지 봐줘
```

### 범위 누락(off-by-one) 확인

`SUM`이 데이터 첫 행이나 마지막 행을 빠뜨린 곳을 찾아, 합계가 미묘하게 안 맞는 원인을 짚어줍니다.

```
합계가 안 맞는데 집계 범위 빠진 데 있는지 봐줘
```

---

## 출력 옵션

| 출력 방식 | 설명 | 사용 시점 |
|---|---|---|
| 발견 테이블 + 요약 | Sheet·Cell·Severity·Category·Issue·Fix 표 + "Critical N · Warning N · Info N" 요약 한 줄 | 사람이 눈으로 훑어볼 때 (기본) |
| 기계용 JSON | 발견 목록을 구조화된 데이터로 반환 | 다른 도구·스크립트로 넘길 때 |
| 실시간 하이라이트 + 코멘트 | 열어둔 엑셀에 Critical 연빨강 / Warning 연노랑 + 코멘트, 숨김은 시트탭 노트 | 엑셀을 열어두고 함께 볼 때 (Windows + Office) |
| 지적 없이 결과만 | 열린 시트를 건드리지 않고 발견 목록만 반환 | 시트를 그대로 두고 확인만 할 때 |

원하는 방식을 자연어로 말하면 됩니다 — 예: "결과만 JSON으로 받을게", "셀은 건드리지 말고 목록만 보여줘".

---

## 열어둔 엑셀 실시간 지적 (Windows + Office)

Windows + Microsoft Office 환경에서 엑셀을 열어둔 상태라면, 파일을 따로 저장하지 않고도 열린 워크북에 직접 지적을 남길 수 있습니다. "지금 열어둔 엑셀 감사해서 위험한 셀에 표시해줘"처럼 말하면 됩니다.

- 셀 단위 발견(수식 오류·하드코드·off-by-one·순환참조·깨진 링크·복붙·단위 급변)은 하이라이트(Critical 연빨강 / Warning 연노랑) + 코멘트로, 숨긴 시트·행 같은 Info는 시트탭 노트로 표시됩니다.
- 열린 워크북의 **실제 계산값**을 읽으므로, 파일 감사의 캐시 한계(아래 팁 참조)가 없습니다.
- 파일 감사와 **같은 9종 감사 규칙**을 공유합니다(두 경로 결과가 같도록 보장).
- 사전에 hyve를 켜고 설정의 MCP 탭에서 office 프리셋을 등록해두면 됩니다.
- macOS/Linux이거나 Office가 없으면 이 경로는 쓸 수 없어, 안내 후 파일 기반 감사로 전환합니다.

---

## 팁

- **보고 우선**: 확인 없이 셀을 바꾸지 않습니다. 특정 발견을 고쳐달라고 하면 그 셀만 수정하고 원본 백업을 남깁니다.
- **정확성 우선**: 확신이 낮은 발견은 severity를 낮추고 근거를 함께 냅니다. 멀쩡한 셀을 틀렸다고 하지 않는 쪽으로 보수적으로 판정합니다.
- **동적 에러는 한 번 열어 저장 후 감사**: 파일 감사는 수식을 재계산하지 않아, 한 번도 엑셀/LibreOffice로 열어 저장한 적 없는 파일은 `#REF!`·`#DIV/0!` 같은 동적 에러 값을 놓칠 수 있습니다. 한 번 열어 계산·저장한 뒤 감사하면 정확합니다.
- **문자열 기반 검사는 캐시 한계 없음**: 하드코드·off-by-one·순환참조·깨진 링크·복붙 검사는 수식 문자열을 보므로 위 한계와 무관하게 동작합니다.
- **전체 워크북 한 번에**: 시트를 지정하지 않으면 숨은 시트까지 포함해 전체를 감사합니다.

---

## 제한사항

- **재무모델 무결성은 범위 외**: BS balance·cash tie-out·재무제표 3표 정합, DCF/LBO/3-statement/Comps 같은 모델별 버그는 잡지 않습니다. 본 스킬은 수식·데이터 레벨 감사입니다.
- **Excel 애드인 경로 아님**: Office JS 애드인이 아니라 파일 기반 감사입니다.
- **파일 감사는 재계산 없음**: 동적 에러 값은 파일에 마지막 저장된 캐시값 기준으로만 읽힙니다(실시간 지적 경로는 실제 계산값을 읽어 이 한계가 없음).
- **실시간 지적은 Windows + Office 전용**: macOS/Linux 또는 Office 미설치 시 파일 기반 감사로 안내 후 전환합니다.
- **실험 단계(experimental)**: 인큐베이션 중인 스킬로, 발견 결과는 참고용으로 검토 후 활용하세요.
