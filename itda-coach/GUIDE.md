# itda-coach 시작 가이드 (처음이세요?)

코드 몰라도 됩니다. **그냥 평소 말로** 고민을 적으면 코치가 알아서 다음을 안내해요. 스킬 이름을 외울 필요도 없습니다.

## 1. 막막하면 이 한 줄부터

Claude Cowork 채팅창에 **그대로** 적어 보세요:

> 업무 중에 반복되거나 불편한 게 있는데, 뭘 자동화하면 좋을지 같이 골라줄래요?

그러면 코치가 질문 몇 개로 같이 좁혀 줍니다. (길게 안 써도 돼요. 떠오르는 대로.)

## 2. 이럴 땐 이렇게 말하세요 (복붙용)

| 지금 내 상황 | 채팅에 이렇게 |
|---|---|
| 매주 손으로 만드는 게 있다 | "매주 ○○ 보고서를 손으로 만드는데, 자동화해보고 싶어요." |
| 하고 싶은 건 큰데 오늘 1시간 안엔 뭘 할지 모르겠다 | "이거 너무 큰데, 오늘 1시간 안에 만들 수 있는 한 조각으로 잘라줄래요?" |
| 엑셀 같은 파일은 있는데 뭘 해야 할지 막막하다 | "이 파일로 뭘 할 수 있는지 같이 봐줄래요?" (+ 파일을 작업 폴더에 둠) |
| 같은 작업을 매번 반복한다, 한 번 만들어두고 싶다 | "이 작업을 매번 똑같이 하고 싶어요. 한 번 만들어두고 부를 수 있게 해줄래요?" |

어떤 표현을 써도 코치가 알아서 맞는 단계로 안내합니다. **단계를 직접 고르지 않아도 돼요.**

## 3. 결과는 "오늘 눈에 보이는 하나"

코치는 정답을 대신 끝내주기보다, **오늘 1시간 안에 눈에 보이는 결과 하나**(표·요약·문서 한 장)를 같이 만들고 다음엔 혼자 하는 법을 챙겨줍니다. 크게 잡아 못 끝내는 것보다, 작게 하나 끝내는 게 낫습니다.

## 4. 꼭 아는 단어 3개 (나머진 몰라도 됨)

- **워크스페이스 = 내 컴퓨터의 폴더 하나.** Claude에게 "이 폴더를 봐 줘" 하고 정해 둔 곳이에요. 거기에 파일을 넣으면 Claude가 열어볼 수 있어요. (강의에선 어떤 폴더를 쓸지 진행자가 정해 줍니다.)
- **Cowork = 지금 쓰는 이 채팅 방식.** "Code"라는 개발자용 도구 얘기가 나와도 **지금은 몰라도 됩니다** — 더 본격적으로 키울 때 쓰는 거예요.
- **커넥터 = 메일·드라이브 같은 외부 서비스에 미리 연결해 둔 통로.** 없어도 대부분 됩니다.

## 5. 회사 데이터, 안전하게 (안심하세요)

- **파일은 정해 둔 폴더 안에만 둡니다 — 그게 곧 "밖으로 안 내보낸다"는 뜻이에요.** 따로 외부 사이트에 올리지 않는 한, 작업은 그 폴더 안에서 이뤄집니다.
- **연습할 땐 진짜 데이터 말고 "가짜로 바꾼 표본"이면 충분합니다.** 이게 *마스킹* — 예: 거래처명을 'A사·B사'로, 실제 금액을 임의 숫자로, 사람 이름은 지움. (가릴 것: 거래처·계좌번호·실명·실제 금액)
- 진짜 데이터를 넣는 건 **회사 보안 규정을 확인한 뒤**. "BigQuery" 같은 **사외 클라우드** 얘기가 나오면 그건 사내 승인이 필요한 외부 서비스라는 신호예요.

## 6. 스킬 4개? — 외우지 마세요

코치가 상황에 맞는 도구를 알아서 씁니다. 굳이 궁금하면:

| 영어 이름 | 한국어로 | 언제 |
|---|---|---|
| `problem-guide` | 문제 구체화 | "원하는 대로 안 나와요", 막연할 때 |
| `analysis-guide` | 데이터 분석 길잡이 | 파일은 있는데 뭘 할지 막막할 때 |
| `hour-slice` | 1시간 조각 자르기 | 너무 큰데 오늘 뭘 할지 모를 때 |
| `miniskill-forge` | 미니스킬 만들기 | 반복 작업을 한 번 만들어둘 때 |

이 안내 파일들(SKILL.md)은 **Claude가 읽어요. 여러분은 말만 하면 됩니다.**
