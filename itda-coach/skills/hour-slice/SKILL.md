# hour-slice — 오늘 1시간 안에 만들 한 조각으로 자르기

## 이 스킬이 푸는 문제

"업무를 이렇게 바꾸고 싶다"는 그림은 있는데, 막상 **오늘 1시간 안에 뭘 만들 수 있을지** 감이 안 온다. 크게 잡으면 1시간 안에 아무것도 못 끝내 "역시 어렵네"로 좌절하고, 줄이자니 **뭘 줄여야 할지**도 모르겠다. 이 스킬은 그 사이를 메운다 — 큰 문제를 **손에 있는 것만으로 ~1시간 안에 눈에 보이는 결과가 나오는 한 조각**으로 잘라, "오늘은 이거 하나"를 또렷이 손에 쥐여 준다.

## 무엇이 "1시간 조각"인가 — 그리고 무엇이 아닌가

- **맞다(○)**: 첨부한 CSV 한 개로 표·그래프 한 장 / 회의록 한 건을 정해진 양식으로 요약 / 메일 10통 분류 규칙 초안 / 보고서 1장 템플릿화 — *지금 손에 있는 것*으로 *눈에 보이는 결과 1개*.
- **아니다(✗)**: "전사 대시보드 구축", "사내 시스템 연동 자동화", "완전 무인 자동화" — 새 계정·권한 신청·여러 산출물·실시간이 끼면 1시간을 넘는다. → 이건 **자르기 대상**이지 오늘 만들 것이 아니다.

## 핵심 원칙

- **가시적 결과 우선**: 1시간 끝에 *눈에 보이는 산출물 1개*가 남는 조각을 고른다. "기반 작업만"·"설계만"은 데모로 약하다.
- **손에 있는 표면만**: 새 설치·새 계정·승인 대기는 1시간 초과 신호. 오늘 쓸 수 있는 표면(아래 체크리스트) 안에서만 자른다.
- **더 작게가 기본값**: 애매하면 더 자른다. 작아서 일찍 끝나면 한 조각 더 붙이면 된다 — 커서 못 끝내는 것보다 낫다.
- **판단이 막혀도 멈추지 않는다**: AI가 100% 판단 못 하는 부분은 사람이 사후 가감하는 *부분 자동화*로 줄여서 시작한다.

## 입력

구체화된 문제 한 줄(또는 problem-guide 산출물). 아직 *무엇을* 할지조차 안 정해졌으면 find-work(itda-work)로 발굴부터, 문제는 있는데 막연하면 problem-guide로 먼저 좁히고 오라고 안내하되, 가벼운 건 여기서 한 번에 좁혀도 된다.

## 진행 절차

### 1단계 — 문제 재진술 (1줄)
"지금 풀려는 건 이거죠?"로 한 문장 확인. 어긋나면 사용자 언어로 고친다.

### 2단계 — 후보 조각 분해 (3~5개)
큰 문제를 *부분*으로 쪼개 3~5개의 후보 조각을 나열한다. 각 조각은 **"입력 → 한 동작 → 산출물 1개"**로 쓴다. 예 — "월간 거래처 리스크 리포트"를 쪼개면:
- ⓐ 거래처 1곳 뉴스 5건을 한 줄 요약 표로
- ⓑ 지난달 리포트 1건을 정해진 5단 양식으로 재구성
- ⓒ 거래처 목록 CSV에서 위험 키워드 포함 행만 추리기

### 3단계 — 1시간 실현가능성 채점
각 후보를 아래 **체크리스트**로 본다. 표면이 불확실하면 **AskUserQuestion으로 "오늘 쓸 수 있는 게 어디까지인지" 먼저 확인**한다(아래 예).

### 4단계 — 한 조각 채택 + 명세
체크리스트 5축이 다 왼쪽인 **가장 작은** 조각을 고르고, "오늘의 한 조각" 템플릿을 채운다. **"이번엔 안 하는 것"을 반드시 적는다** — 범위가 다시 부풀지 않게.

### 5단계 — 핸드오프
- 데이터를 앞에 두고 *지금 같이 실행* → **analysis-guide**
- 이 조각이 *반복*이라 매번 자동으로 → **miniskill-forge**
- 조각이 여전히 크면 → 2단계로 돌아가 더 자른다

## 1시간 실현가능성 체크리스트 (이 스킬의 핵심)

| 축 | 1시간 조각이면 (왼쪽) | 초과 신호 → 자르기 (오른쪽) |
|---|---|---|
| **표면** | Cowork 단독 / 첨부파일 / 이미 설치된 스킬·커넥터 / 브라우저로 끝 | 새 설치·새 계정·승인 대기·미연동 사내 시스템 |
| **데이터** | 지금 손에 있다(첨부·복붙·공개) | 추출 요청·권한 신청 대기 → 표본·더미로 대체 |
| **산출물** | 눈에 보이는 1개(표·그래프·요약·문서 1장) | 여러 개·실시간·완전 무인 |
| **판단** | 규칙이 명료하거나, 모호한 부분은 사람이 사후 가감 | AI가 100% 판단해야만 함 → 부분 자동화로 축소 |
| **검증** | 1시간 끝에 결과가 맞는지 눈으로 확인 가능 | 맞는지 확인에 또 며칠 |

5축이 다 왼쪽이면 1시간 조각. 하나라도 오른쪽이면 **그 축을 줄여** 다시 본다.

### AskUserQuestion 예 — 표면 확인 (3단계)
- 질문: "오늘 이 자리에서 바로 쓸 수 있는 건 어디까지인가요?"
- 옵션: (1) Claude Cowork 채팅만 (2) +내 파일 첨부 가능 (3) +이미 설치한 스킬/커넥터 있음 (4) +브라우저로 사이트 접근 OK
- 답에 따라 후보 조각의 실현가능성이 갈린다(예: "사내 ERP 연동"이 필요한 조각은 (1)~(4) 어디에도 안 들어오면 오늘은 표본 CSV로 대체).

## 출력 — "오늘의 한 조각" 명세

```markdown
## 오늘의 한 조각
- 만들 것(한 줄):
- 입력(지금 손에 있는 것):
- 산출물(눈에 보이는 1개):
- 쓰는 표면: <Cowork 단독 / +첨부 / +설치된 스킬 / +브라우저>
- 성공 판정(1시간 끝에 이게 보이면 OK):
- 이번엔 안 하는 것 (out-of-scope):
  -
- 다음 단계: <analysis-guide 라이브 실행 / miniskill-forge 굳히기>
```

## 종료 / 핸드오프

조각이 정해지면 그 한 문단을 사용자에게 건네고, 5단계 핸드오프 중 하나로 넘긴다. 1시간이 남으면 "안 하는 것"에서 한 조각을 더 꺼내 붙인다 — **작게 끝내고 키우는 게, 크게 시작해 못 끝내는 것보다 항상 낫다.**
