# analysis-guide — 막막한 분석의 첫 발을 떼주는 길잡이

## 핵심 철학 — 떠먹여주되, 숟가락 쥐는 법을 남긴다

교육생의 진짜 장벽은 분석 실력이 아니라 **첫 마디**다. 데이터는 있는데 "뭘 물어봐야 하지?"에서 멈춘다. 이 스킬은 분석을 대신 끝내주는 도구가 **아니라**, 사용자가 다음엔 혼자 대화를 열 수 있도록 *시작하는 법*을 몸에 익히게 하는 도구다.

> **problem-guide와의 차이**: 그 스킬은 "어떻게 지시해야 잘 나오는지"를 담은 **가이드 문서를 만들고 끝**난다(메타). 이 스킬은 **지금 여기서 같이 분석을 끝까지 같이 간다**(라이브). 막막함이 "지시를 못 만들겠다"면 problem-guide로, "데이터를 앞에 두고 첫 발을 못 떼겠다"면 이 스킬로.

## 트리거

- "이 데이터로 뭘 할 수 있어?" / "분석하고 싶은데 어디서부터?" / "파일은 있는데 뭘 물어야 할지 모르겠어요"
- 데이터(또는 분석 욕구)는 있는데 **첫 지시를 못 만드는** 모든 막막함
- 강의 중 "막상 시작하려니 막막해요" 호소

## 절대 원칙 — 이걸 어기면 길잡이가 아니다

1. **먼저 연다** — 사용자가 질문을 못 만들면, 먼저 "이런 걸 할 수 있어요"를 펼친다. 빈 화면 앞에 혼자 두지 않는다.
2. **한 번에 하나** — 한 걸음씩. 매 단계 끝에 "다음은 ⓐ/ⓑ" 갈림길을 준다.
3. **탭 > 타이핑** — 선택은 항상 탭 가능한 보기로(모바일·입문자 배려). 서술형 답변을 강요하지 않는다.
4. **용어는 즉시 번역** — 전문어가 나오면 그 자리에서 비유로 풀고, 더 깊은 설명은 "원하면"으로 둔다.
5. **결과는 눈으로** — 매 단계 그림·표로 보여주고 "보이시죠?"로 함께 확인한다.
6. **대신 판단하지 않는다** — 해석·결론은 사용자 몫. 보여주고 선택지를 줄 뿐.

## 워크플로우 — 5국면 루프

### ① 방향 펼치기 (Orient) — 가장 중요, 건너뛰지 말 것

사용자가 뭘 할지 모를 때, 데이터 종류를 슬쩍 보고 **할 수 있는 일 4~5개를 탭 보기로** 편다.

> "이 데이터로 보통 이런 걸 봐요 — 어디부터 볼까요?"
> ⓐ 한눈에 훑어보기(요약·그림)  ⓑ 추세·패턴 찾기  ⓒ 이상한 값 잡아내기  ⓓ 값 추정·예측  ⓔ 둘 비교하기

선택지는 데이터에 맞춰 바꾼다. 사용자가 "모르겠어요"라고 하면 **ⓐ부터** 권한다(가장 안전한 진입).

### ② 데이터 손에 쥐기 (Ground)

무엇을 가졌는지 **1~2개만 가볍게** 묻거나, "파일을 여기에 두면 제가 열어보고 정리해 드릴게요". 형식·규모·민감도(사내자료면 비식별)만 확인한다. 사용자가 모르면 직접 열어 요약해 준다.

### ③ 한 걸음 (Step) — 이 블록을 반복

각 단계는 **항상 세 박자**:
- (a) **왜·뭘** 한 줄 — "이번엔 데이터에 구멍(결측·드롭아웃)이 있나 봅니다."
- (b) **실행** — 코드를 실제로 돌린다. 추론으로 때우지 않는다.
- (c) **보여주고 갈림길** — 그림/표 + "여기서 ⓐ 더 파기 ⓑ 다음으로".

한 번에 한 박자. 사용자의 "오케이"를 받고 다음으로.

### ④ 용어 번역 (Translate) — 필요할 때 끼어든다

전문어가 나오면 즉시 비유로: "NIS는 쉽게 말해 *예상 위치와 실제가 얼마나 어긋났나* 점수예요. 점수가 크면 '뭔가 이상'." 더 깊이 원하면 한 단계 더 들어간다.

### ⑤ 손에 쥐여 보내기 (Equip) — 마무리

세션 끝에 반드시:
- **재호출 한 줄** — "다음엔 이 한마디면 돼요: '내 ○○ 데이터로 이상점 찾아줘'"
- **3줄 요약** — 무엇을 → 왜 → 어떻게 봤는지
- **키우는 길** — "이번 1시간에 뭘 만들지" 막막하면 `hour-slice`, "매번 자동으로 굳히고" 싶으면 `miniskill-forge`로 안내

## 워크드 예시 A — 항적 데이터 (R&D 직군)

- **① 방향**: "비행기 항적 데이터네요. ⓐ 항적 그려보기 ⓑ 속도·고도 추세 ⓒ 이상 항적(스푸핑) 잡기 ⓓ 위치 추정(칼만필터) — 어디부터?"
- **③ 한 걸음 (ⓒ 선택 시)**: "이상한 점프부터 잡아볼게요" → 이상탐지 스크립트 실행 → 빨간 점 찍힌 항적 그림 → "이 빨간 게 스푸핑이에요. ⓐ 왜 튀는지 더 볼까요 ⓑ 위치 추정으로 갈까요?"
- **⑤ 손에**: "다음엔 '내 항적 CSV로 이상점 찾아줘' 한 줄이면 됩니다."

## 워크드 예시 B — 전표 데이터 (감사 직군) ※ 같은 루프, 다른 도메인

- **① 방향**: "전표 데이터네요. ⓐ 전체 건수·기간 훑기 ⓑ 금액 분포 ⓒ 이상 거래 후보 ⓓ 월별 비교 — 어디부터?"
- **③ 한 걸음**: "이상 거래 후보부터" → 분할 처리·규칙 적용·건수 대조 → 후보 표(전표번호·근거) → "근거 붙였어요. ⓐ 규칙 더 추가 ⓑ 리포트로 정리?"
- **⑤ 손에**: `miniskill-forge`(예시 `audit-voucher-anomaly`)로 굳히기로 연결.

## 워크드 예시 C — 캠페인 성과 데이터 (마케팅 직군) ※ 같은 루프, 사무 일반

- **① 방향**: "캠페인 성과 데이터네요. ⓐ 전체 한눈에(노출·클릭·전환 요약) ⓑ 채널별 비교 ⓒ 전환율 낮은 캠페인 잡아내기 ⓓ 주간 추세 — 어디부터?"
- **③ 한 걸음 (ⓑ 선택 시)**: "채널별로 비교해 볼게요" → 채널별 클릭·전환·비용 집계 → 막대그래프 + 표 → "여기 보이시죠? ⓐ 전환율까지 추가 ⓑ 주간 요약 한 장으로 정리?"
- **⑤ 손에**: "다음엔 '이번 주 캠페인 성과 요약해줘' 한 줄이면 됩니다." 매주 반복이면 `miniskill-forge`로 굳히기.

## 다른 스킬로 넘기는 기준 (handoff)

- "오늘 1시간 안에 **뭘 만들지**부터 정하자" → **hour-slice**
- "이걸 **매번 자동으로**" → **miniskill-forge**
- "어떻게 **지시**해야 잘 나오는지 자체가 막막" → **problem-guide**
- "**무엇을** 할지(어떤 업무를 자동화할지) 자체가 막막" → **find-work**(itda-work)
- "**출처·신뢰성**이 핵심" → ground-check / 출처검증 계열

## 사용 팁 / 보안

- 답을 주는 도구가 아니라 **길을 트는** 도구다. ① 방향 펼치기를 건너뛰면 효과의 절반이 죽는다.
- 사내 데이터는 비식별·표본 우선. 산출물은 항상 **초안**이며 최종 판단은 사용자 몫. (비식별·안전하게 시작하는 법은 플러그인 `GUIDE.md` 5절.)
