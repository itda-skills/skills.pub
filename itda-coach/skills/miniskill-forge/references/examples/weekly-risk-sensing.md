# 예시: 경영진단·감사 — 주간 리스크 센싱 리포트

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 경영진단·감사 담당자
- 반복 작업: 피감·진단 대상사의 소송·제재·신용 이벤트와 산업·규제 동향을 매주 같은 축으로 모니터링해 1쪽 리포트로 작성
- 굳히면 좋아지는 것: 누락 방지(손 검색은 이슈를 빠뜨림) + 일관성(작성자·주차마다 형식·깊이가 흔들리던 것을 고정)

## 워크스페이스에 둘 자료 (있는 것만)
- 주간 보고 양식 1부(과거 리포트 1~2건)
- 모니터링 키워드·대상사 목록 메모(예: `센싱_키워드.md` — 규제·제재·소송·신용 키워드 + 피감/계열/거래처 회사명)
- 자주 보는 공개 출처 목록(DART·산업/규제 전문지·통계 포털 URL). 사내 감사증적·전표·원장 같은 비공개 자료는 두지 않음

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 경영진단·감사 담당자입니다. 매주 같은 형식의 '주간 리스크 센싱 리포트'를 만들고 싶습니다. 입력: 모니터링 키워드 목록(예: 소송·제재·과징금·신용등급·회계부정·규제 변화)과 대상사 목록(피감·계열·관계사·주요 거래처), 조회 기간(기본 최근 7일). 처리: 공개 웹과 공시(DART)만 사용해 해당 기간 이슈를 수집하고, 우리 조직·감사 대상에 대한 영향도(상/중/하)로 분류하고, 확인된 사실과 추정을 분리하고, 각 항목의 1차 출처 URL을 확인합니다. 출력: 제목·작성일·핵심 요약 3줄·이슈 표(제목/한 줄 요약/영향도/출처 URL, 영향도 높은 순)·다음 주 모니터링 포인트로 구성된 1쪽 마크다운. 제약: 사내 감사증적·전표·원장 등 비공개 데이터는 사용·업로드하지 않고 공개 출처만 사용, 모든 수치·이슈에 출처 표기, 출처 불명확 항목은 [확인 필요] 표시, 단정 대신 사실/추정 구분. 이 리포트는 담당자 점검용 초안이며 최종 판단은 담당자·감사위원회가 함을 명시. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `risk-sensing-report`
- 트리거: "리스크 센싱 돌려줘", "주간 센싱 리포트 만들어줘", "이슈 센싱해줘" (기간 미명시 시 최근 7일)
- 워크플로우 핵심: (1) `센싱_키워드.md`에서 키워드·대상사 읽기(없으면 기본 축) → (2) 키워드·대상사별 공개 웹+공시(DART) 수집 → (3) 조직·감사 대상 영향도(상/중/하) 분류 → (4) 사실/추정 분리·1차 출처 URL 확인([확인 필요]) → (5) 1쪽 마크다운 작성, `센싱리포트_{날짜}.md` 저장
- 출력 포맷: 핵심 요약 3줄 + 이슈 표(제목·요약·영향도·출처, 영향도순) + 다음 주 모니터링 포인트, "최종 판단은 담당자·감사위원회" 명시

## 사용 예시 프롬프트
- "리스크 센싱 돌려줘. 기간은 최근 5일로."
- "주간 센싱 리포트 만들어줘. 이번엔 {대상사}의 소송·제재 이슈를 우선으로."
- "이번 센싱 결과를 감사위원회 보고용 1쪽 초안으로 이어서 만들어줘. 출처 미확인 항목은 따로 빼줘."
