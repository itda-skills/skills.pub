# 예시: 정보보안·감사 — 업로드 전 민감정보 스캔·마스킹 체크

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 정보보안·감사 담당자(자료를 외부로 공유·업로드하기 전 점검하는 누구나)
- 반복 작업: 입력 텍스트/표에서 민감정보를 탐지하고 마스킹을 제안한 뒤 "그대로 가능/마스킹 후 가능/부적합"으로 안전 판정
- 굳히면 좋아지는 것: 일관성(손으로 "민감정보 있나 봐줘"는 매번 기준이 흔들림 → 누가/언제든 같은 탐지 항목·판정 기준) + 재현성

## 워크스페이스에 둘 자료 (있는 것만)
- 조직 식별자 형식 메모(사번 패턴, 내부 프로젝트 코드 규칙 예: `PRJ-####`) — 탐지 정확도 향상
- 마스킹된 가짜·비식별 검증 표본(실제 기밀 금지)

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "업로드·외부 공유 전에 입력 텍스트의 민감정보를 점검하는 미니스킬을 만들고 싶어. 탐지: 사번·주민등록번호·계좌번호·전화번호·이메일·개인 이름·사내 시스템/프로젝트 코드·피감 대상사명 같은 고유명사 후보를 모두 찾고, 확실하지 않아도 후보로 포함해 '확인 필요'로 표시해줘. 마스킹 제안: 각 항목을 자릿수 마스킹(`1234-**-****`)/가명처리(사번→`EMP-001`, 매핑은 별도 표로만)/범주화(금액·날짜를 구간·연도로) 중 하나로 제안. 안전 판정: 전체를 '그대로 가능(공개 정보뿐·식별자 없음)/마스킹 후 가능/부적합(기밀 원본, 사내 처리 권고)' 중 하나로 판정하고 근거를 한두 문장 첨부, 애매하면 단정 대신 '확인 필요'. 원칙: 최종 결정은 사용자가 하고 이 스킬은 보조 판정만 한다, 완전 탐지를 보장하지 않는다, 검증은 가짜·비식별 표본으로만 한다. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함. 우리 조직 식별자 형식(사번 패턴·내부 코드 규칙)은 내가 보태서 탐지 정확도를 올릴게."

## 도출될 SKILL.md 골격
- name: `presend-pii-scan`
- 트리거: "민감정보 점검", "올려도 되는지 봐줘"
- 워크플로우 핵심: (1) 탐지 — 주민번호/계좌/사번/전화/이메일/개인 이름/사내 코드/피감 대상사명을 후보까지 모두, 불확실은 '확인 필요' → (2) 마스킹 제안 — 자릿수 마스킹/가명처리(매핑 별도 표)/범주화 → (3) 안전 판정 — 그대로 가능/마스킹 후 가능/부적합 + 근거. 보조 판정만, 최종 결정은 사용자
- 출력 포맷: 탐지 표(유형/원문 일부 가림/위험도/마스킹 제안) + 판정(그대로 가능|마스킹 후 가능|부적합) + 근거 한두 문장

- "이 미니스킬을 가짜 표본으로 테스트하자. 일부러 사번·계좌를 섞은 예시를 넣을 테니 다 잡는지·과탐 없는지 확인해줘."
