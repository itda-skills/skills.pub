# 예시: 내부감사·경영진단 — 대용량 전표·원장 이상 거래 빠짐없이 스크리닝

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).
>
> ⚠️ **시작 전**: 민감한 전표는 `pii-scan-mask`로 먼저 가린 표본을 만든 뒤 이 점검을 돌린다(가리고 → 점검).

## 적합 상황
- 누가: 내부감사·경영진단 담당자
- 반복 작업: 전표·원장 데이터에서 이상 거래 후보를 매 건 같은 절차(건수 확인 → 구간 분할 → 이상규칙 적용 → 누락 대조 → 후보 표)로 점검
- 굳히면 좋아지는 것: 일관성(누가 쓰든 같은 규칙·같은 포맷) + 누락 방지(대용량을 한 번에 처리할 때 조용히 빠지는 행을 분할·건수 대조로 차단)

## 워크스페이스에 둘 자료 (있는 것만)
- 점검 규칙 메모 — 조직에서 자주 보는 이상 패턴 + 임계치(휴일·심야 시간대, 라운드 금액 단위, 한도 금액)
- 마스킹한 전표 표본 1~2건 — 컬럼 구조는 실제와 같되 금액·거래처는 비식별 더미
- 이상 후보 양식(선택) — 후보 표 헤더 구조(전표번호·적용 규칙·판단 근거·확인필요)

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 내부감사·경영진단 담당자입니다. 전표·원장 데이터의 이상 거래 점검을 자주 반복합니다. 입력: 워크스페이스 폴더의 전표 CSV/엑셀 1개(컬럼은 전표번호·일자(입력일시)·계정·금액·거래처·입력자·승인자 등). 사내 민감 데이터는 외부로 보내지 않고 워크스페이스 안에서만 다룹니다. 워크플로우: (1) 먼저 전체 행 수·컬럼·결측치·중복을 코드로 집계해 기준선('원본 N건')을 잡고, (2) 1,000행 단위로 구간 분할해 처리하며 각 구간 처리 건수를 보고하고, (3) 이상규칙(휴일·심야·주말 입력, 라운드 금액, 임계치 직하 금액, 동일금액 반복, 승인자=입력자, 한도 초과 분할 지급)을 적용해 의심 건을 뽑고, (4) 마지막에 '처리 건수 합 = 원본 건수'를 대조해 누락을 점검합니다(불일치 시 미처리 구간 재처리). 출력: 의심 거래 표(전표번호·적용 규칙·판단 근거·확인필요 Y/N) + 처리 요약 로그(원본 N / 처리 N / 누락 N / 중복제거 N) + 다음 확인 액션 목록. 모든 판단은 단정이 아니라 '추가 확인 필요 후보'로 표기. 제약: 사내 데이터 외부 업로드 금지, 추측 금지, 근거(전표번호) 필수. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `voucher-anomaly-screen`
- 트리거: "전표 이상점검 돌려줘", "원장 이상 징후 스크리닝", "이 전표 예외 거래 찾아줘"
- 워크플로우 핵심: (1) 기준선 집계로 원본 N건 확정 → (2) 1,000행 단위 분할 처리·구간별 건수 보고 → (3) 이상규칙 적용(시간/금액/통제 축) → (4) 처리 건수 = 원본 건수 누락 대조, 불일치 시 미처리 구간 재처리
- 출력 포맷: 의심 거래 표(전표번호·규칙·근거·확인필요) + 처리 요약 로그(원본/처리/누락/중복제거) + 다음 확인 액션, 모든 판단은 '추가 확인 필요 후보'(최종 판단은 담당자·감사위원회)

## 사용 예시 프롬프트
- "워크스페이스의 2026_3분기_전표.csv로 전표 이상점검 돌려줘."
- "이번 점검은 '승인자=입력자'와 '한도 초과 분할 지급'만 집중해서 봐줘."
- "방금 결과에서 처리 요약 로그만 다시 보여주고, 처리 건수가 원본과 맞는지 확인해줘."
