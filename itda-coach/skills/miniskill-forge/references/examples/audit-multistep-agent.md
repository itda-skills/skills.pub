# 예시: 감사·진단 — 대상사 배경 분석 멀티스텝(수집→분석→검증→초안)

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 감사·경영진단 담당자
- 반복 작업: 대상사 배경 분석을 기업 개요·재무 수집 → 시장·경쟁 분석 → 1차 출처 검증 → 보고서 초안의 멀티스텝으로 한 번에 처리
- 굳히면 좋아지는 것: 일관성(고정된 단계 순서로 매번 같은 형식) + 검증 누락 방지(보고 전 교차검증이 시퀀스에 내장돼 환각 차단) + 인수인계 용이

## 워크스페이스에 둘 자료 (있는 것만)
- 분석 양식 메모 — 대상사 배경 분석 표준 목차(개요·재무·시장·리스크·결론)와 항목별 필요 데이터
- 관심 축·체크 항목(소송·제재·신용 이벤트, 산업·규제 동향 등)
- 스킬 조합 메모(선택) — 단계별로 쓸 스킬(dart→market-scan→ground-check→draft-post). 공개 출처·비식별 자료만

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 내부감사·경영진단 담당자입니다. 대상사 배경 분석을 매번 같은 멀티스텝 절차로 수행하는 미니스킬을 만들고 싶습니다. 입력: 대상사명(1개) 또는 대상사 목록. 사내 미공개 자료는 쓰지 않고 공개 출처만 사용합니다. 멀티스텝 워크플로우: (1) 기업 개요·재무 수집(dart) — 프로필·재무·인력·주요 공시, (2) 시장·경쟁 분석(market-scan) — 산업 동향·경쟁사·시장 규모, (3) 1차 출처 검증(ground-check) — 수치·고유명사를 공개 1차 출처로 교차검증, 확인 안 되는 항목 분리, (4) 보고서 초안(draft-post) — 표준 목차(개요-재무-시장-리스크-결론)로 초안 작성. 출력: 단계별 산출 요약 + 최종 보고 초안 + '확인 필요 항목' 목록(단정 금지). 각 수치·인용에 출처를 붙입니다. 제약: 공개 출처만, 추측 금지, 확인된 사실과 추정 분리, 사내 데이터 외부 전송 금지. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우(단계별 스킬 호출)·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `audit-target-multistep`
- 트리거: "대상사 배경 분석 돌려줘", "이 회사 실사 초안 만들어줘"
- 워크플로우 핵심: (1) dart로 기업 개요·재무·인력·공시 수집 → (2) market-scan으로 산업 동향·경쟁사·시장 규모 → (3) ground-check로 수치·고유명사를 1차 출처 교차검증·확인 불가 항목 분리 → (4) draft-post로 표준 목차(개요-재무-시장-리스크-결론) 초안. 여러 스킬을 한 시퀀스로 묶는 오케스트레이션 명세
- 출력 포맷: 단계별 산출 요약 + 최종 보고 초안 + 확인 필요 항목 목록(단정 금지, 출처 미확인 분리). 모든 수치·인용에 출처, "최종 판단은 담당자·감사위원회"

- "방금 초안에서 출처가 안 붙은 수치만 따로 표로 뽑아줘."
