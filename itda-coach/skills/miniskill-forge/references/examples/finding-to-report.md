# 예시: 감사·진단 — 점검 메모를 정형 발견사항 보고서로 변환

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 감사·진단 담당자
- 반복 작업: 점검 메모를 받아 발견사항 5단(현황-기준-차이-영향-권고) 구조 + 경영진 한 장 요약 보고서 초안으로 변환
- 굳히면 좋아지는 것: 일관성(누가 쓰든 같은 모양) + 검증 누락 방지(5단 분해·가드가 스킬에 박힘) + 인수인계 용이

## 워크스페이스에 둘 자료 (있는 것만)
- 발견사항 보고 양식 1부(5단 헤더 구조 참고용)
- 점검 메모(채팅에 붙여넣거나 워크스페이스 파일로)

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 감사·진단 담당자입니다. 점검 메모를 받아 '현황-기준-차이-영향-권고' 5단 발견사항 보고서 초안 + 경영진 한 장 요약으로 변환하는 작업을 반복합니다. 입력: 점검 메모(채팅 텍스트 또는 워크스페이스 폴더의 파일). 절차: (1) 메모를 발견사항 단위로 분해, (2) 각 발견사항을 5단으로 구조화 — 현황은 관찰된 사실만(판단·추측 금지), 기준은 근거 규정·조항을 특정(메모에 없으면 '확인 필요'), 차이는 현황과 기준의 간극, 영향은 리스크·금액·범위 끝에 '(검수 필요)', 권고는 개선 방향 끝에 '(검수 필요)', (3) 발견사항을 표로 모으고 중요도는 '상/중/하 후보'로만, (4) 경영진 한 장 요약(핵심 3줄 + 주요 발견 Top 3 + 즉시 조치)을 붙임. 가드: 메모에 없는 수치·고유명사 생성 금지, 검증 안 된 수치는 '출처 확인 필요', 사내 데이터는 워크스페이스 밖으로 내보내지 않음, 산출물 끝에 '본 초안은 보조 자료이며 최종 감사의견은 담당자·감사위원회가 확정'을 명시. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `finding-to-report`
- 트리거: "발견사항 정리해줘", "점검 메모 보고서로 바꿔줘"
- 워크플로우 핵심: (1) 메모를 발견사항 단위로 분해 → (2) 각 건을 현황-기준-차이-영향-권고 5단 구조화(현황=사실만, 기준=조항 특정/없으면 '확인 필요', 영향·권고=끝에 '(검수 필요)') → (3) 표로 집계·중요도는 '상/중/하 후보' → (4) 경영진 한 장 요약(핵심 3줄·주요 발견 Top 3·즉시 조치) 첨부
- 출력 포맷: 발견사항 요약표(연번·요약·현황·기준·차이·영향(검수 필요)·권고(검수 필요)·중요도 후보) + 경영진 한 장 요약. 산출물 끝에 "최종 감사의견은 담당자·감사위원회 확정" 명시

## 사용 예시 프롬프트
- "발견사항 정리해줘. 워크스페이스의 점검메모.txt를 보고서 초안으로."
- "이번엔 중요도 '상 후보'로 분류된 발견만 경영진 요약에 넣어줘."
- "방금 초안에서 '확인 필요'로 남은 기준 조항만 표로 따로 뽑아줘."
