# 예시: 해외물류·글로벌 SCM — 주간 물류 이슈 센싱 리포트

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 해외물류·글로벌 SCM 담당자
- 반복 작업: 운하 통항·항만 적체·선사 운임·관세/무역 규제 같은 비슷한 축의 이슈를 매주 검색·정리해 1쪽 센싱 리포트로 작성
- 굳히면 좋아지는 것: 일관성(작성자·주차마다 흔들리던 형식·검증 기준을 한 문장 호출로 고정) + 시간 절감

## 워크스페이스에 둘 자료 (있는 것만)
- 주간 보고 양식 1부(과거 보고서 1~2건, 형식 참고용)
- 모니터링 키워드·무역로·해외 거점 목록 메모(예: `센싱_키워드.md`)
- 자주 보는 공개 출처 목록(운임지수 제공처·물류 전문지·통계 포털 URL)

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 해외물류(글로벌 SCM) 담당자입니다. 매주 같은 형식의 '글로벌 물류 이슈 센싱 리포트'를 만들고 싶습니다. 입력: 모니터링 키워드 목록(예: 수에즈·파나마 운하, 주요 항만 적체, 선사 운임 GRI, 미·EU 관세), 조회 기간(기본 최근 7일). 처리: 공개 웹 출처만 사용해 해당 기간 이슈를 수집하고, 우리 공급망 영향도(상/중/하)로 분류하고, 확인된 사실과 추정을 분리하고, 각 항목의 1차 출처 URL을 확인합니다. 출력: 제목·작성일·핵심 요약 3줄·이슈 표(제목/한 줄 요약/영향도/출처 URL, 영향도 높은 순)·운임·환율 메모·다음 주 모니터링 포인트로 구성된 1쪽 마크다운. 제약: 사내 운임표·계약·화주 정보 등 비공개 데이터는 사용·업로드하지 않고 공개 출처만 사용, 모든 수치·이슈에 출처 표기, 출처 불명확 항목은 [검증필요] 표시. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `scm-sensing-report`
- 트리거: "센싱 돌려줘", "주간 물류 리포트 만들어줘", "이슈 센싱해줘" (기간 미명시 시 최근 7일)
- 워크플로우 핵심: (1) `센싱_키워드.md`에서 키워드·무역로 읽기(없으면 기본 축) → (2) 키워드별 공개 웹 수집 → (3) 공급망 영향도(상/중/하) 분류 → (4) 사실/추정 분리·1차 출처 URL 확인([검증필요]) → (5) 운임·환율 메모 → (6) 1쪽 마크다운 작성, `센싱리포트_{날짜}.md` 저장
- 출력 포맷: 핵심 요약 3줄 + 이슈 표(제목·요약·영향도·출처, 영향도순) + 운임·환율 메모 + 다음 주 모니터링 포인트
