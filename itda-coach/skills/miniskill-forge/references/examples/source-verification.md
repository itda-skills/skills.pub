# 예시: 감사·진단 — 산출물 수치·고유명사 1차 출처 검증

> miniskill-forge 도출 시 직군 근접 예시로 참조하는 일반화본(참가자 고유정보 제거).

## 적합 상황
- 누가: 감사·진단 담당자
- 반복 작업: 보고서·메모를 제출하기 전, 그 안의 수치·회사명·법령 조항·핵심 주장을 1차 출처로 대조하고 hedge 표현을 걷어내는 검증
- 굳히면 좋아지는 것: 일관성(작성자·날짜마다 흔들리던 꼼꼼함을 고정) + 신뢰도(수치 하나·고유명사 하나가 틀려도 무너지는 보고 품질 방어)

## 워크스페이스에 둘 자료 (있는 것만)
- 검증할 보고서·메모 초안(마크다운·텍스트). 긴 글은 파일로
- 자주 보는 1차 출처 목록(DART·law.go.kr·KOSIS·담당 부처 공식 사이트 URL)
- 과거 검증 결과 양식 1부(출처 대조표 형식 참고용). 공개 출처만 두고 비공개 자료는 넣지 않음

## 도출 프롬프트 (Cowork 채팅에 붙여넣기)
> "나는 감사·진단 담당자입니다. 보고서·메모를 제출하기 전 그 안의 수치·고유명사·인용을 1차 출처로 검증하는 작업을 매번 같은 형식으로 하고 싶습니다. 입력: 검증할 초안(채팅에 붙여넣거나 워크스페이스 폴더의 파일). 처리: (1) 초안에서 숫자·고유명사(회사명·금액·날짜·비율·법령 조항)·핵심 주장을 모두 추출, (2) 각 항목의 1차 출처(공시 원문·법령 원문·통계 원본·기관 공식 발표)를 공개 웹에서 찾아 대조, (3) 블로그·뉴스·요약본은 1차로 인정하지 말고 그것이 인용한 원본을 직접 확인, (4) 확인된 항목은 '사실 한 줄 / 출처 URL / 확인 날짜' 기록, 확인 안 되는 항목은 '출처 미확인', (5) '보통·대체로·~할 수 있습니다·~로 알려져 있습니다' 같은 hedge 표현을 적발해 1차 출처 인용으로 바꾸거나 뺌. 출력: 항목별 검증 대조표(항목/초안 내용/1차 출처 URL/확인 날짜/판정[확인·불일치·출처 미확인]) + 맨 아래 '추가 확인 필요' 목록. 제약: 사내 비공개 데이터는 사용·업로드하지 않고 공개 1차 출처만 사용, 추측으로 빈칸을 메우지 않음, 단정 대신 확인/미확인 구분. 최종 판단은 담당자·감사위원회가 함을 명시. 이 흐름을 SKILL.md 1개로 정리해줘 — 트리거·입력 가정·워크플로우·출력 포맷 섹션 포함."

## 도출될 SKILL.md 골격
- name: `source-verify`
- 트리거: "출처 검증해줘", "1차 소스 대조해줘", "이 보고서 출처 확인해줘"
- 워크플로우 핵심: (1) 초안에서 숫자·고유명사·핵심 주장 추출 → (2) 각 항목을 1차 출처(공시·법령·통계 원문)로 공개 웹 대조 → (3) 블로그·뉴스·요약본은 인용 원본까지 직접 확인 → (4) 확인=사실 한 줄/URL/확인 날짜, 불가=출처 미확인 → (5) hedge 표현 적발·1차 출처 인용으로 교체/삭제 → 대조표 작성, `출처검증_{날짜}.md` 저장
- 출력 포맷: 검증 대조표(항목·초안 내용·1차 출처 URL·확인 날짜·판정[확인/불일치/출처 미확인]) + '추가 확인 필요' 목록. "최종 판단은 담당자·감사위원회" 명시
