# itda-coach

> **🟢 처음이세요? → [GUIDE.md](GUIDE.md) 한 장만 보세요.** 코드 몰라도 됩니다. 채팅창에 평소 말로 고민을 적으면 코치가 안내해요.

비개발자가 막연한 업무 불편을 **손에 잡히는 한 조각**으로 바꾸도록 옆에서 이끄는 코칭(facilitation) 스킬팩입니다. "막상 뭘 해볼지 막막하다"를 "오늘 1시간 안에 이걸 만든다"로 단계별로 좁혀 주는 도구 4개를 담았습니다.

## 스킬

| 스킬 | 막막함의 종류 | 하는 일 |
|------|--------------|---------|
| `problem-guide` (문제 구체화) | "원하는 대로 안 나와요" / 막연한 문제 한 줄 | 역질문 5축으로 문제를 구체화하고 Cowork/Code 두 갈래 해결 가이드를 만든다 |
| `analysis-guide` (데이터 분석 길잡이) | 데이터는 있는데 첫 발을 못 떼겠다 | 데이터를 앞에 두고 한 걸음씩 같이 분석하며 '다음엔 혼자 시작하는 법'을 남긴다 |
| `hour-slice` (1시간 조각 자르기) | 하고 싶은 건 큰데 1시간 안엔 뭘? | 구체화된 문제를 **오늘 ~1시간 안에 눈에 보이는 결과가 나오는 한 조각**으로 자른다 |
| `miniskill-forge` (미니스킬 만들기) | 이걸 매번 자동으로 굳히고 싶다 | 직군의 가장 반복적인 작업을 재사용 미니스킬로 Cowork에서 도출한다 |

## funnel

```
막연한 불편 ─▶ problem-guide(구체화) ─▶ hour-slice(1시간 조각)
                                          ├─▶ analysis-guide(데이터 라이브 실행)
                                          └─▶ miniskill-forge(반복이면 미니스킬로 굳히기)
```

각 스킬은 독립 호출도 되지만, 위 순서로 이어 쓰면 "막막함 → 오늘 만들 한 조각"이 가장 매끄럽다.

## 설계 원칙

- **떠먹이되 숟가락 쥐는 법을 남긴다** — 대신 끝내주는 게 아니라, 다음엔 혼자 시작하도록 *시작하는 법*을 남긴다.
- **추천보다 선택지** — 메타 결정(트랙·길 선택)은 탭 가능한 보기로, 맥락 추출(후보·입력·기준)은 사용자 본인 언어로 받는다.
- **1시간 실현가능성** — 손에 있는 표면(Cowork 단독·첨부파일·설치된 스킬·커넥터)만으로 가시적 결과가 나오는 크기로 자른다.
- **Cowork 친화** — 비개발자·모바일 입문자를 1순위로 가정한다. 사내 데이터는 비식별·표본 우선, 산출물은 항상 초안.
