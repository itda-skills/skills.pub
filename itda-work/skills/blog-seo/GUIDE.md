## 빠른 시작

네이버 API로 블루키워드를 찾는 가장 빠른 방법입니다.

```
파이썬 독학 블루키워드 찾아줘
```

```
경쟁 적은 키워드 분석해줘
```

```
블로그 키워드 포화지수 확인해줘
```

이 한 줄이면 스킬이 시드 키워드를 확장하고, 검색량·문서수를 조회해 S~D 5단계 등급이 매겨진 리포트를 돌려줍니다.

## 활용 시나리오

### 블로그 주제 추천

시드 키워드 몇 개로 관련 키워드를 확장해 블로그 글감을 추려냅니다.

```
"파이썬 독학"으로 쓸 만한 블로그 주제 추천해줘
```

### 블루키워드 선별

`--min-grade A` 로 올려 S~A등급만 필터링하면 경쟁이 적고 검색량이 보장된 키워드만 남습니다.

```
파이썬 관련 블루키워드(A등급 이상)만 골라줘
```

### 트렌드 분석

`--trend` 옵션을 켜면 네이버 데이터랩으로 계절성·상승세를 함께 확인합니다.

```
파이썬 독학 키워드의 최근 트렌드까지 확인해줘
```

> ⚠️ `--trend`는 키워드 1개당 데이터랩 API 1회를 소모합니다. 일일 한도 1,000회를 빠르게 소진하므로 필터링된 키워드에만 적용하세요.

## 출력 옵션

| 옵션 | 설명 | 사용 시점 |
|------|------|-----------|
| `--format md` | 표 형식의 마크다운 리포트 (기본값) | 문서·노션에 붙여넣기 |
| `--format json` | 구조화된 JSON | 다른 스크립트와 파이프라인 연동 |
| `--format csv --output result.csv` | CSV 파일 저장 | 스프레드시트 대량 분석 |

## 팁

- **두 단계 실행 패턴**: 1차로 `--trend` 없이 넓게 탐색 → 추린 키워드만 `--keywords`로 다시 넣어 `--trend` 재실행. 데이터랩 한도를 크게 절약합니다
- **`--min-grade A` 로 블루키워드만**: 기본값 B 대신 A로 올리면 유망 키워드 이상만 걸러집니다
- **캐시 활용**: 1시간 이내 동일 키워드를 같은 CWD에서 재실행하면 블로그 문서수 API를 다시 호출하지 않습니다 (`.itda-skills/blog-seo-cache.json`)
- **⚠️ `--trend` 남용 금지**: 많은 시드 + `--trend` 조합은 하루 1,000회 한도를 한 번에 소진할 수 있습니다. 1차 필터링 후에만 사용하세요
- **⚠️ `--keywords` 최대 5개**: 5개 초과 지정 시 잘림 또는 오류가 발생합니다. 쉼표로 분리
- **⚠️ API 키 5개 전원 필수**: `NAVER_SEARCHAD_*` 3개 + `NAVER_CLIENT_*` 2개를 모두 설정해야 지표 계산이 가능합니다. 검색광고 API는 `ads.naver.com` 광고주 계정이 있어야 발급됩니다
