# pptx-design 사용 가이드

콘텐츠와 데이터만 주면, **디자인이 입혀진 16:9 PPTX 발표자료**를 macOS/Linux·헤드리스·Office 없이 새로 만들어 주는 스킬입니다. 차트는 PowerPoint 에서 편집 가능한 네이티브 객체로, 한글은 또렷한 굵은 고딕으로, 그라디언트·모티프 배경까지 한 번에 처리합니다.

이 가이드는 **(1) 다양한 디자인을 어떻게 골라 쓰는지**, **(2) 자사만의 브랜드 디자인을 어떻게 입히는지**, **(3) 무엇을 주의해야 하는지**를 다룹니다.

---

## 1. 빠른 시작 — 이렇게 말하면 됩니다

```
"이 내용으로 발표자료 만들어줘"                         ← 주제에 맞는 내장 디자인 자동 선택
"삼성전자 주가전망 PPT, stripe 느낌으로 11장"           ← 디자인 톤 지정
"우리 회사 브랜드 컬러(#1A73E8)로 IR 덱 5장"            ← 자사 컬러 지정
"이 DESIGN.md 적용해서 슬라이드 만들어줘 (URL/파일)"     ← 웹 DESIGN.md 토큰 적용
"다크 톤 + 네온, e스포츠 시장 보고서"                    ← 팔레트·분위기 지정
```

필요한 입력은 3가지(뒤 2개는 선택):

| 입력 | 필수 | 설명 |
|---|---|---|
| **콘텐츠** | ✅ | 슬라이드 아웃라인(제목/본문/시각요소). 모호하면 한 번 확인합니다. |
| **수치 데이터** | 차트·표 있으면 | JSON·표. 모든 차트/표 수치의 단일 출처(손입력 금지, 1:1 인용). |
| **디자인 지정** | 선택 | DESIGN.md(파일/URL) 또는 "stripe 느낌"·"우리 브랜드 컬러"·"다크 미니멀" 같은 한마디. 없으면 주제에 맞는 내장 팔레트 자동 선택. |

## 2. 다양한 디자인 활용법

### 2.1 "디자인을 어떻게 정하지?" — 세 가지 길

1. **그냥 맡기기** — 디자인 언급 없이 주제만 주면, 주제·톤에 맞는 **내장 팔레트**를 골라 적용합니다("데이터 리포트 → 따뜻한 에디토리얼", "핀테크 → 인디고 그라디언트" 식).
2. **톤·분위기 한마디** — "다크 럭셔리", "프린트 신문 감각", "파스텔 친근하게", "네온 다크", "미니멀 거대 타이포"처럼 말하면 그에 맞는 팔레트·모티프를 구성합니다.
3. **레퍼런스 DESIGN.md** — awesome-design-md 류 웹 디자인 시스템 문서를 주면 색·타이포·모티프 토큰을 해석해 반영합니다(웹 전용 토큰은 자동 필터 — §4 주의점 참고).

## 3. 자사만의 커스텀 디자인 작성법

회사 브랜드를 그대로 입히는 방법입니다. **핵심은 "토큰을 정하고 → 그 hex/폰트를 실제 도형·차트·텍스트에 1:1로 박는다"** 입니다.

### 3.1 가장 쉬운 길 — 브랜드 정보를 말로 주기

```
"우리 브랜드 컬러는 #1A73E8(파랑)·#202124(잉크)·#FBBC04(옐로 액센트)이고,
 배경은 화이트, 폰트 느낌은 모던 산세. 이 톤으로 4장 만들어줘."
```

이 정도면 충분합니다. 색은 도형·차트·강조 텍스트에 그대로 반영되고, 한글은 안전 고딕으로 자동 처리됩니다.

### 3.2 제대로 정의하기 — 브랜드 토큰 6종

| 토큰 | 무엇 | 예 |
|---|---|---|
| **canvas / surface** | 배경·카드 색 | `#FFFFFF` / `#F6F8FC` |
| **ink** | 본문 글자색(보통 순수 검정 대신 짙은 네이비) | `#0D253D` |
| **primary** | 지배 브랜드 색(시각 비중 60–70%) | `#1A73E8` |
| **accent / semantic** | 액센트 1개 + 의미색(상승/하락) | `#FBBC04` / 상승 `#1E8E3E` 하락 `#D93025` |
| **폰트** | 영문 디스플레이(라틴 전용) + 한글은 안전 고딕 고정 | "Helvetica Neue" / Noto Sans KR |
| **모티프** | 반복 시각 요소 1개 | 좌측 액센트 바 · 둥근 카드 · 그라디언트 띠 |

> **지배(dominance) 규칙**: 한 색이 60–70%를 지배하고 보조 1–2 + 날카로운 액센트 1. 모든 색에 같은 무게를 주면 "AI 슬라이드" 티가 납니다.

### 3.3 적용 흐름(스킬이 내부에서 하는 일)

1. **팔레트 박기** — 배경 `set_bg`, 도형 `rect(fill=브랜드hex)`, 강조 텍스트 색, 차트 `palette=[브랜드hex…]` 에 hex 를 1:1 사용(차트 기본 파랑/주황은 추방).
2. **한글 안전** — 한글 텍스트는 `kr_font_name()`(Noto Sans KR·Pretendard 우선) 으로 고정. 자사 폰트가 라틴 디스플레이(세리프·thin)여도 **영문/숫자 run 에만** 적용하고, 한글은 안전 고딕으로 자동 강제됩니다(가드).
3. **모티프 반복** — 정한 모티프 하나를 표지~결론 전 슬라이드에 반복.
4. **차트는 네이티브 편집 객체** — 받는 사람이 PowerPoint 에서 수치를 직접 고칠 수 있게(고디자인 특수 차트만 이미지로).
5. **검증** — HARD GATE(경계이탈·빈슬라이드·토큰누락 0) + 한글 타이포 advisory + 렌더 이미지 육안 QA.

### 3.4 사내 표준 디자인을 재사용하려면 — DESIGN.md

자주 쓰는 브랜드 룩이면 **DESIGN.md** 한 장으로 박제해 두고 매번 가리키면 됩니다(awesome-design-md 포맷):

```markdown
---
colors: { primary: "#1A73E8", ink: "#0D253D", canvas: "#FFFFFF",
          surface: "#F6F8FC", accent: "#FBBC04", up: "#1E8E3E", down: "#D93025" }
typography: { display: "Helvetica Neue", body: "system-sans" }
rounded: 12
motif: "좌측 4px 액센트 바 + 둥근 카드"
do:   ["여백으로 위계", "의미색 절제", "표지·결론 다크"]
dont: ["제목 밑 액센트 라인", "블루 디폴트", "모든 색 균등"]
---
```

→ "사내 DESIGN.md 로 만들어줘"라고 하면 토큰을 그대로 해석해 반영합니다. 자세한 토큰 매핑은 `references/design-md-mapping.md`(3열 필터 표).

## 4. 주의점 (꼭 알아두기)

### 4.1 ★픽셀(인치) 단위 위치 조정의 어려움 — 가장 큰 마찰

웹의 CSS flexbox/그리드 같은 **자동 정렬이 없습니다.** 모든 요소를 **인치 절대 좌표**(예: `x=0.85, y=2.3, w=6.0`)로 직접 놓습니다. 그래서:

- **제목이 2줄로 래핑되면 아래 요소가 자동으로 안 밀립니다** → 좌표를 수동으로 내려야 합니다.
- 칸 너비를 좁게 잡으면 텍스트가 과도하게 줄바꿈되거나 박스 밖으로 넘칠 수 있습니다.
- "딱 8px 간격" 같은 픽셀 정밀 정렬은 인치 환산으로 근사합니다(둥근 모서리 radius 도 px 가 아니라 0~0.5 비율).

**그래서 이렇게 보완합니다(스킬 내장):**

1. **render → 이미지로 직접 확인 → 좌표 수정** 루프를 돕니다("첫 렌더는 거의 틀리다"를 전제로 버그 헌트).
2. **자동 검증 게이트**가 경계이탈(슬라이드 밖)·요소 겹침·빈 슬라이드·필수 텍스트 누락을 잡아 줍니다.
3. 텍스트 박스는 기본 `margin=0`, 도형은 기본 테두리·그림자 제거 — 정렬 어긋남의 흔한 원인을 미리 차단합니다.

→ **실무 팁**: "표지 제목이 한 줄/두 줄 어느 쪽일지" 같은 미세 레이아웃은 한 번에 완벽하기 어렵습니다. **여백을 넉넉히** 잡고, 렌더 이미지를 함께 보며 1~2회 조정하는 흐름이 정상입니다. 정밀 픽셀 레이아웃이 절대적으로 중요하면 마지막에 PowerPoint 에서 손보는 것이 가장 빠릅니다.

### 4.2 LibreOffice 프리뷰 ↔ PowerPoint 실물은 다를 수 있음

검증·미리보기는 LibreOffice 헤드리스 렌더 기준입니다. **PowerPoint 실물 렌더(특히 한글 폰트·자간)는 다소 다를 수 있으니**, 중요 산출물은 PowerPoint 에서 한 번 열어 확인하길 권합니다.

### 4.3 그라디언트·글로우는 래스터(이미지)로 구워집니다

python-pptx 가 그라디언트 fill 을 직접 지원하지 않아, 그라디언트·글로우·메시 배경은 **고해상도 PNG 로 베이크**해 깝니다. 벡터가 아니므로 크게 확대하면 픽셀화하고, **텍스트 위 그라디언트(글자 자체에 그라디언트)는 불가**합니다.

### 4.4 시스템 한글 폰트에 의존

한글은 시스템에 설치된 폰트로 렌더됩니다. 또렷한 굵은 고딕을 위해 **Noto Sans KR 또는 Pretendard** 설치를 권장합니다(없으면 Apple SD Gothic Neo·Malgun Gothic·Nanum 순). 한글 세리프는 시스템에 거의 없어, 세리프가 브랜드여도 한글은 고딕으로 표현됩니다.

### 4.5 이 스킬의 비목표 (다른 경로로)

- **기존 회사 템플릿/브랜드 마스터 편집·병합** → python-pptx 라운드트립은 lossy. hyve PowerPoint **COM 도메인**(`office_edit`, Windows).
- **Visible 실시간 시연(RPA)·발표자 노트·SmartArt·전환/애니메이션** → COM 도메인.
- 본 스킬은 **macOS/Linux·헤드리스·배치 신규 생성** 전용 보완재입니다.

## 5. 산출물 체크리스트

발표자료를 받을 때 함께 확인하면 좋은 것:

- [ ] **HARD GATE PASS** (경계이탈·퇴화도형·빈슬라이드·필수토큰 = 0)
- [ ] **한글 타이포 advisory** (음수/과대 자간·비안전 폰트 = 0 권장)
- [ ] **렌더 미리보기 이미지**로 한글 가독성·겹침·정렬 육안 확인 1회
- [ ] **차트 수치가 입력 데이터와 1:1 일치**(음수도 정확히)
- [ ] 적용한 **팔레트 hex·모티프**와 그 반영 위치
- [ ] 재현 한계에 걸린 토큰(독점 서체·그라디언트 등)과 대체 처리
