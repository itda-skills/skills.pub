# 기본 PPTX 디자인 가이드 (Anthropic `pptx` 스킬 흡수 · superset 토대)

`pptx-design` 은 Anthropic 기본 `pptx` 스킬의 **superset** 이다(SPEC-PPTX-DESIGN-002). 이 문서는 기본 스킬의 디자인 가이드(`SKILL.md` §Design Ideas / §QA)에 담긴 **디자인 원칙과 기능적 데이터**(색 팔레트 hex·폰트 페어링·레이아웃·QA 루프)를 **한국어로 번안·요약**해, DESIGN.md 가 없을 때의 **1급 디자인 경로**로 삼는 **스킬 내부 참조 문서**다. 한글 안전 가드·deckkit 매핑·네이티브 차트 매핑은 본 스킬이 보강한 부분이다.

> **귀속(attribution)**: 디자인 원칙(예: "every slide needs a visual"·명암 대비·모티프 커밋·QA 버그헌트)과 기능적 데이터(색 hex 값·헤더/본문 폰트 페어링)는 Anthropic agent-skills `skills/pptx/SKILL.md`(라이선스: **Proprietary**, 해당 저장소 `LICENSE.txt`)에 공개된 가이드에서 가져왔다. 본 문서는 원문 표현을 그대로 복제·재배포하는 산출물이 아니라, 공개된 원칙·사실 데이터를 한국어로 번안한 **내부 작업 참조**다. 원 가이드의 라이선스·전체 문맥은 출처를 직접 확인하라.

> **superset 원칙**: 기본 스킬로 만들 수 있는 모든 덱은 pptx-design 으로도 (최소 동등 품질로) 만들 수 있어야 한다. 이 가이드(색·타이포·레이아웃·QA)는 그 **공통 토대**이고, 그 위에 DESIGN.md 적용·3층+타이포 검증·matplotlib 고디자인 차트가 **추가 옵션**으로 얹힌다.

DESIGN.md 가 **있으면** `design-md-mapping.md` 의 토큰 매핑이 우선한다. DESIGN.md 가 **없으면** 이 문서의 팔레트·타이포·레이아웃 원칙 + `design-md-mapping.md` §3 내장 추천 팔레트를 1급으로 쓴다.

## 0. 시작 전 — 지루한 슬라이드를 만들지 마라

흰 배경 위 평범한 불릿은 아무도 감동시키지 못한다. 각 슬라이드마다 아래를 고민한다.

- **주제에 맞는 과감한 팔레트 선택** — "이 색을 전혀 다른 발표에 그대로 옮겨도 말이 되면, 충분히 구체적인 선택이 아니다." 색이 이 주제를 위해 디자인된 느낌이어야 한다.
- **균등이 아니라 지배(dominance)** — 한 색이 60~70% 시각 비중을 지배하고, 1~2개 보조 톤 + 날카로운 액센트 1개. 모든 색에 동등한 무게를 주지 않는다.
- **명암 대비(dark/light)** — 표지·결론은 다크 배경, 본문은 라이트("샌드위치" 구조). 또는 프리미엄 느낌을 위해 전체 다크로 커밋.
- **시각 모티프 1개에 커밋** — 둥근 이미지 프레임·색 원 안의 아이콘·한쪽만 두꺼운 보더 같은 **하나의 요소를 골라 모든 슬라이드에 반복**.

## 2. 슬라이드마다 시각 요소

**모든 슬라이드는 시각 요소(이미지·차트·아이콘·도형) 하나가 필요하다.** 텍스트만 있는 슬라이드는 잊혀진다.

**레이아웃 옵션:**
- 2단(텍스트 좌 / 일러스트·차트 우)
- 아이콘 + 텍스트 행(색 원 안 아이콘 · 굵은 헤더 · 아래 설명)
- 2×2 / 2×3 그리드(한쪽 이미지, 다른 쪽 콘텐츠 블록 그리드)
- 하프블리드 이미지(좌/우 전체)에 콘텐츠 오버레이

**데이터 표현:**
- 큰 스탯 콜아웃(60~72pt 큰 숫자 + 아래 작은 라벨)
- 비교 컬럼(before/after · pros/cons · 나란히 옵션)
- 타임라인·프로세스 플로우(번호 단계 · 화살표)

**시각 폴리시:**
- 섹션 헤더 옆 작은 색 원 안 아이콘
- 핵심 스탯·태그라인에 이탤릭 액센트 텍스트

## 3. 타이포그래피

**개성 있는 폰트 페어링을 고른다** — Arial 로 도망가지 않는다. 개성 있는 헤더 폰트 + 깔끔한 본문 폰트.

| Header Font | Body Font |
|-------------|-----------|
| Georgia | Calibri |
| Arial Black | Arial |
| Cambria | Calibri |
| Trebuchet MS | Calibri |
| Impact | Arial |
| Palatino | Garamond |
| Consolas | Calibri |

| Element | Size |
|---------|------|
| Slide title | 36–44pt bold |
| Section header | 20–24pt bold |
| Body text | 14–16pt |
| Captions | 10–12pt muted |

> ⚠️ **한글 보강(본 스킬 필수)**: 위 헤더 폰트 페어링(Georgia·Impact·Palatino 등)은 **라틴(영문/숫자) run 전용**이다. 한글 run 에 적용하면 LibreOffice 가 세리프/명조/붓글씨로 폴백한다. 한글 run 은 **항상** `dk.kr_font_name()`(안전 고딕) 을 쓴다 — `dk.set_run_font` 의 한글 가드가 비-한글 폰트를 자동으로 안전 고딕으로 강제하고 음수 자간을 0 으로 클램프한다. 상세는 `design-md-mapping.md` §타이포 필터.

## 4. 여백(Spacing)

- 최소 마진 0.5"(본 스킬 권장 0.4" 이상)
- 콘텐츠 블록 간 0.3~0.5"
- 숨 쉴 공간을 남긴다 — 모든 인치를 채우지 않는다.

## 5. 피해야 할 것 (AI 슬라이드 티)

- **같은 레이아웃 반복 금지** — 컬럼·카드·콜아웃을 슬라이드마다 변주.
- **본문 가운데 정렬 금지** — 문단·리스트는 좌측 정렬, 제목만 가운데.
- **크기 대비 인색 금지** — 제목 36pt+ 로 본문 14~16pt 와 확실히 구분.
- **블루 디폴트 금지** — 주제를 반영한 색.
- **여백 무작위 혼용 금지** — 0.3" 또는 0.5" 중 하나로 일관.
- **한 슬라이드만 꾸미고 나머지 방치 금지** — 전부 커밋하거나 전부 단순하게.
- **텍스트 전용 슬라이드 금지** — 이미지·아이콘·차트·도형을 더한다.
- **텍스트 박스 패딩 망각 금지** — 선·도형을 텍스트 가장자리에 맞출 땐 `margin=0`(deckkit `add_text` 는 기본 margin=0).
- **저대비 요소 금지** — 밝은 배경 위 밝은 텍스트, 다크 위 다크 아이콘 금지.
- **★제목 밑 액센트 라인 절대 금지(NEVER)** — AI 생성 슬라이드의 전형. 여백·배경색으로 위계를 표현.

## 6. QA (필수) — "문제가 있다고 가정하라"

**첫 렌더는 거의 항상 틀리다.** QA 를 확인 단계가 아니라 **버그 헌트**로 접근한다. 첫 검사에서 문제를 0개 찾았다면 충분히 보지 않은 것이다.

**콘텐츠 QA**: 누락·오타·순서 오류 확인(텍스트 추출 대조). 본 스킬은 `verify.py` 의 `--tokens` 로 자동화.

**시각 QA**: 슬라이드를 이미지로 변환해 직접 본다. 본 스킬은 `render.py` → 슬라이드 JPG 를 **직접 Read**. 아래를 찾는다:
- 겹침(텍스트가 도형 위로, 선이 글자 위로, 요소 적층)
- 가장자리·박스 경계 오버플로/잘림
- 단일 행 기준 장식선인데 제목이 2줄로 래핑된 경우
- 출처·푸터가 위 콘텐츠와 충돌
- 요소 간 < 0.3" 근접, 불균등 간격
- 가장자리 마진 < 0.5"
- 컬럼 정렬 불일치
- 저대비 텍스트·아이콘
- 너무 좁은 박스로 과도 래핑
- 남은 플레이스홀더

**검증 루프**: 생성 → 이미지 변환 → 검사 → 문제 목록화(0개면 더 비판적으로 재검) → 수정 → **영향 슬라이드 재검**(한 수정이 다른 문제를 부른다) → 새 문제 없을 때까지 반복. **최소 1회 수정-재검 사이클 전에는 성공 선언 금지.**

> 본 스킬은 위 QA 규율을 **관문4(★검증 게이트)** 로 제도화한다: `verify.py` 의 지오메트리/콘텐츠/OCR/타이포 자동 검사 + 렌더 이미지 시각 QA. HARD GATE PASS + 최소 1회 수정-재검이 산출 조건이다.
