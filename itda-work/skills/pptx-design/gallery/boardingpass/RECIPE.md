**한 줄 콘셉트** — ★레이아웃 아키타입: **보딩패스/티켓**. 헤더 바 + 모노스페이스 필드 + **천공(점선)** + **펀치 노치** + **바코드 스텁**의 스큐어모픽 구조 문서로 조판하고, 데이터를 탑승권 필드(PASSENGERS·REVENUE·LOAD 등)에 매핑한다. 보딩패스라는 아티팩트가 항공이라는 주제 그 자체와 맞물린다. 어떤 기존 아키타입과도 다른 구조-문서 메타포. 14번째 구성, 전용 헬퍼 신작.

## ★레이아웃 아키타입: 보딩패스 (전용 헬퍼)
- `ticket(x, y, w, h, headline, fields, stub_label, stub_val)` — **보딩패스 1장 통합 빌더**: 오렌지 헤더 바("✈ GLOBAL AVIATION · BOARDING PASS" + AV 2026) + 헤드라인 + 모노 필드 행(라벨↑값↓) + 천공 + 스텁(라벨·값·바코드).
- `perf(px, y0, y1)` — **천공 점선**(작은 사각 반복) + **펀치 노치**(배경색 원이 티켓 상·하 모서리를 파먹는 효과).
- `barcode(x, y, w, h)` — **고정 패턴**(BARPAT) 막대로 재현 가능한 바코드(Math.random 미사용 → byte 재현).
- 필드: 모노 대문자 라벨 + 굵은 모노 값. **신규 헬퍼 — 기존 아키타입 헬퍼 미재사용.**

## 레이아웃 레시피 (6 슬라이드)
1. **표지** — 타이틀 + **히어로 탑승권**(2025→2026 라우트 + PASSENGERS/REVENUE/LOAD 필드 + 스텁 PASSENGER WORLD + 바코드).
2. **여객** — 탑승권 밴드(PASSENGERS 5.2B·YOY +4.4%·RPK +5.3%·LOAD 83.8%) + 노트.
3. **수익·마진** — 탑승권 밴드(REVENUE $1.053T·NET PROFIT $41B·MARGIN 3.9%·PER PAX ~$7.9) + "샌드위치 한 개 값" 노트.
4. **지역·탑승률** — 탑승권 밴드(APAC RPK 34.5%·APAC INTL +10.9%·APAC LOAD 85.8%·WORLD LOAD 83.8%) + 노트.
5. **인사이트** — 스텁 카드 3(액센트 상단 바 + 본문 + 바코드).
6. **클로징** — 스테이트먼트 + 바코드 스트립 + 콜로폰.
