---

웹사이트를 사람 대신 조작해 데이터를 모아주는 스킬입니다 — 로그인, 폼 입력, 클릭 탐색, 무한스크롤 수집, 차단 사이트 우회까지.
**Claude에게 말로 부탁하면 됩니다** — "이 사이트 로그인해서 데이터 모아줘", "폼 채워서 검색해줘"처럼요.

---

## 빠른 시작

```
이 사이트 로그인해서 데이터 모아줘
폼 채워서 검색해줘
무한스크롤 전부 수집해줘
```

Claude가 작업 유형을 판별해 알맞은 자동화 방식(단발 읽기 / 화면 조작 / 데이터 추출 / 대량 수집 / 로그인·차단 사이트 / 웹메일)을 선택합니다.

---

## 사전 준비

- **hyve 앱 연결이 필요합니다.** 데스크톱에서 hyve 트레이 앱을 켜고, **hyve 설정 > MCP 탭**에서 **웹(web) 프리셋**을 사용 중인 클라이언트(Claude Desktop/Cowork 등)에 등록하세요.
- **최초 사용 시 동의**: 처음 자동화를 실행하면 자동화 책임에 대한 동의를 확인합니다. 일부 사이트는 사이트별 동의를 추가로 요구합니다(24시간 유지).
- **브라우저 로그인 상태는 재사용됩니다**: 공통 브라우저 프로필 하나를 쓰기 때문에, 한 번 로그인해 두면 이후 작업에서 그 로그인 상태가 그대로 활용됩니다.

---

## 활용 시나리오

### 페이지 하나 빠르게 읽기

JavaScript로 그려지는 페이지도 렌더링해서 본문을 읽어줍니다.

```
이 페이지 본문 읽어서 요약해줘
```

### 폼 입력·클릭이 필요한 화면 조작

검색 폼을 채우고, 버튼을 누르고, 새 창·탭 전환까지 여러 단계를 이어서 진행합니다.

```
이 사이트에서 상품명으로 검색하고 결과 첫 페이지를 정리해줘
```

### 화면 속 데이터 뽑기

목록·표 데이터를 구조화해서 추출합니다. 최신 웹앱(SPA)은 화면에 보이는 값 대신 **서버 응답 원본**을 캡처해 가공되지 않은 전체 필드를 확보합니다 — 화면 파싱보다 훨씬 정확합니다.

```
이 목록에서 이름·가격·평점을 표로 뽑아줘
```

### 목록·무한스크롤 전부 수집

스크롤할 때마다 로드되는 항목을 끝까지 모아 **파일로 저장**합니다. 수백 건이 넘는 대량 데이터도 대화 창을 채우지 않고 처리됩니다.

```
이 게시판 글 목록 전부 수집해서 파일로 저장해줘
```

### 로그인·차단 사이트

세 가지 방식을 지원합니다. ① 화면에 뜬 브라우저 창에서 **직접 로그인한 뒤 넘겨주기**, ② 봇 차단이 강한 사이트는 화면에 뜬 창에서 **직접 로그인·잠깐 사용해 세션을 데워 둔 뒤** 이어서 조회하기, ③ 비밀번호를 Claude에게 보여주지 않고 hyve가 대신 입력하는 **비밀 주입** 방식.

```
이 사이트는 내가 로그인할게. 창 띄워줘 — 끝나면 이어서 수집해줘
```

### 웹 전용 메일 확인

IMAP이 막힌 사내 그룹웨어·공제회 웹메일의 목록·본문을 확인합니다. 일반 메일(네이버·Gmail 등)은 email 스킬이 우선입니다.

```
그룹웨어 웹메일 안 읽은 메일 목록 보여줘
```

### 저장된 로그인·쿠키 정리

여러 계정을 격리해 쓰다 보면 프로필이 쌓입니다. 어떤 프로필이 저장돼 있는지 확인하고, 필요 없는 로그인·쿠키를 **완전히 지울** 수 있습니다. 이건 세션만 닫는 것이 아니라 **실제 데이터 삭제**라서, 지운 프로필로 다시 접속하면 로그인이 사라져 처음부터 다시 로그인해야 합니다(되돌릴 수 없습니다).

```
저장된 브라우저 프로필 목록 보여줘
이제 안 쓰는 이 계정 로그인 완전히 지워줘
```

- 지금 그 프로필로 작업이 진행 중이면 삭제가 거부됩니다(사용 중, `profile_in_use`) — 먼저 작업을 끝내세요.
- 기본 프로필(`default`)은 로그인 재사용의 공용 저장소라 삭제 대상이 아닙니다.
- macOS는 14 이상에서만 프로필 목록·삭제를 지원합니다(그 미만은 지원하지 않는다고 알려드립니다).

## 출력 옵션

| 옵션 | 설명 | 사용 시점 |
|------|------|-----------|
| 본문 텍스트 | 페이지 본문만 간결하게 | 읽기·요약 목적 |
| 구조화 목록 | 이름·가격 등 원하는 필드를 표로 | 목록·표 데이터 추출 |
| 파일 저장 | 수집 결과를 서버 파일(JSON)로 저장 | 수백 건 이상 대량 수집 |
| 요약만 받기 | 개수·집계만 대화로, 원본은 파일로 | 대량 데이터의 확인 목적 |

원하는 출력 방식을 자연어로 말해주면 됩니다.

---

## 팁

- **대량 수집은 "파일로 저장해줘"**: 수백 건을 대화로 받으면 느리고 비쌉니다. 파일 저장을 요청하면 개수 확인 후 필요한 부분만 꺼내 볼 수 있습니다.
- **읽기만 할 거면 web-reader가 먼저**: 정적인 페이지를 마크다운으로 읽는 용도는 web-reader 스킬이 더 가볍습니다. 거기서 실패하는 동적·차단 페이지가 이 스킬의 영역입니다.
- **웹앱 데이터는 원본 캡처가 정확**: 화면에 보이는 값은 가공·축약된 일부입니다. "서버 응답 원본으로 수집해줘"라고 하면 전체 필드를 확보합니다.
- **로그인이 애매하면 직접 하세요**: 추가 인증(OTP·보안키)이 있거나 허가가 불확실한 사이트는 창을 띄워 직접 로그인하고 넘겨주는 방식이 가장 안전합니다.

---

## 제한사항

- **hyve 필수**: 이 스킬 단독으로는 동작하지 않습니다. hyve 앱 가동 + 웹(web) 프리셋 등록이 선행돼야 합니다.
- **로그인 자동화는 조건부**: 본인 계정·사이트 허가·추가 인증 없음이 확인된 경우에만 자동 입력합니다. 불확실하면 직접 로그인 방식으로 전환합니다.
- **봇 차단이 강한 사이트는 준비가 필요**: 화면에 뜬 브라우저 창에서 직접 로그인·잠깐 사용해 세션을 데워 두는 준비가 필요할 수 있습니다(예: coupang 스킬).
- **여러 계정 병행 가능**: 계정을 분리해 지정하면 로그인이 서로 섞이지 않게 격리돼 여러 작업을 동시에 진행할 수 있습니다. 같은 로그인으로 창을 여러 개 열면 같은 세션을 공유합니다. 안 쓰는 프로필은 "저장된 로그인 지워줘"로 완전히 정리할 수 있습니다(위 시나리오 참고).
- **불가역 동작은 직전 확인**: 메일 발송·폼 제출처럼 되돌릴 수 없는 동작은 실행 직전에 내용을 확인받습니다. 웹메일은 열람만으로 '읽음' 처리될 수 있어 먼저 알려드립니다.
