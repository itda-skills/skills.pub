# ground-check 사용 가이드

LLM이 만든 비교표·보고서가 **틀린 사실로 채워지지 않도록** 1차 소스로 검증하고, 검증된 내용으로만 다시 정리해 주는 스킬입니다. 비개발자 실무자도 자연어로 부탁만 하면 자동 활성화됩니다.

## 어떤 상황에서 쓰나요?

- 윗선 보고용 **경쟁사 비교표**를 만들어야 하는데 LLM이 지어낸 숫자가 섞일까 걱정될 때
- 정부 정책·법령·고시 내용을 정리하는데 **출처 명확한 1차 소스**만 인용하고 싶을 때
- 신제품·신기능 스펙을 **공식 문서 기반으로만** 정리하고 싶을 때
- "보통 그렇다", "대체로 그렇다" 같은 **얼버무리는 표현이 들어간 보고서**가 곤란할 때
- 사용자에게 "어디서 봤어요?" 라고 물었을 때 **URL과 확인 날짜를 함께 댈 수 있어야** 할 때

## 사용 방법

자연어로 다음과 같이 요청하시면 됩니다.

```
"Claude Chat / Claude Code / Cowork 비교표 만들어줘. 1차 소스로 검증해서"
"이번 분기 OpenAI 요금제 정리해줘. 공식 자료만 써서"
"NIA 보고서랑 KISDI 자료 기반으로 AI 정책 비교표 만들어줘"
"전기차 보조금 정책 시·도별로 정리해줘. 정부 공식 사이트에서만"
```

스킬이 활성화되면 자동으로 다음 3단계를 진행합니다.

1. **조사 + 초안** — 셀마다 1차 소스 URL + 확인 날짜 + 사실 한 줄을 기록
2. **독립 검증 라운드** — 다른 1차 소스로 교차 확인 (최대 3회)
3. **실사용 예시 확장** — 각 핵심 항목별로 4줄 예시 첨부

## 산출물 형식

각 셀(표 1칸)마다 다음 정보가 포함됩니다.

```
[셀 ID] 비교대상=Claude Pro / 항목=월간 사용량

사실 한 줄 : 무료 플랜의 약 5배, 5시간 단위로 한도 갱신
출처 URL  : https://support.anthropic.com/en/articles/8324991-about-claude-pro-usage
확인 시각 : 2026-05-12
```

확인 못 한 항목은 **"미확인"** 으로 표기되고 추측으로 채워지지 않습니다.

## 비개발자 시나리오 5종

### 시나리오 1 — Claude 플랜 비교표

> "Claude Chat / Code / Cowork 차이점 비교표 만들어줘. 1차 소스로"

스킬이 anthropic.com, support.anthropic.com, docs.claude.com 만 검색해 비교표를 만듭니다. 블로그·뉴스 인용은 자동 차단되고, 확인 못 한 항목은 "미확인" 으로 남습니다.

### 시나리오 2 — 정부 정책 요약

> "전기차 충전 인프라 지원 정책 요약해줘. 환경부 공식 자료만 써서"

`*.go.kr`, `data.go.kr`, `law.go.kr` 한정 검색. 블로그·언론사 정리는 인용에 사용하지 않습니다. 정책 시점이 중요하므로 확인 날짜가 모든 항목에 함께 표기됩니다.

### 시나리오 3 — 경쟁사 제품 스펙 비교

> "삼성 갤럭시 워치7 vs 애플워치 시리즈10 스펙 비교해줘. 제조사 공식 사이트로만"

`samsung.com`, `apple.com` 공식 도메인만 검색합니다. IT 매체 리뷰·유튜브 영상은 1차 소스로 인정하지 않습니다.

### 시나리오 4 — 법령 인용 보고서

> "개인정보 보호법 제15조 내용 정리해줘. 법령 원문 인용으로"

`law.go.kr` 한정 검색. 조문 번호·시행일·최종 개정일이 함께 표기됩니다. 변호사 블로그·뉴스 해설은 인용 대상에서 자동 제외됩니다.

### 시나리오 5 — API 요금제 비교

> "OpenAI / Anthropic / Google API 요금제 비교표 만들어줘. 공식 가격 페이지 기반으로"

각 회사 공식 pricing 페이지를 직접 열어 가격을 추출합니다. "보통 X달러" 같은 hedge 표현은 자동 FAIL 처리되어 재작성됩니다.

## 본 스킬이 다루지 않는 경우

본 스킬은 **공개 웹의 1차 소스만** 다룹니다. 다음 요청은 본 스킬 범위 밖이며, 별도 안내합니다.

| 요청 예시 | 본 스킬 처리 |
|----------|--------------|
| "이 PDF 내용으로 보고서 써줘" | ❌ 마운트 파일 출처 — 별도 추출 스킬 필요 |
| "Drive에 있는 회의록 기반으로 정리해줘" | ❌ Connector 출처 (Google Drive) — 별도 안내 |
| "Notion에 적어둔 정책 문서로 비교해줘" | ❌ Connector 출처 (Notion) — 별도 안내 |
| "Gmail에 온 공지사항 요약해줘" | ❌ Connector 출처 (Gmail) — 별도 안내 |
| "회사 내부 위키 기반으로 만들어줘" | ❌ 사문서 출처 — 추출·인용 스킬 필요 |

이런 경우 "재조회 가능한 외부 권위에 대한 사실 검증"이 아니라 "재조회 불가한 사문서로부터의 사실 추출"이므로, 검증 절차가 본질적으로 다릅니다.

## 자주 묻는 질문

**Q. 검색에 시간이 너무 걸려요. 빨리 끝낼 수는 없나요?**
A. 본 스킬은 **검증 절차 자체가 핵심**입니다. 절차를 줄이면 환각·hedge 위험이 다시 들어옵니다. "그냥 빠르게 정리해줘" 요청은 본 스킬을 활성화하지 않고 사용하시는 게 맞습니다.

**Q. 영어 자료는 어떻게 처리되나요?**
A. 영어 1차 소스(예: anthropic.com, openai.com)를 열어 사실을 추출한 뒤 **한국어로 산출**합니다. 본문 인용이 필요한 경우 원문 + 번역을 함께 표기합니다.

**Q. "미확인" 항목은 어떻게 처리하나요?**
A. 미확인은 솔직한 표기입니다. 추측으로 메우지 않습니다. 본 스킬은 시도한 URL 리스트와 사유를 함께 노출해, 사용자가 직접 공식 채널로 문의할 수 있도록 합니다.

**Q. 블로그에 정리된 비교표를 참고하면 안 되나요?**
A. 블로그는 2차 인용이라서 사실이 변형됐을 가능성이 있습니다. 블로그가 인용한 1차 소스를 직접 다시 열어 확인합니다.

**Q. WebFetch가 안 되는 페이지는 어떻게 하나요?**
A. WebFetch가 실패하면 자동으로 **web-reader 스킬로 fallback** 해서 본문을 다시 가져옵니다. web-reader도 실패하면 그 URL은 포기하고 다른 1차 소스를 찾습니다.

## 한계와 주의

- **시점 명시 필수**: 요금·정책·기능 지원 여부는 변경 빈도가 높습니다. 결과의 "확인 시각" 을 반드시 함께 확인하세요. 3개월 이상 지난 결과는 재실행 권장.
- **외국어 자료**: 1차 소스가 외국어면 본문 인용은 원문 + 번역 병기. 단순 번역만으로 검증 종료하지 않습니다.
- **공식 도메인 부재**: 권위 있는 발화자(예: 정부 정책의 담당 부처)가 공식 도메인을 운영하지 않으면 "미확인" 강등이 발생할 수 있습니다.
- **3 라운드 한계**: 검증 라운드는 최대 3회입니다. 3회 후에도 불일치·출처 부족이면 그 셀은 "미확인" 강등됩니다 — 사용자가 직접 해당 기관에 문의하세요.
