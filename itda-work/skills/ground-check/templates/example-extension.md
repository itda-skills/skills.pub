# Example Extension 양식

Task 3 에서 PASS된 산출물의 **핵심 항목별로 실사용 예시 1개씩** 작성한다. 각 예시는 4줄 구조를 따른다.

## 4줄 구조 (필수)

```
- 상황       : <사용자가 처한 실무 상황. 1-2 문장>
- 입력/지시 예시: <사용자가 실제로 입력할 자연어 요청. 따옴표 포함>
- 기대 결과   : <Claude가 본 항목 기능으로 어떤 산출물을 만들어 주는가>
- 주의점     : <오해·오용 가능성, 또는 본 항목 기능이 적용되지 않는 경우>
```

## 작성 규칙

- 각 예시는 **본 산출물의 핵심 항목 중 하나에 대응** 한다 (1셀 = 1예시 권장).
- "상황"은 1차 소스로 확인된 사실을 어떻게 활용하는지 보여준다.
- "입력/지시 예시"는 실제 사용자 발화체. 키보드로 그대로 입력 가능해야 한다.
- "기대 결과"는 추상적 요약이 아니라 **구체적 산출 형식** (표 / 한 문단 / 체크리스트 등).
- "주의점"은 본 기능의 한계·전제·예외 조건. 1-2 문장.
- 4줄 외 추가 정보 금지. 단순함 유지.

## 예시 1 — Claude Code 시스템 요구사항

```
- 상황       : 회사 IT팀이 Claude Code 도입 가능 여부를 검토 중이며,
                  Windows 환경 지원 여부와 WSL 필요성을 확인하려 한다.
- 입력/지시 예시: "Claude Code 윈도우에서 쓰려면 WSL 필요한지 정리해줘"
- 기대 결과   : docs.claude.com/en/docs/claude-code/setup 1차 소스를 인용한
                  "Windows 지원 여부 + WSL 요구사항 + 출처 URL + 확인 날짜"
                  3-tuple 표.
- 주의점     : 시스템 요구사항은 정책 변경 빈도가 높다. 확인 시각을 반드시
                  사용자에게 함께 노출하고, 3개월 이상 지났으면 재확인을 권한다.
```

## 예시 2 — Claude Pro 사용량 한도

```
- 상황       : 사용자가 Claude Pro 구독 전에 무료 플랜 대비 얼마나 더 사용
                  가능한지 정확한 수치로 비교하고 싶다.
- 입력/지시 예시: "Claude Pro 한도가 무료 플랜의 몇 배인지 공식 자료로 확인해줘"
- 기대 결과   : support.anthropic.com 공식 문서를 직접 인용한 한 문단 답변.
                  "5배", "5시간 단위 갱신" 등 명시 수치 포함. 출처 URL + 확인 날짜.
- 주의점     : 사용량 정책은 분기별로 변경된다. 시점 표기가 없으면 인용 금지.
                  광고성 블로그 글의 "약 5배"는 1차 소스로 인정하지 않는다.
```

## 예시 3 — Cowork 미확인 항목 처리

```
- 상황       : 사용자가 Cowork 동시 세션 한도를 알고 싶지만 공식 문서에 명시가 없다.
- 입력/지시 예시: "Cowork 동시 세션 몇 개까지 돌릴 수 있어?"
- 기대 결과   : "공식 문서에서 동시 세션 수 명시 페이지를 찾지 못했습니다.
                  현재 확인된 자료에서는 미확인 항목입니다. 시도한 URL 3개 + 확인 시각."
                  미확인 사유 + 추측 금지 안내.
- 주의점     : "보통 N개 정도" 같은 hedge 답변 금지. 블로그·SNS 후기는 1차
                  소스로 인정하지 않는다. 사용자가 직접 Anthropic 지원팀에 문의 권유.
```

## 작성 시 흔한 실수

- ❌ "기대 결과" 가 추상적 요약 ("정리된 표를 받습니다")
- ❌ "주의점"이 일반론 ("정확성에 유의하세요")
- ❌ 5줄 이상의 장문 예시 (구조 파괴)
- ❌ hedge 표현 포함된 "기대 결과" ("보통 5배 정도라고 답변")
- ✅ 구체 수치·구체 URL·구체 형식 명시
