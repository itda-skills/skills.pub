# 대한민국 공무원 보고서

## 개요

- **스타일명**: 공무원 보고서 (업무보고/현황보고/기획보고)
- **톤**: 간결체·경어체 혼용. 격식 있고 객관적인 문어체.
- **대상 독자**: 상급자 (과장, 국장, 실장, 차관, 장관 등), 관계 부처 담당자
- **적정 분량**: A4 1-3매 (핵심 보고서), A4 5-10매 (상세 보고서)

## 인터뷰 질문

- 보고 대상은 누구인가요? (과장/국장/장관/국무회의 등)
- 보고서 유형은? (업무보고/현황보고/기획보고/결과보고/검토보고)
- 관련 법령이나 정책이 있나요? (예: 「전자정부법」 제○조)
- 관련 부처나 협조 기관이 있나요?
- 추진 일정이나 예산 정보가 있나요?
- 보고 배경이 되는 지시사항이나 회의 결과가 있나요?

## 필수 구성 요소

보고서 유형에 따라 구성이 달라지나, 기본 골격은 다음과 같다.

### 1. 업무보고 / 현황보고

1. **제목**: 핵심 내용을 압축한 한 줄 (예: "2026년 상반기 디지털 전환 추진 현황")
2. **추진배경**: 왜 이 업무를 추진하는지 1-3문장. 상위 정책·지시사항 언급.
3. **추진현황**: 현재까지의 실적을 수치와 함께 제시. 표·그래프 활용 권장.
4. **문제점 및 애로사항**: 추진 과정에서의 장애 요인. 솔직하되 해결 방향 함께 제시.
5. **향후계획**: 향후 추진 일정·방법을 구체적으로 기술. 일정표 포함 권장.
6. **건의사항** (선택): 상급자의 결정이 필요한 사항.

### 2. 기획보고

1. **제목**
2. **추진배경 및 필요성**: 현 상황의 문제점과 개선 필요성.
3. **추진목표**: 달성하고자 하는 구체적 목표 (정량 지표 포함).
4. **추진전략 및 방안**: 핵심 전략을 3-5개 이내로 구조화.
5. **세부 추진계획**: 전략별 실행 과제, 일정, 담당, 예산.
6. **기대효과**: 정량적·정성적 기대효과.
7. **향후일정**: 마일스톤 일정표.

### 3. 검토보고

1. **제목**
2. **검토배경**: 검토를 요청받은 경위.
3. **현행 제도/현황**: 현재 상태 요약.
4. **검토의견**: 장단점 분석, 대안 비교.
5. **결론 및 건의**: 검토 결론과 건의사항.

## 문체 규칙

### 어미

- **기본 어미**: "~임", "~함", "~였음", "~할 예정임", "~필요함"
- **건의/요청**: "~바람", "~하여 주시기 바람"
- **계획**: "~할 계획임", "~추진할 예정임"
- **판단**: "~것으로 판단됨", "~것으로 사료됨"
- 구어체 어미 사용 금지: "~거든요", "~잖아요", "~인데요"

### 문장 규칙

- 한 문장 **50자 이내** 권장 (길어도 80자 이내)
- 한 문단 3-5문장 이내
- 주어 생략 가능 (관행적으로 주어 생략이 일반적)
- 수동태보다 능동태 사용: "검토되었음" → "검토하였음"
- 이중 부정 금지: "하지 않을 수 없다" → "해야 한다"

### 숫자·단위 규칙

- 숫자는 아라비아 숫자 사용: "3개 부처", "2026년"
- 금액은 원 단위: "50억원", "1,200만원"
- 비율은 소수점 한 자리: "23.5%"
- 날짜: "2026. 3. 28.(목)" 또는 "'26. 3. 28."

### 법령 인용

- 법률명은 「」(낫표) 사용: 「전자정부법」
- 조항은 "제○조제○항" 붙여쓰기
- 시행령·시행규칙은 정확한 명칭 사용

### 금지 표현

| 금지 | 대체 |
|------|------|
| ~라고 생각합니다 | ~것으로 판단됨 |
| 엄청, 매우, 굉장히 | 상당한, 현저한 |
| 좀, 약간, 대충 | 일부, 다소, 개략적으로 |
| ~해봤는데 | ~검토한 결과 |
| 이것, 그것 | 해당 사항, 동 건 |
| 개인적인 의견 | 검토 의견 |

## Frontmatter 템플릿

```yaml
---
title: "{보고서 제목}"
description: "{1문장 요약}"
created_at: {YYYY-MM-DD}
type: {업무보고|현황보고|기획보고|결과보고|검토보고}
department: "{부서명}"
drafter: "{작성자 직급·성명}"
recipient: "{보고 대상}"
classification: {일반|대외비|비밀}
---
```

## 예시 패턴

### 도입부 예시 (추진배경)

```markdown
## 추진배경

○ 「디지털플랫폼정부 추진계획」('23.4.) 후속 과제로 부처별 디지털 전환
  로드맵 수립·이행 추진 중

○ '26년 상반기 추진 현황을 점검하고 하반기 중점 과제를 보고하고자 함
```

### 본문 예시 (추진현황)

```markdown
## 추진현황

### 1. 디지털 전환 기반 구축

| 구분 | 목표 | 실적 | 달성률 |
|------|------|------|--------|
| 클라우드 전환 | 50개 시스템 | 38개 완료 | 76% |
| API 개방 | 200종 | 165종 공개 | 82.5% |
| AI 시범 도입 | 10개 과제 | 7개 진행 중 | 70% |

○ 클라우드 전환은 '26. 6월까지 목표 달성 가능할 것으로 전망됨
○ API 개방은 보안 검토 지연으로 일부 과제 일정 조정 필요
```

### 마무리 예시 (향후계획)

```markdown
## 향후계획

○ ('26. 4월) 하반기 중점 과제 확정 및 부처 협의
○ ('26. 5월) 2차 시스템 클라우드 전환 착수
○ ('26. 6월) 상반기 성과 평가 및 하반기 계획 보고

## 건의사항

○ 클라우드 전환 가속을 위한 추가 예산 5억원 확보 필요
○ AI 시범 과제 확대를 위한 범부처 협의체 구성 건의
```
