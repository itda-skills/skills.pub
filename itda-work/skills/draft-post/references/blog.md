# 블로그 포스팅

## 개요

- **스타일명**: 블로그 포스팅
- **톤**: 친근하고 실용적인 한국어 (기술 용어는 영어 유지)
- **대상 독자**: 개발자, 기술에 관심 있는 일반 독자
- **적정 분량**: 1500-3000자

## 인터뷰 질문

- 카테고리가 무엇인가요? (tutorial, guide, release, insight)
- 관련 스킬이나 도구가 있나요?
- 코드 예시나 명령어를 포함할까요?
- 참고할 문서, URL, 기존 블로그 글이 있나요?
- 톤을 기본(친근+실용)으로 할까요, 다른 톤을 원하시나요?

## 필수 구성 요소

1. **도입부**: 문제 제기 또는 상황 설정으로 시작. 독자가 "이건 나한테 필요한 글이다"라고 느끼게 한다.
2. **본문**: H2(`##`)로 주요 섹션을 구분. 단계별 설명 + 코드 예시. 한 섹션이 너무 길면 H3(`###`)로 분리.
3. **마무리**: 요약 + 다음 단계 제안. 독자가 바로 실행할 수 있는 액션 아이템을 포함.

## 문체 규칙

- **어미**: "~합니다", "~해요" 혼용 가능. 자연스러운 대화체.
- **문장 길이**: 제한 없으나 한 문단 3-5문장 권장.
- **금지 표현**: 없음 (자유로운 표현 가능)
- **코드 블록**: 반드시 언어 태그 포함 (```python, ```bash 등)
- **기술 용어**: 영어 그대로 사용 (번역하지 않음). 첫 등장 시 간단한 설명 추가 가능.
- **링크**: 참고 자료는 인라인 링크 또는 하단 참고 섹션에 모아서 표시.

## Frontmatter 템플릿

```yaml
---
title: "{글 제목}"
description: "{1-2문장 요약}"
created_at: {YYYY-MM-DD}
category: {tutorial|guide|release|insight}
tags:
  - tag1
  - tag2
published: false
---
```

## 예시 패턴

### 도입부 예시

> "프로젝트에서 API 키를 하드코딩하고 계신가요? 이번 글에서는 환경변수를 안전하게 관리하는 3가지 방법을 소개합니다."

### 본문 예시

```markdown
## 1. .env 파일 사용하기

가장 기본적인 방법입니다. 프로젝트 루트에 `.env` 파일을 만들고...

​```bash
# .env
API_KEY=your-api-key-here
​```

## 2. 시크릿 매니저 활용

프로덕션 환경에서는 클라우드 시크릿 매니저를 추천합니다...
```

### 마무리 예시

> "이제 API 키를 안전하게 관리할 수 있게 되었습니다. 다음 단계로는 CI/CD 파이프라인에서의 시크릿 관리를 살펴보세요."
