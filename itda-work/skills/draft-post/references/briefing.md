# 브리핑 자료

## 개요

- **스타일명**: 브리핑 자료 (간부 보고용 요약 보고)
- **톤**: 극도로 간결한 요약체. 핵심만 남기고 모두 제거.
- **대상 독자**: 고위 의사결정권자 (장·차관, 기관장, CEO)
- **적정 분량**: A4 1매 (최대 2매). 30초~2분 내 파악 가능해야 함.

## 인터뷰 질문

- 브리핑 대상은 누구인가요? (장관/차관/국장)
- 보고 사안의 핵심은 무엇인가요? (한 문장으로)
- 의사결정이 필요한 사항이 있나요?
- 관련 배경을 아는 분인가요, 처음 접하는 사안인가요?
- 긴급도는? (즉시 보고/정례 보고/참고 보고)

## 필수 구성 요소

1. **제목**: 사안을 한 줄로 압축.
2. **핵심 요약 (1-3줄)**: 전체를 30초 안에 파악할 수 있는 요약. 결론 선행.
3. **현황**: 수치 중심으로 현재 상태를 3-5줄 이내로.
4. **쟁점/문제점**: 의사결정이 필요한 핵심 이슈 1-3개.
5. **대안/건의**: 선택지를 제시하고 추천안 명시.
6. **참고사항** (선택): 배경 정보. 별도 붙임 가능.

## 문체 규칙

- **어미**: "~임", "~함", "~필요" (최소 어미)
- **문장 길이**: 한 문장 30자 이내 (절대 원칙)
- **문단**: 한 항목 2-3줄 이내
- **구조화**: 글머리(○, -, ·)와 표를 극대화
- **수식어**: 전면 제거. 수치와 팩트만 남김.
- **색상 강조**: 핵심 수치나 결론에 **굵은 글씨** 활용

### 핵심 원칙: "1-3-5 법칙"

- **1**문장 핵심 요약
- **3**개 이내 쟁점
- **5**줄 이내 현황

### 금지 표현

| 금지 | 이유 |
|------|------|
| 장문 서술 | 30초 내 파악 불가 |
| ~라고 생각됨 | 결론을 명확히 |
| 배경 설명이 긴 도입부 | 결론부터 시작 |
| 추상적 방향 제시 | 구체적 대안 제시 |

## Frontmatter 템플릿

```yaml
---

---
```

## 예시 패턴

### 전체 구조 예시

```markdown
# AI 인증제 시행 관련 브리핑

## 핵심

○ 4.1. AI 인증제 시행 예정, **52개 서비스 중 23개 미준비** → 일정 조정 필요

## 현황

○ 인증 완료: 29개(55.8%), 진행 중: 15개(28.8%), 미착수: 8개(15.4%)
○ 미준비 주요 원인: 데이터 품질 미달(12건), 보안 점검 미완(11건)

## 쟁점

○ 예정대로 4.1. 시행 시 미준비 기관 혼란 우려
○ 연기 시 국회 보고 일정과 충돌

## 건의 (택1)

| 대안 | 내용 | 장점 | 단점 |
|------|------|------|------|
| A안 (추천) | 4.1. 시행 + 6개월 유예기간 | 일정 준수 | 실효성 우려 |
| B안 | 7.1.로 연기 | 준비 충분 | 국회 일정 충돌 |
```
