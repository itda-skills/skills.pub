# 공문 / 시행문 / 업무연락

## 개요

- **스타일명**: 공문·시행문·업무연락
- **톤**: 격식 경어체. 공식적이고 명확한 문어체.
- **대상 독자**: 수신 기관 담당자(대외), 같은 기관 내 타 부서(대내)
- **적정 분량**: A4 1-2매

### 대외용(시행문) vs 대내용(업무연락) 차이

| 구분 | 대외용 (시행문) | 대내용 (업무연락) |
|------|----------------|------------------|
| 유통 범위 | 타 기관 | 같은 기관 내 타 부서 |
| 격식 수준 | 높음 (관인, 발신명의) | 중간 (관인 생략) |
| 인사말 | "귀 기관의 무궁한 발전을 기원합니다." | 생략 또는 간략 인사 |
| 법적 효력 | 공식 행정행위 | 내부 업무 조정 |
| 문서번호 | 시행번호 필수 | 기관별 상이 |

생성 시 대외용/대내용을 먼저 확인하고, 해당하는 양식을 적용한다.

## 인터뷰 질문

- 대외 공문인가요, 대내 업무연락인가요?
- 공문의 목적은 무엇인가요? (협조요청/통보/회신/의견조회)
- 발신 기관(부서)과 수신 기관(부서)은 어디인가요?
- 근거 법령이나 상위 지시사항이 있나요?
- 회신 기한이 있나요?
- 첨부할 자료가 있나요?

## 필수 구성 요소

### 두문

1. **발신 기관명**: 기관 로고 및 정식 명칭.
2. **수신**: "수신: ○○부장관(○○과장)" 또는 "수신처 참조"
3. **제목**: "(경유)" 표시 포함 가능. 핵심 내용 한 줄.

### 본문

4. **근거/배경**: "귀 기관의 무궁한 발전을 기원합니다." (인사말) + "~에 관한 건으로" (용건)
5. **세부 내용**: 요청/통보/협조 사항을 번호로 구분하여 기술.
6. **요청사항**: "~하여 주시기 바랍니다" 형식으로 명확히 기술.

### 결문

7. **붙임**: 첨부 자료 목록. "붙임 1. ○○○ 1부."
8. **발신자**: 기관장 직인 (또는 "○○부장관")
9. **기안·검토·협조·결재 정보**
10. **시행·접수 번호**: "시행 ○○과-○○○호(YYYY.MM.DD.)"
11. **담당자 연락처**

## 문체 규칙

- **어미**: "~합니다", "~바랍니다", "~알려드립니다"
- **인사말**: 첫 문장은 반드시 "귀 기관의 무궁한 발전을 기원합니다."
- **용건 연결**: "~에 관한 건으로 다음과 같이 알려드리오니"
- **요청**: "~하여 주시기 바랍니다", "~협조하여 주시기 바랍니다"
- **회신 요청**: "~까지 회신하여 주시기 바랍니다"
- **수신 경어**: "귀 기관", "귀 부(府)"

### 번호 체계

```
1. → 가. → (1) → (가) → ① → ㉮
```

### 금지 표현

| 금지 | 대체 |
|------|------|
| ~해주세요 | ~하여 주시기 바랍니다 |
| ~합시다 | ~하여 주시기 바랍니다 |
| ~인데요 | ~인바 |
| ~좀 | (삭제) |

## Frontmatter 템플릿

```yaml
---
title: "{공문 제목}"
description: "{1문장 요약}"
created_at: {YYYY-MM-DD}
scope: {대외|대내}
type: {협조요청|통보|회신|의견조회|업무연락}
sender:
  organization: "{발신 기관명}"
  department: "{발신 부서}"
  drafter: "{기안자 직급·성명}"
recipient: "{수신 기관(대외) 또는 수신 부서(대내)}"
reference: "{근거 문서번호 또는 법령}"
reply_due: "{회신 기한 YYYY-MM-DD}"
---
```

## 예시 패턴

### 협조요청 공문 예시

```markdown
수신: 행정안전부장관(디지털정부과장)
(경유)
제목: 공공 AI 서비스 품질 인증 관련 자료 협조 요청

1. 귀 기관의 무궁한 발전을 기원합니다.

2. 「공공 AI 서비스 품질 인증제 운영 규정」(과기정통부 고시 제2026-○호)에 따라
   공공기관 AI 서비스 현황 조사를 실시하고자 합니다.

3. 이와 관련하여 아래 자료를 2026. 4. 15.(화)까지 제출하여 주시기 바랍니다.

   가. 소관 AI 서비스 목록 및 운영 현황
   나. AI 서비스별 데이터 품질 자체 점검 결과
   다. 향후 AI 서비스 도입 계획

붙임 1. 자료 제출 양식 1부.
붙임 2. AI 서비스 품질 인증 안내서 1부.  끝.

                                    과학기술정보통신부장관
```

### 업무연락(대내용) 예시

```markdown
수신: 정보보호과장
제목: AI 서비스 보안 점검 일정 협조 요청

1. 「공공 AI 서비스 품질 인증제」 시행('26. 4. 1.) 관련입니다.

2. 인증 대상 52개 서비스 중 보안 점검이 미완료된 11개 서비스에 대해
   아래와 같이 점검 일정을 협조 요청드리오니, 검토 후 회신하여 주시기 바랍니다.

   가. 점검 대상: 미완료 11개 서비스 (별첨 목록 참조)
   나. 희망 기간: 2026. 4. 1. ~ 4. 15.
   다. 회신 기한: 2026. 3. 31.(월)

붙임 보안 점검 미완료 서비스 목록 1부.  끝.
```
