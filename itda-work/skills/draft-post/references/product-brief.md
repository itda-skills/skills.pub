# 제품/서비스 소개서

## 개요

- **스타일명**: 제품·서비스 소개서 (Product Brief, 회사소개서 포함)
- **톤**: 전문적이면서도 설득적. 고객 관점에서 가치를 전달.
- **대상 독자**: 잠재 고객, B2B 파트너, 영업 대상, 내부 영업팀
- **적정 분량**: A4 3-10매 (또는 슬라이드 10-20장)

## 인터뷰 질문

- 소개할 제품/서비스는 무엇인가요?
- 주요 타겟 고객은 누구인가요? (B2B/B2C, 산업군)
- 고객이 겪는 핵심 문제(Pain Point)는?
- 경쟁 제품 대비 차별점은?
- 주요 고객 사례나 도입 효과가 있나요?
- 요금 체계를 포함할까요?

## 필수 구성 요소

1. **캐치프레이즈**: 한 줄 가치 제안 (Value Proposition).
2. **문제 제기**: 타겟 고객이 겪는 문제를 공감 가능하게.
3. **솔루션 소개**: 제품/서비스가 문제를 어떻게 해결하는지.
4. **핵심 기능**: 주요 기능 3-5개를 아이콘+한줄 설명으로.
5. **차별점**: 경쟁사 대비 우위 (비교표 권장).
6. **고객 사례**: 도입 기업명, 도입 효과 (수치 포함).
7. **도입 프로세스**: 문의→상담→도입→지원 단계.
8. **요금 안내** (선택): 플랜별 비교 또는 "문의" 유도.
9. **연락처/CTA**: 데모 신청, 상담 문의 방법.

## 문체 규칙

- **어미**: "~합니다", "~됩니다" (경어체) 또는 "~해요" (친근체). 브랜드 톤에 따라.
- **문장 길이**: 한 문장 40자 이내. 슬라이드형은 더 짧게.
- **고객 중심**: "우리 제품은 ~" 보다 "고객은 ~할 수 있습니다"
- **수치 강조**: "처리 속도 3배 향상", "비용 40% 절감" 등 정량적 효과
- **시각화**: 기능 비교표, 도입 효과 차트, 프로세스 다이어그램

### 금지 표현

| 금지 | 대체 |
|------|------|
| 최고의, 최첨단 | 구체적 수치나 기술명 |
| 모든 문제를 해결 | 특정 문제에 대한 구체적 해결 |
| 경쟁사 직접 비하 | 객관적 비교표로 대체 |
| 기술 용어 남발 | 고객이 이해하는 언어로 |

## Frontmatter 템플릿

```yaml
---
title: "{제품/서비스명} 소개"
description: "{한 줄 가치 제안}"
created_at: {YYYY-MM-DD}
type: {제품소개서|서비스소개서|회사소개서}
company: "{회사명}"
product: "{제품/서비스명}"
target: "{타겟 고객}"
---
```

## 예시 패턴

### 캐치프레이즈 + 문제 제기 예시

```markdown
# itda-skills

**Claude Cowork를 위한 실무 스킬 마켓플레이스**

---

## 이런 불편함, 겪어보셨나요?

○ Claude Cowork에 같은 지시를 매번 반복 입력
○ 팀원마다 다른 프롬프트 품질 → 일관성 없는 결과
○ 유용한 워크플로우를 공유할 방법이 없음
```

### 핵심 기능 예시

```markdown
## 핵심 기능

| 기능 | 설명 |
|------|------|
| 원클릭 설치 | `claude plugin install` 한 줄로 스킬 추가 |
| 인터뷰 기반 | 맥락을 충분히 파악한 후 작업 수행 |
| 확장 가능 | references/에 파일 추가만으로 새 도메인 지원 |
| 크로스 플랫폼 | macOS, Linux, Windows 모두 지원 |
```
