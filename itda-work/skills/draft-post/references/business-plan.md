# 사업계획서

## 개요

- **스타일명**: 사업계획서 (IR 덱, 정부지원사업 신청서)
- **톤**: 설득적이되 객관적. 데이터 기반 논증. 자신감 있되 과장 없이.
- **대상 독자**: 투자자(VC), 심사위원, 경영진
- **적정 분량**: 슬라이드 15장 이내(IR) / A4 10-20매(서면)

## 인터뷰 질문

- 어떤 문제를 해결하는 사업인가요? (한 문장으로)
- 목표 시장은? (TAM/SAM/SOM 수준)
- 현재 제품/서비스 단계는? (아이디어/MVP/PMF/스케일업)
- 핵심 경쟁 우위는 무엇인가요?
- 팀 구성은? (핵심 인력 배경)
- 매출/Traction이 있나요?
- 투자 유치 목표 금액과 용도는?
- IR 덱인가요, 정부지원사업 신청용인가요?

## 필수 구성 요소

### IR 덱 (투자 유치용)

1. **커버**: 회사명, 한 줄 소개, 로고
2. **문제 (Problem)**: 해결하려는 문제를 공감 가능하게
3. **솔루션 (Solution)**: 문제 해결 방법, 제품 데모/스크린샷
4. **시장 (Market)**: TAM → SAM → SOM, 시장 성장률
5. **비즈니스 모델**: 수익 구조, 단가, 결제 주기
6. **경쟁 분석**: 경쟁사 비교 매트릭스, 차별점
7. **Traction**: 핵심 지표 (MAU, MRR, 성장률)
8. **팀**: 핵심 인력 소개, 도메인 전문성
9. **재무 계획**: 3개년 매출·비용 전망
10. **Ask**: 투자 요청 금액, 자금 사용 계획

### 정부지원사업 신청서

1. **사업 개요**: 사업명, 대표자, 기업 현황
2. **기술·제품 현황**: 기술 개발 이력, 보유 IP
3. **사업화 계획**: 시장 분석, 마케팅 전략, 판로 확보
4. **연구개발 계획**: R&D 내용, 일정, 인력 투입
5. **자금 소요 계획**: 정부 지원금 + 자부담 항목별 산출
6. **기대효과**: 매출, 고용, 기술 파급 효과

## 문체 규칙

- **어미**: IR 덱은 "~합니다"/"~입니다" (프레젠테이션체), 신청서는 "~임"/"~함" (간결체)
- **문장 길이**: 슬라이드당 핵심 메시지 1개, 본문 3줄 이내
- **수치**: 모든 주장에 데이터 근거 필수
- **시각화**: 차트, 표, 인포그래픽 활용 권장
- **스토리텔링**: 문제 → 솔루션 → 증거 → 비전 흐름

### 금지 표현

| 금지 | 대체 |
|------|------|
| 세계 최초, 유일한 | 주요 차별점은 ~ |
| 시장을 독점할 | 시장점유율 ○% 달성 목표 |
| 실패할 리 없다 | 리스크 요인과 대응 방안 |
| 경쟁사 없음 | 직간접 경쟁사 분석 제시 |

## Frontmatter 템플릿

```yaml
---
title: "{사업계획서 제목}"
description: "{한 줄 사업 소개}"
created_at: {YYYY-MM-DD}
type: {IR덱|정부지원사업|내부사업계획}
company: "{회사명}"
stage: {아이디어|MVP|PMF|스케일업}
funding_target: "{투자 유치 목표}"
---
```

## 예시 패턴

### 문제 정의 예시

```markdown
## Problem

○ 중소기업 70%가 AI 도입을 원하지만, 실제 도입률은 8.3%에 불과
○ 원인: 전문인력 부족(52%), 비용 부담(34%), 데이터 미확보(28%)
○ 기존 솔루션은 대기업 맞춤형 → 중소기업은 도입 엄두도 못 냄
```

### Traction 예시

```markdown
## Traction

| 지표 | '25 Q1 | '25 Q4 | '26 Q1 | QoQ |
|------|--------|--------|--------|-----|
| MAU | 1,200 | 4,800 | 12,500 | +160% |
| MRR | 8M | 32M | 85M | +166% |
| 유료 전환율 | 3.2% | 5.8% | 8.1% | +40% |

○ NPS 72점 (SaaS 평균 41점 대비 우수)
○ 리텐션: 3개월 잔존율 78%
```
