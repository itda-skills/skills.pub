# 뉴스레터

## 개요

- **스타일명**: 뉴스레터 (이메일 뉴스레터, 사내/외부 정기 발행)
- **톤**: 친근하면서도 전문적. 독자와 대화하는 느낌. 브랜드 페르소나 유지.
- **대상 독자**: 구독자 (고객, 잠재 고객, 임직원, 커뮤니티 멤버)
- **적정 분량**: 본문 500-1500자 (읽기 3-5분)

## 인터뷰 질문

- 뉴스레터의 주제/테마는 무엇인가요?
- 발행 주체(브랜드/개인/팀)는?
- 정기 발행인가요? (주간/격주/월간)
- 독자 페르소나는? (개발자/마케터/경영진/일반)
- CTA(Call to Action)가 있나요? (링크 클릭, 이벤트 참여 등)
- 브랜드 톤이 정해져 있나요?

## 필수 구성 요소

1. **발신자 인사**: 브랜드 페르소나에 맞는 짧은 인사. (예: "안녕하세요, ○○팀입니다.")
2. **도입부 훅**: 첫 2-3줄에서 흥미를 끄는 질문, 통계, 또는 이슈 제기.
3. **본문**: 핵심 콘텐츠 1-3개. 각 콘텐츠에 소제목 + 요약 + 상세.
4. **CTA**: 행동 유도 (링크, 버튼, 참여 안내). 본문당 1개 CTA 원칙.
5. **마무리**: 다음 호 예고 또는 간단한 인사.
6. **푸터**: 구독 해지 링크, 발행처 정보, 개인정보 처리방침.

## 문체 규칙

- **어미**: "~해요"/"~합니다" 혼용 가능. 브랜드 톤에 따라 선택.
- **문장 길이**: 한 문장 40자 이내. 모바일 가독성 우선.
- **문단**: 2-3문장 이내. 여백 충분히.
- **이모지**: 소제목이나 글머리에 적절히 활용 가능 (과용 금지).
- **링크**: 인라인 링크보다 별도 CTA 버튼/텍스트 권장.
- **개인화**: "○○님" 등 독자 이름 삽입 가능.

### 모바일 최적화

- 제목: 30자 이내 (모바일 미리보기 영역)
- 본문 폰트: 14px 이상 상정
- 이미지: 텍스트 대체 가능하도록 (이미지 차단 환경 대비)
- 1단 레이아웃 권장

```yaml
title: "{뉴스레터 제목}"

created_at: {YYYY-MM-DD}
type: 뉴스레터
brand: "{발행 브랜드/팀명}"
issue_number: {호수}
frequency: {주간|격주|월간}
cta_url: "{주요 CTA 링크}"
---
```

## 예시 패턴

### 도입부 예시

```markdown
안녕하세요, itda 팀입니다.

이번 주 가장 많이 받은 질문: "Claude Code에서 MCP 서버를 여러 개 동시에 쓸 수 있나요?"

답은 '네'입니다. 그리고 생각보다 훨씬 쉽습니다.
오늘은 MCP 멀티 서버 설정법을 3단계로 정리했습니다.
```

### CTA 예시

```markdown
---

**이번 주 추천 스킬**

itda-web-reader가 v1.2로 업데이트되었습니다.
PDF 파싱 속도가 3배 빨라졌어요.

[업데이트 내용 보기 →](https://example.com/changelog)

---
```

### 마무리 예시

```markdown
다음 주에는 "Claude Code 훅(Hooks)으로 자동화하기"를 다룰 예정입니다.
궁금한 점이나 다뤘으면 하는 주제가 있다면 회신해 주세요.

좋은 한 주 되세요!
itda 팀 드림
```
