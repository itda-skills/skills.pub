## 빠른 시작

이미지 정보 조회부터 리사이즈·포맷 변환까지 한 줄 요청으로 처리합니다.

```
이 사진 정보 알려줘
이미지 크기 줄여줘 (1920px로)
PNG를 JPG로 바꿔줘
```

파일 경로만 알려주면 스킬이 자동으로 operation을 판단하고 실행합니다.

## 활용 시나리오

### 웹 업로드용 리사이즈·변환

블로그·SNS에 올리기 전 용량을 줄입니다.

```
photo.jpg를 가로 1920px로 줄여줘
스크린샷을 JPG로 바꿔서 용량 줄여줘
```

### 인쇄 준비 (DPI 300)

온라인 인쇄 업체가 요구하는 300 DPI로 맞춥니다.

```
이 사진 DPI 300으로 바꿔줘
```

### 파일 관리 (여백 정리·회전)

스캔본의 흰 여백 제거나 사진 방향 보정에 사용합니다.

```
스캔본 여백 자동으로 잘라줘
사진 90도 회전해줘
```

## 출력 옵션

사용자가 출력 경로를 지정하지 않으면 입력 파일과 같은 디렉토리에 규칙 접미사를 붙여 저장합니다.

| 입력 파일 | 작업 | 기본 출력 파일 |
|-----------|------|----------------|
| `photo.jpg` | resize | `photo-resized.jpg` |
| `scan.jpg` | set-dpi | `scan-300dpi.jpg` |
| `logo.png` | convert | `logo.jpg` |
| `image.png` | crop-edges | `image-cropped.png` |
| `photo.jpg` | rotate | `photo-rotated.jpg` |

resize 작업은 `--target-width`, `--target-height`, `--multiplier` 중 하나 이상으로 크기를 지정하며, `--resize-mode` 로 비율 유지 방식을 선택합니다 (`fit` · `fill` · `exact`).

## 팁

- **자동 여백 크롭**: `crop-edges` 에 `--auto-detect` 를 붙이면 스캔본 흰 여백을 자동으로 감지해 제거합니다. 감지 민감도는 `--threshold` 로 조정합니다 (0~255, 기본 10).
- **저장 전 미리보기**: 모든 write 명령에 `--dry-run` 을 붙이면 실제 파일을 생성하지 않고 예상 수치만 반환합니다. 큰 이미지를 다루기 전에 먼저 확인하세요.
- **set-dpi 는 고품질 기본값**: `--jpeg-quality` 기본값이 다른 명령(85)과 달리 95 입니다. 인쇄용 DPI 변경 시 화질 손실을 최소화합니다.
- **⚠️ 주의사항**: (1) JPEG·PNG 만 지원합니다 — WebP·HEIC·TIFF 는 `UNSUPPORTED_FORMAT` 에러를 반환합니다. (2) 10MB 이상 이미지는 처리 시간이 더 걸릴 수 있습니다. (3) `--overwrite` 기본값은 false 이므로 출력 파일이 이미 있으면 에러를 반환합니다. 덮어쓰기를 원할 때만 명시적으로 추가하세요.
