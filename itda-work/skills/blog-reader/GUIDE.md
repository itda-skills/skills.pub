## 빠른 시작

네이버 블로그의 글 목록·본문·댓글·블로그 내 검색을 로그인 없이 읽는 가장 간단한 방법입니다.

```
네이버 블로그 글 가져와줘
```

```
이 블로그 본문이랑 댓글 같이 보여줘
```

```
이 블로그 최근 7일 글 보여줘
```

이렇게 말하면 스킬이 자동으로 적절한 서브커맨드(`list`/`post`/`comments`/`search`/`read`)를 골라 공개 포스트를 조회합니다.

## 활용 시나리오

### 최근 글 목록 조회

특정 블로그의 최근 포스트를 날짜 내림차순으로 가져옵니다.

```
아스트로유지 블로그 최근 7일 포스팅 가져와줘
```

내부적으로 `list --blog-id astroyuji --days 7` 형태로 호출됩니다. `--limit`(기본 20), `--category` 필터도 함께 쓸 수 있습니다.

### 본문 + 댓글 한 번에 보기

URL 하나로 본문과 댓글 트리를 동시에 가져오고 싶을 때는 `read`가 편리합니다.

```
이 블로그 포스팅 본문이랑 댓글 같이 보여줘
```

`read --url <포스트 URL>` 로 본문(`article`)과 댓글(`comments`)을 한 응답에 담아 반환합니다. 본문만 필요하면 `post`, 댓글만 필요하면 `comments`를 사용합니다.

### 블로그 내 키워드 검색

특정 블로그 안에서 키워드로 포스트를 찾습니다.

```
이 블로그에서 '인공지능' 검색해줘
```

`search --blog-id <ID> --query 인공지능` 으로 동작합니다.

## 출력 옵션

| 옵션 | 설명 | 사용 시점 |
|------|------|-----------|
| `--format json` | UTF-8 pretty-print JSON (기본) | 후처리·파싱이 필요할 때 |
| `--format markdown` | 제목·메타·본문·댓글을 마크다운으로 | 사람이 바로 읽을 때 |
| `--no-image-urls` | 본문 이미지 URL을 `[이미지: alt]`로 치환, `images` 배열 비움 (`post`·`read` 전용) | 이미지가 많은 글에서 토큰을 절감할 때 |
| `--output 경로` | 결과를 파일로 저장 (기본 stdout) | 수집 결과를 로컬에 보관할 때 |
| `--body-format {markdown,html}` | 본문 형식 (기본 markdown) | 원본 HTML이 필요할 때 |

## 팁

- **자가차단 방지**: 동일 블로그에 대한 반복 호출을 줄이고, 필요한 정보를 한 번에 일괄 조회한 뒤 로컬에 저장해 쓰세요. `--throttle` 기본값 0.5초면 일반 사용에 충분하며, 대량 수집 시 `--throttle 1.0` 이상을 권장합니다.
- **이미지 URL 제거는 명시 요청 시에만**: "이미지는 빼고", "본문 텍스트만" 처럼 명시적으로 요청할 때만 `--no-image-urls`가 적용됩니다. 단순 "요약해줘"에는 적용되지 않습니다. 실측으로 이미지 26장 여행 포스트가 18,723 B → 6,522 B(-65%)로 줄었습니다.
- **댓글 필터**: `--filter-author 닉네임` 으로 특정 작성자 댓글만 평탄 리스트로 모을 수 있습니다(정확 일치).
- **Exit Code로 상황 구분**: 0 성공 / 2 인자 오류 / 3 결과 없음 / 4 anti-bot 차단(403·429) / 5 미지원 플랫폼(네이버 외) / 6 비공개·삭제된 글.
- **조회 전용·네이버 전용**: 작성·수정·댓글 달기는 지원하지 않고, anti-bot 우회도 하지 않습니다. Tistory·Velog·Brunch는 아직 미지원입니다.
