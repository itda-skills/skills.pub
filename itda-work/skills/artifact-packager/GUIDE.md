# artifact-packager 사용 가이드

Claude가 만든 HTML 산출물이나 정적 웹 문서를 내 PC에서 바로 열어볼 수 있는 형태로 묶어줍니다.
**Claude에게 말로 부탁하면 됩니다.** 터미널 명령을 직접 입력할 필요는 없습니다.

---

## 이렇게 말하세요

| 하고 싶은 것 | Claude에게 이렇게 말하세요 |
|--------------|----------------------------|
| HTML 산출물 실행 준비 | "이 HTML 산출물을 내 PC에서 실행할 수 있게 만들어줘" |
| 단일 실행파일로 묶기 | "Claude Artifact를 실행파일 하나로 묶어줘" |
| zip으로 묶기 | "이 웹 산출물을 zip 파일로 묶어서 전달하기 쉽게 만들어줘" |
| 비밀번호 걸기 | "실행하면 사용자 이름과 비밀번호를 물어보게 해줘" |
| zip과 비밀번호 함께 요청 | "이 웹 산출물을 zip으로 묶고, 열 때 비밀번호를 확인하게 해줘" |
| 받을 컴퓨터에 맞추기 | "Windows 노트북에서 실행할 수 있게 만들어줘" |
| 전달용 설명 포함 | "받는 사람이 어떻게 열면 되는지도 같이 적어줘" |

---

## 어떤 산출물을 맡길 수 있나요?

- 단일 HTML 파일을 맡길 수 있습니다.
- `index.html`이 들어 있는 정적 웹 폴더를 맡길 수 있습니다.
- Claude Artifact처럼 이미 만들어진 정적 웹 산출물을 맡길 수 있습니다.

---

## 어느 컴퓨터에서 쓸지 알려주세요

산출물을 받을 사람이 Windows에서 쓸지, Mac에서 쓸지, Linux에서 쓸지에 따라 실행파일이 달라집니다. 요청에 운영체제를 말하지 않으면 Claude가 어느 컴퓨터에서 사용할지 물어봅니다.

Mac의 경우 Apple Silicon Mac인지 Intel Mac인지도 물어볼 수 있습니다. 여러 사람에게 전달해야 한다면 Windows용과 Mac용처럼 여러 종류로 나누어 만들어달라고 말하면 됩니다.

---

## 산출물을 받은 사람이 실행하는 법

단일 실행파일로 받은 경우, 받은 실행파일을 실행하면 터미널에 로컬 주소가 표시됩니다. 그 주소를 브라우저에서 열면 웹 산출물을 볼 수 있습니다.

zip 파일로 받은 경우, 먼저 압축을 풀고 안내된 실행 파일을 실행합니다. 실행 후 터미널에 표시되는 로컬 주소를 브라우저에서 열면 됩니다.

비밀번호 보호를 요청한 경우, 브라우저에서 주소를 열 때 사용자 이름과 비밀번호를 입력해야 합니다. 전달할 때는 실행파일이나 zip 파일과 함께 접속 정보도 별도로 알려주세요.

---

## 한계

- `index.html`을 포함한 정적 산출물만 다룹니다.
- React/Vite 같은 프로젝트를 빌드하는 작업 자체는 이 스킬 범위가 아닙니다. 이미 만들어진 정적 산출물을 기준으로 묶어달라고 요청하세요.
- 폐쇄망·오프라인 환경 지원은 향후 과제입니다.
- 외부 CDN, 원격 API, 웹폰트, 외부 이미지에 의존하는 페이지는 네트워크가 없으면 정상 표시되지 않을 수 있습니다.
- 실행 명령과 세부 옵션은 Claude가 참고하는 `SKILL.md`에만 두며, 이 가이드에는 중복해서 적지 않습니다.
