# Mirror-back 패턴 가이드

Stage 2 mirror-back 진행 시 이 패턴을 참고한다.

## 기본 Mirror-back 템플릿

사용자 입력을 받은 직후 다음 구조로 정리해 출력한다:

```
제가 이해한 바를 다시 한 번 정리해드릴게요.

- 목표: [사용자가 달성하고 싶은 것 한 줄]
- 다루는 자료: [어떤 데이터·파일·서비스]
- 얼마나 자주: [매일 / 매주 / 월 1회 / 요청 시]
- 원하는 결과물: [어디에, 어떤 형태로]

이 정리 중에 빠지거나 잘못된 부분이 있나요?
```

## find-work 메모 첨부 시 — 축약 mirror-back

find-work 메모(또는 같은 세션 이어받기)는 이미 인터뷰로 합의된 산출물이다. 전체 4항목 재확인은 이중 노동 — 아래 축약형을 쓴다:

```
find-work에서 정리한 내용을 이어받을게요. 요지는 —

- [메모 요지 2~3줄: 문제 정의 + 채택한 실행 경로 + 기대 가치]

메모 내용에서 달라진 점이나 추가할 조건이 있나요? 없으면 바로 계획으로 넘어갈게요.
```

- "없어요" / "그대로예요" → Stage 3으로 이동.
- 변경점이 있으면 **해당 항목만** 아래 일반 mirror-back(전체 정리본 재출력)으로 확인 후 이동.
- 자유 텍스트 입력에는 이 축약형을 쓰지 않는다 — 합의된 적 없는 입력은 전체 mirror-back 대상이다.

## 긍정 응답 패턴 (Stage 3으로 이동)

다음 표현은 사용자가 mirror-back 결과를 승인한 것으로 본다:

- "맞아요", "정확해요", "맞습니다", "그렇습니다"
- "이게 다예요", "네", "그거요", "딱 그거예요"
- "그대로 진행해줘", "계속 진행해"

## 수정 요청 패턴 (해당 항목만 반영 후 재출력)

사용자가 수정을 요청하면 해당 항목만 바꿔서 전체 정리본을 다시 출력한다.

수정 요청 예시:
- "빈도가 달라요 — 매주가 아니라 월 1회예요"
- "자료가 더 있어요 — 이메일 말고 엑셀 파일도 포함해야 해요"
- "결과물 위치가 달라요 — 슬랙이 아니라 이메일로 받고 싶어요"

수정 후 재출력 예시:
```
네, 수정했어요.

- 목표: [이전과 동일]
- 다루는 자료: [수정된 내용]
- 얼마나 자주: 월 1회
- 원하는 결과물: [이전과 동일]

이제 이 정리가 맞나요?
```

## 3회 초과 시 항목 단위 확인 모드

mirror-back을 3번 주고받은 뒤에도 계속 수정 요청이 오면, 항목을 하나씩 따로 확인한다.

항목 단위 확인 예시:
```
목표 한 줄부터 하나씩 확인해볼게요.

"매주 거래처 이메일을 분류해서 슬랙에 요약을 보낸다" — 이 목표가 맞나요?
```

→ 확인되면 다음 항목으로 이동:
```
자료는요 — "받은편지함의 거래처 이메일"을 다루는 것이 맞나요?
```

→ 이런 식으로 네 항목(목표 → 자료 → 빈도 → 결과물)을 순서대로 확인한다.

## 반복 종료 패턴

다음 상황이 되면 mirror-back을 종료하고 Stage 3으로 이동한다:

1. 사용자가 긍정 응답을 줬다.
2. 항목 단위 확인 모드에서 네 항목을 모두 확인했다.
3. 사용자가 종료 키워드("여기까지", "그만", "보류")를 발화했다.

## 주의사항

- 사용자가 아주 상세하게 적어줬더라도 mirror-back을 건너뛰지 않는다.
- 미리 결론을 내리지 않는다 — 사용자가 놓친 조건이 있을 수 있다.
- 수정 요청이 오면 해당 부분만 바꾸고, 확인되지 않은 다른 항목을 임의로 바꾸지 않는다.
