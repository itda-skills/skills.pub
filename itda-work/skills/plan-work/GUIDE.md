# plan-work 상세 가이드

## 빠른 시작

Claude에게 이렇게 말해보세요:

> "거래처 메일을 매주 정리해서 슬랙에 보내는 거 계획 세워줘"

> "find-work 메모 받았는데, 이걸 어떻게 진행하면 될지 계획을 잡아줘" (메모 첨부)

> "이거 어떤 itda 스킬로 풀 수 있어?"

---

## find-work와 어떻게 다른가

두 스킬은 형제 관계입니다. 상황에 따라 다르게 씁니다.

| 내 상황 | 쓸 스킬 |
|--------|--------|
| 뭘 자동화할지 아이디어가 없어요 | find-work |
| 자동화할 건 알아요. 어떻게 시작할지 모르겠어요 | **plan-work** |
| find-work 메모 받았고, 다음 단계가 궁금해요 | **plan-work** (메모 첨부) |

보통 find-work → plan-work 순서로 이어집니다.

---

## 활용 시나리오

### 시나리오 1: 자유롭게 적기

Claude를 켜고 하고 싶은 걸 한두 문장으로 적으세요.

> "매주 월요일 아침, 거래처 이메일에서 견적 요청만 골라 슬랙 채널에 요약을 보내고 싶어요."

plan-work가 이 내용을 정리하고, itda-* 스킬과 어떻게 연결할지 한 장 메모로 만들어줍니다.

### 시나리오 2: find-work 메모 첨부

find-work를 먼저 돌리고 받은 메모 파일을 첨부하세요.

> [메모 첨부] "이 메모 기반으로 계획 세워줘"

plan-work가 메모에서 핵심 정보를 뽑아 mirror-back부터 시작합니다.

### 시나리오 3: 이전 plan-work 메모 다듬기

예전에 만든 plan-work 메모를 첨부하고, 특정 부분만 다듬을 수 있습니다.

> [메모 첨부] "단계별 계획 부분을 좀 더 구체적으로 다듬어줘"

변경하지 않는 섹션은 그대로 보존합니다.

### 시나리오 4: 아직 정리가 안 됐을 때

Stage 1에서 "아직 정리가 안 됐어요" 옵션을 고르면, plan-work가 find-work를 안내하고 종료합니다. 먼저 find-work로 아이디어를 잡고 오세요.

---

## 산출 메모 구조

plan-work가 만들어주는 메모는 항상 이 여섯 섹션으로 구성됩니다:

### 1. 요구사항
사용자가 말한 것과 mirror-back으로 합의한 목표를 함께 담습니다.

### 2. 단계별 계획
어떤 itda 스킬을 어떤 순서로 부를지, Claude에게 어떻게 말하면 되는지를 자연어 예시로 알려줍니다.

예시:
> "Claude에게 이렇게 말하세요 — '거래처 메일 폴더에서 이번 주 견적 요청만 골라줘'"

### 3. 선행 자료 안내
자동화를 시작하기 전에 준비해야 할 자료를 네 가지로 설명합니다:
- 자료 이름 (예: 거래처 이메일 목록)
- 무엇인지 (1~2문장)
- 형식 (예: 엑셀 파일, 텍스트 메모)
- 어디에 보관할지 (예: 바탕화면에 새 폴더 만들어서)

### 4. 필요한 키·접근 권한
API 키나 비밀번호가 필요하다면 어디서 발급하는지 한 줄로 알려줍니다.

예시:
> NAVER_EMAIL: 네이버 계정 이메일 주소를 입력하면 됩니다.

### 5. 다음 세션에서 시작하기
메모를 저장한 뒤 Claude를 다시 켤 때, 그대로 따라 치면 되는 문장 2~3개를 적어줍니다.

예시:
> "이메일 스킬로 거래처 메일 읽어줘"

### 6. 실패 시 대처
각 단계가 잘 안 됐을 때 어떻게 하면 되는지 간단히 알려줍니다.

예시:
> 메일이 한 통도 안 잡히면, 검색어를 '견적' 대신 '문의'로 바꿔서 다시 시도해 보세요.

---

## 완성 메모 예시

```markdown
# 실행 계획: 거래처메일-슬랙-요약

작성일: 2026-05-21 10:30

## 요구사항

목표: 매주 월요일 아침, 거래처 이메일에서 견적 요청만 골라 슬랙 채널에 요약을 보낸다.

- 다루는 자료: 네이버 받은편지함
- 빈도: 매주 월요일 아침
- 원하는 결과물: 슬랙 #영업팀 채널에 요약 메시지

## 단계별 계획

1단계 — 이메일 읽기
Claude에게 이렇게 말하세요:
"이번 주 받은 거래처 이메일 중 제목에 '견적' 또는 '요청'이 있는 것만 읽어줘"

2단계 — 슬랙 전송
Claude에게 이렇게 말하세요:
"방금 정리한 내용을 슬랙 #영업팀 채널에 보내줘"

## 선행 자료 안내

**네이버 앱 비밀번호**
- 무엇인지: 이메일 스킬이 네이버 받은편지함에 접근할 때 필요한 별도 비밀번호입니다.
- 형식: 숫자·영문 조합 16자리 문자열
- 어디서 만드는지: 네이버 로그인 → 내 정보 → 보안설정 → 앱 비밀번호 → 이메일 선택 후 발급

## 필요한 키·접근 권한

- NAVER_EMAIL: 네이버 계정 이메일 주소
- NAVER_APP_PASSWORD: 네이버 앱 비밀번호 (위 안내 참고)

> 네이버 앱 비밀번호 발급 절차는 [네이버 앱 비밀번호 발급 가이드](https://skills.itda.work/credentials/naver-app-password/)를 참고하세요.

## 다음 세션에서 시작하기

1. "이메일 스킬로 이번 주 거래처 견적 요청 이메일 읽어줘"
2. "방금 정리한 메일 슬랙 영업팀 채널에 요약해서 보내줘"

## 실패 시 대처

이메일이 잡히지 않을 때: 검색어를 '견적' 대신 '문의', '요청', '가격' 으로 바꿔 재시도
슬랙 전송 실패 시: 슬랙 채널명이 정확한지 확인 후 다시 시도
```

---

## 진행 흐름 요약

```
Stage 1  입력 방식 선택 (4지선다)
    ↓
Stage 2  Mirror-back — "이렇게 이해했어요, 맞나요?"
    ↓ (최대 3회 수정)
Stage 3  계획 수립 + Ground-check
    ↓
Stage 4  컨펌 게이트 — 저장 / 수정 / 다시 / 보류
    ↓
        메모 파일 저장
```

---

## 종료 신호

진행 중 언제든 이 표현을 쓰면 현 단계에서 바로 컨펌 게이트로 이동합니다:

- "여기까지" / "그만" / "보류"

컨펌 게이트에서 "보류"를 고르면 채워진 내용만 `미정` 마커와 함께 저장됩니다.

---

## 팁

**find-work 메모를 최대한 활용하세요.** find-work 메모를 첨부하면 확인 절차가 "달라진 점 있나요?" 한 번으로 줄어듭니다 — 이미 인터뷰로 합의된 내용이니까요. 메모의 가치 추정과 검증 방법도 실행 계획에 그대로 이어집니다. find-work 직후 같은 대화라면 첨부 없이 "이어서 계획 세워줘"라고만 해도 됩니다.

**메모는 하나씩 만드세요.** 한 세션에 하나의 자동화 계획을 만드는 것이 적합합니다. 여러 자동화를 묶으려면 세션을 나눠서 각각 메모를 만드세요.

**이전 메모를 첨부하면 이어서 다듬을 수 있습니다.** 저장한 메모를 다음 세션에 첨부하고 "단계별 계획 부분 다듬어줘"라고 하면 그 부분만 재작업합니다.

---

## 제한사항

- plan-work는 계획 메모만 만듭니다. 실행은 사용자가 직접 해야 합니다.
- 다른 itda-* 스킬을 이 스킬이 직접 호출하지 않습니다.
- 자동으로 이전 세션을 불러오지 않습니다. 이어 작업하려면 이전 메모를 직접 첨부하세요.
- 투자·금융 관련 디스클레이머는 해당 스킬이 직접 안내합니다.
