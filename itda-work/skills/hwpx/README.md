# hwpx — HWP/HWPX 문서 리더 for Claude Cowork

한글(HWP5, HWPX) 문서를 Markdown/HTML로 변환하는 Claude Cowork 스킬입니다.

```
"이 HWP 파일 읽어줘" → Claude가 자동으로 변환 + 내용 요약
```

---

## 왜 이 스킬을 써야 하는가

> **클로드 단독(olefile 즉석 스크립트)도 HWP 본문 추출이 가능합니다.** 차이는 반복 비용입니다.

### 클로드 단독 vs hwpx 스킬 비교

| 항목 | 클로드 단독 (olefile 등 즉석 스크립트) | hwpx 스킬 |
|------|---|---|
| **1회 호출 토큰** | **~114,000** (실측 평균, 보도자료 2건) | **~3,100** (실측, 한강 보도자료) |
| **10회 누적 토큰** | **~1,144,000** (매번 파서 재작성) | **~28,000** (1회 후 캐시) |
| **비율 (10회 누적)** | — | **0.003** (Track A 대비, AC-DIFF-003 임계값 0.5 달성) |
| **디버깅 라운드** | **평균 4회** (실측: 한강 3회, 강북 5회) | **0회** (hwpx 바이너리 내장 처리) |
| **새 엣지케이스 (이미지 포함 파일 등)** | 추가 코드 작성 필요 (즉석 디버깅 재발생) | 동일 호출 (내장 처리기로 흡수) |
| HWPX 본문 추출 | 부분 가능 (네임스페이스 누락 빈번) | 완전 추출 |
| HWP5 본문 추출 | **가능** (50줄 olefile+zlib+struct 스크립트로 시연됨) | 완전 추출 |
| 표 구조 보존 | 0.625 (실측 평균, 병합 셀 미처리) | 한국 공공기관 규격 행-열 보존 |
| 이미지 추출 | 이론상 가능 (BinData 코드 추가 작성 시) / **실제로는 매번 비용** | 자동 추출 (코드 작성 비용 0) |
| 이미지 캡션 | 0 (지원 안 함) | Sonnet 서브에이전트 자동 캡션 삽입 |
| Markdown 변환 | raw text dump (heading/list 손실) | 정식 Markdown (heading·list·표) |
| 양산성 | 매 파일마다 즉석 코드 작성, 안정성 보장 안 됨 | 1회 호출, 동일 결과 보장 |

> **실측 출처**: `evals/results/track_a_hangang.json` + `track_a_gangbuk.json`, 측정일 2026-05-11
> Track A: 한강 잠퍼자기 대회 보도자료 116,988토큰 + 강북등발전계정 보도자료 111,765토큰 → 평균 114,377토큰/회.
> Track B: hwpx 스킬 한강 보도자료 실측 3,113토큰/회.
> 10회 누적 ratio = 28,017 / 1,143,770 = **0.003** — AC-DIFF-003 임계값(≤0.5) 약 185배 달성.

### 언제 클로드 단독을 써도 되나

- 표 없이 본문만 필요한 간단한 문서
- 1~2회 일회성 처리

### 언제 이 스킬을 써야 하나

- 표 구조가 중요한 문서 (담당부서 표, 일정표 등) — 클로드 단독은 표를 평탄화합니다
- 이미지가 포함된 문서 — 클로드 단독은 이미지를 무시합니다
- 양산 환경 (10개 이상 일괄 처리, 안정적 Markdown 변환 필요) — 클로드 단독은 매번 즉석 코드 작성으로 안정성 보장 어렵습니다

## 주요 기능

### 1. HWP/HWPX 원클릭 변환

HWP5(.hwp)와 HWPX(.hwpx) 문서를 Markdown 또는 HTML로 변환합니다.
파일을 업로드하고 "읽어줘"라고 말하면 끝.

- **Markdown 변환**: 이미지 자동 추출 + 텍스트 구조 보존
- **HTML 변환**: 원본 서식 보존, 이미지 Base64 인라인 임베디드

### 2. AI 이미지 캡션 자동 생성

문서에서 추출된 이미지를 AI가 분석하여 한국어 설명을 자동 삽입합니다.

```markdown
# 변환 전
![](보도자료_2024/image_0001.png)

# 변환 후
![조직도: 본부장 아래 3개 팀이 배치된 구조](보도자료_2024/image_0001.png)
```

- Sonnet 모델 서브에이전트가 이미지별로 병렬 분석
- 512px 리사이즈로 토큰 최적화 (원본 대비 ~90% 절감)
- "이미지 설명 없이" 키워드로 옵트아웃 가능

### 3. 공공문서 Table 평탄화

한국 공공기관 문서 특유의 레이아웃 테이블을 LLM 친화적 리스트로 변환합니다.

```markdown
# 변환 전 (레이아웃 테이블)
| 기관명 | 한국도로공사 |
| 담당부서 | 도로관리처 |

# 변환 후 (깔끔한 불릿 리스트)
- **기관명**: 한국도로공사
- **담당부서**: 도로관리처
```

4가지 규칙으로 키-값, 레이아웃, 다열 데이터, 양식 테이블을 자동 분류 후 평탄화합니다.
비교표, 교차분석표 등 구조가 중요한 테이블은 자동으로 보존합니다.

### 4. 크로스 플랫폼 지원

| 환경 | 지원 방식 |
|------|----------|
| Linux (Cowork) | 바이너리 번들 내장, 자동 추출 |
| macOS | PATH 자동 탐색 |
| Windows | PATH 자동 탐색 |

## 사용 예시

```
# 기본 사용
"이 HWP 파일 읽어줘"
"한글 문서 내용 보여줘"

# 옵션
"HWP를 마크다운으로 변환해줘"
"이 hwpx 파일을 HTML로 변환해줘"
"표 형식 그대로 보여줘"          # 테이블 평탄화 건너뛰기
"이미지 설명 없이 변환해줘"      # 이미지 캡션 건너뛰기
```

## 지원 포맷

| 입력 | 출력 | 특징 |
|------|------|------|
| HWP5 (.hwp) | Markdown | 이미지 추출 + AI 캡션 |
| HWP5 (.hwp) | HTML | Base64 임베디드 이미지 |
| HWPX (.hwpx) | Markdown | 이미지 추출 + AI 캡션 |
| HWPX (.hwpx) | HTML | 원본 서식 보존 |

### 지원 문서 요소

단락, 제목(H1-H6), 표(셀 병합 포함), 목록, 이미지, 볼드/이탤릭/밑줄, 하이퍼링크(HWPX), 코드 블록(HWPX), 수평선(HWPX)

---

## 설계 철학

- **바이너리 중심**: Go 네이티브 CLI(`hwpx`)가 파싱/변환 담당. 빠르고 안정적
- **무의존성**: Python 패키지 설치 불필요
- **LLM 최적화**: 테이블 평탄화 + 이미지 캡션으로 후속 AI 작업 품질 향상
- **토큰 효율**: Sonnet 서브에이전트 + 이미지 리사이즈로 context 소모 최소화

---

## 요구 사항

- `hwpx` CLI v1.0.2 이상 권장 (v1.0.1 이하도 β fallback 으로 동작)
- Linux: 스킬에 번들 내장 (자동 설치)
- macOS/Windows: [hwpx 설치](https://github.com/nicecai/hwpx) 후 PATH에 추가
