# 판단 기준이 막힐 때 — 세 길의 트레이드오프

2단계 3번 "판단 기준" 단계에서 사용자가 막혔을 때 객관적으로 제시할 세 가지 길. **추천 부여 금지** — 트레이드오프만 보여주고 사용자가 선택.

## 길 A — 부분 자동화 (Hybrid)

데이터로 표현 가능한 부분만 자동화, 모호한 부분은 사용자가 사후 가감.

**언제 적합한가**
- 판단 기준의 30~70% 정도는 명료한데 나머지는 직감인 경우
- 즉시 가치가 필요한 경우
- 사용자가 매 실행에 5~15분 정도 개입할 의향이 있는 경우

**장점**
- 빠른 시작 (1~2주 안에 첫 결과)
- 자동화된 부분에서 즉시 시간 절감
- 5번 검증 단계가 자연스럽게 작동 — 사용자 개입 지점이 곧 오류 발견 지점

**단점**
- 사용자 개입이 매 실행에 계속 남음
- "자동화 완성"이 아니라 "보조 도구" 수준에 머무를 수 있음
- 개입 부분이 줄어들지 않으면 사용자가 흥미를 잃기 쉬움

## 길 B — 기준 문서화 선행

1~4주 본인의 판단을 로그로 남긴 뒤 자동화 설계로 복귀.

**언제 적합한가**
- 판단 기준이 본인 머릿속에만 있고 글로 풀리지 않는 경우
- 자동화 후 사용자 개입을 최소화하고 싶은 경우
- 다른 사람에게 인수인계가 필요한 영역

**장점**
- 최종 자동화율이 높음
- 본인의 암묵지가 명시지로 전환되는 부수 효과 (인수인계, 신입 교육 자산)
- 자동화 설계 단계에서 시행착오 적음

**단점**
- 즉시 가치 없음 — 1~4주 동안 평소대로 일하면서 기록만 추가
- 기록 자체가 추가 부담
- 기록을 안 하면 영영 자동화 못 함

## 길 C — 보류

자동화 시점이 아직 아니라고 판단되면 메모에 명시하고 다른 후보로.

**언제 적합한가**
- 빈도가 낮음 (월 1~2회 미만이면 자동화 ROI가 안 나옴)
- 판단 기준이 매번 바뀌는 영역 (시장 트렌드, 정성적 평가 등)
- 본인이 이 작업을 즐기는 경우 (자동화하면 오히려 손실)

**장점**
- 잘못된 자동화 시도로 시간 낭비 회피
- 다음 세션에서 더 좋은 후보로 이동

**단점**
- 사용자 입장에서 "결국 아무것도 못 함"으로 느껴질 수 있음 — 메모에 보류 사유를 명시하고 "이런 조건이 바뀌면 다시 보자"는 트리거를 적어두면 회복 가능

## 길 선택 흐름

```
판단 기준이 명료한가?
├─ 예 → 길 분기 불필요. 그냥 자동화 진행.
└─ 아니오
   ├─ 기준의 일부는 명료한가?
   │  ├─ 예 → 길 A 후보
   │  └─ 아니오 → 다음
   ├─ 1~4주 기록할 의향 있는가?
   │  ├─ 예 → 길 B 후보
   │  └─ 아니오 → 다음
   └─ 빈도가 월 2회 이상인가?
      ├─ 예 → 길 A 후보 (개입 많아도 절감 효과 있음)
      └─ 아니오 → 길 C 후보
```

이 흐름은 결정 보조용. 사용자에게 그대로 보여주지 말 것 — 사용자는 자기 케이스의 트레이드오프를 듣고 직접 고른다.

## 트랙 B의 경우

판단 기준이 없는 게 정상(아직 시도 못 한 일이므로). 이때는 "성공 기준"을 묻는 식으로 변형:

- "이번 시도가 끝났을 때 어떤 결과가 손에 있으면 '잘 됐다'고 판단할 것인가?"
- 답이 안 나오면 길 A로 자동 전환 — "일단 한번 돌려보고 결과를 보고 판단"이 트랙 B의 자연스러운 시작점.
