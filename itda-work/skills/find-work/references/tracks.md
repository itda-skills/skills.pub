# 트랙별 탐색 가지

1단계 문제 발굴 시 사용자가 막힐 때 꺼내는 탐색 가지 카탈로그. 한 번에 한 가지만 판다.

## 트랙 A — 반복 업무

- **시간 가지**: 이번 주/달에 가장 오래 걸린 반복 작업
- **짜증 가지**: 가장 하기 싫은데 반복되는 작업
- **실수 가지**: 자주 빼먹거나 틀려서 후속 작업이 생기는 영역
- **의존 가지**: 다른 사람을 기다리거나 다른 사람이 본인을 기다리는 작업
- **검색 가지**: 매번 같은 자료를 찾아 헤매는 작업

## 트랙 B — 미지의 문제

- **결심 가지**: "언젠가 해봐야지" 하면서 미뤄둔 일
- **답답함 가지**: 정보가 부족해서 결정이 막힌 영역
- **가설 가지**: 검증하고 싶은데 손이 안 가는 가설
- **비교 가지**: "다른 회사/사람은 어떻게 하지" 궁금한 영역
- **발굴 가지**: 새로 발굴하고 싶은 대상(고객 / 시장 / 회사 / 기회)

## 혼합 트랙 처리

한 후보가 두 성격을 모두 가질 때(예: "가끔 하긴 하지만 체계 없는 경쟁사 조사"):

1. "더 자주 마주치는 게 어느 쪽인가요?"로 우세 트랙 결정
2. 우세 트랙의 종료 조건과 가지를 주축으로 사용
3. 메모 헤더에 `트랙: 혼합 — 우세: A` 형식으로 둘 다 기록
4. 8번 "미해결 / 보류" 섹션에 다른 트랙 시각의 추가 질문을 메모로 남겨, 다음 세션에서 재방문 가능하게 함

## 트랙 분류가 흔히 틀리는 경우

- "가끔 한다" → 빈도가 낮으면 트랙 B로 분류해도 무방 (반복 자동화 ROI가 안 나오는 영역)
- "매번 다르게 한다" → 트랙 A처럼 보이지만 판단 기준이 흔들리면 트랙 B의 성공 기준 질문으로 우회
- "해야 하는데 안 하고 있다" → 트랙 B. 트랙 A로 잡으면 "현재 소요 시간 0분"이라는 무의미한 답이 나온다
