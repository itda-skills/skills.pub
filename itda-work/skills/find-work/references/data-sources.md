# 데이터 출처 유형 + 실행 경로 (Claude Cowork 전용)

3단계 "데이터·실행 경로 도출"에서 Claude가 적극적으로 후보를 제시할 때의 카탈로그.

## 데이터 출처 유형

### 사내 데이터
- **본인 PC**: 로컬 파일 (Excel, PPT, PDF, HWP 등). 가장 접근 쉬움.
- **공유 드라이브**: 회사 OneDrive / Google Drive / NAS. 권한 확인 필요.
- **사내 시스템**: ERP / CRM / 그룹웨어. API 제공 여부와 IT 부서 협조가 관건. 보통 가장 큰 진입 장벽.
- **사내 메일·메신저**: Outlook / Gmail (사내 계정) / 슬랙 / 카카오워크 / 팀즈.

### 외부 공개 데이터
- **공공데이터포털 (data.go.kr)**: 정부·지자체 API. 무료, 신청 후 API 키 발급.
- **DART (전자공시)**: 상장사·외부감사 대상 기업 공시. 무료 API.
- **KOSIS (국가통계)**: 통계청 데이터. 무료 API.
- **법령정보센터, 국토부, 통계지리정보(SGIS)**: 도메인별 무료 API 다수.

### 외부 유료 / 제약 있는 데이터
- **상용 API**: 잡코리아, 사람인, 부동산 데이터, 금융 데이터 등. 비용 발생.
- **스크래핑 대상 사이트**: robots.txt, 이용약관 반드시 확인. 로그인 필요 사이트는 Claude in Chrome 경로 고려.
- **소셜 / 커뮤니티**: 네이버 카페·블로그, 디시·뽐뿌·클리앙 등. 자동화 제약 큰 편.

### 이메일 / 메신저
- **Gmail / Outlook**: API 제공. OAuth 인증 필요.
- **슬랙 / 카카오워크 / 팀즈**: API 제공. 봇 토큰 발급.
- **카카오톡 (개인)**: 비즈니스 채널을 제외하면 자동화 제약이 매우 큼. 우회 자체를 비권장.

### 반복 자료
- 매주/매월 같은 위치에서 받아오는 양식·리포트. 형식이 안정적이면 자동화 ROI가 매우 높다.

## 실행 경로 (Claude Cowork 전용)

### Cowork 단독
- 브라우저 + 로컬 파일 처리.
- **가장 빠른 시작.** 트랙 B 첫 시도는 이 경로가 디폴트.
- 한계: 스케줄링 없음, 대량 처리 시 세션 한계.

### Cowork + Claude in Chrome
- 로그인이 필요한 사이트 자동화.
- 사내 시스템 / 사내 메일이 웹 인터페이스인 경우.
- 한계: 사이트 UI 변경에 취약.

### Cowork + 외부 워크플로 도구 (n8n / Make)
- 스케줄링·대량 처리가 필요할 때 실행을 외부로 위임.
- Cowork는 "한 번 설계 + 외부에서 반복 실행" 분업.
- 한계: 외부 도구 학습 비용. n8n self-host의 경우 인프라 필요.

### Cowork + API 직접 호출
- 개발 가능한 사용자에 한정.
- 가장 강력하지만 진입 장벽 최고.

## 경로 선택 휴리스틱

| 상황 | 권장 1순위 |
|------|-----------|
| 트랙 B 첫 시도 (가치 검증 단계) | Cowork 단독 |
| 매주 1회 이상 반복 + 로그인 필요 사이트 | Cowork + Claude in Chrome |
| 매일 자동 실행 + 대량 처리 | Cowork + n8n/Make |
| 데이터가 본인 PC에만 있음 | Cowork 단독 |
| 사내 시스템 API가 있음 | Cowork + API 직접 호출 |
| 사내 시스템에 API 없음, 웹 UI만 있음 | Cowork + Claude in Chrome |

권장 1순위는 어디까지나 출발점. 사용자 환경(개발 가능 여부, 빈도, 트랙)에 따라 조정한다.
