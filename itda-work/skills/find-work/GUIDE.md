# find-work 운영자·강사 가이드

이 문서는 스킬을 호출할 때마다 로드되지 않습니다(SKILL.md 본문과 분리). 강의·온보딩 운영자용.

## 강의 시연 흐름 (권장 30분)

1. **연결 폴더 미리 띄우기** (시연 전): 화면 공유로 `find-work/` 폴더를 Finder/Explorer에 열어둔다. 메모 파일이 떨어지는 순간이 가장 강한 "아하 모먼트".
2. **트랙 A 시연 (10분)**: "매주 거래처 메일 분류" 같은 익숙한 케이스로 1~4단계 풀 사이클. 메모가 폴더에 떨어지는 걸 확인.
3. **트랙 B 시연 (10분)**: "신사업 후보 발굴" 같은 미지 케이스. 1단계 종료조건이 트랙 A보다 까다롭다는 점(구체 산출물 명시)을 의도적으로 노출.
4. **종료 신호 시연 (5분)**: 일부러 "여기까지" 외쳐 4단계 점프. 미완 필드가 `(미정 — 다음 세션에서 확인)`으로 남는 걸 보여줌.
5. **이어 작업 시연 (선택, 5분)**: 첫 시연 결과 메모 파일을 다음 세션에 첨부해 "이거 이어서 더 파보자"고 호출 → 사용자가 명시할 때만 이전 메모를 활용한다는 점을 강조.

## 의도된 시연 포인트

- **1단계에서 "잘 모르겠어요" 한 번**: 우회 질문으로 받아주는 걸 보여주면 수강생 진입 장벽이 낮아짐.
- **2단계 3번(판단 기준)에서 막히는 시나리오**: 길 A 부분 자동화가 자연스러운 출구임을 시연. "정답이 없을 때도 자동화는 가능하다" 메시지.
- **트랙 A·B 모두 시연**: 수강생이 본인 케이스를 어디에 위치시킬지 결정하기 쉬워짐.

## 자주 묻는 질문 대비

**Q. 두 트랙에 다 해당되면?**
→ 우세한 트랙으로 진행. 메모에는 두 성격 모두 표기. 1단계 종료 직전 트랙 합의 질문에서 사용자가 직접 결정.

**Q. 판단 기준을 못 풀면 자동화 못 하나요?**
→ 길 A(부분 자동화)로 시작 가능. "AI가 못 판단하는 부분만 사람이 사후 가감"하는 hybrid 패턴이 현실적인 경우가 많음. 다만 강사가 이걸 "권장"으로 표현하지 말 것 — 세 길의 트레이드오프를 객관적으로 제시.

**Q. 회사 데이터를 외부에 노출해도 되나요?**
→ 3단계 데이터 출처 단계에서 사용자 본인이 판단할 영역. Claude는 외부 API/스크래핑 후보를 제시하되 법무·보안 검토는 사용자 책임임을 명시.

## 운영 팁

- 결과 메모를 모아 익명화 후 사례집을 만들면, 다음 기수 시연용 베이스라인이 된다.
- 메모는 폴더에 그냥 쌓일 뿐 스킬이 자동으로 끌어오지 않는다. 수강생이 본인 메모를 명시적으로 다시 첨부해야 이어 작업이 된다 — 이 점을 강의에서 명확히 짚어줄 것.
